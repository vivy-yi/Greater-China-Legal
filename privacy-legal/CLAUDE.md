## 法律框架

### 核心法规

| 法规 | 生效日期 | 主管机关 | 适用场景 |
|---|---|---|---|
| 《个人信息保护法》（PIPL）| 2021-11-01 | 国家网信办 | 个人信息处理活动 |
| 《数据安全法》（DSL）| 2021-09-01 | 国家网信办 | 数据处理活动安全 |
| 《网络安全法》（CSL）| 2017-06-01 | 公安部/网信办 | 网络运营者义务 |
| GB/T 35273-2020 | 2020-10-01 | 网信办参考 | 个人信息安全规范（推荐标准）|
| 《个人信息安全规范》| 2021-06-01（修订）| 网信办参考 | 操作指引 |
| 《数据出境安全评估办法》| 2022-09-01 | 国家网信办 | 个人信息出境 |
| 《标准合同办法》| 2023-06-01 | 国家网信办 | 个人信息出境标准合同 |
| 《个人信息保护法》司法解释 | 2024-01-01 | 最高法 | 司法适用 |

### 关键定义

**个人信息：** 以电子或其他方式记录的与已识别或可识别的自然人有关的各种信息，不包括匿名化处理后的信息。

**敏感个人信息：** 一旦泄露或非法使用，可能导致自然人人格尊严受到侵害或人身、财产安全受到危害的个人信息。包括：生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹、不满14周岁未成年人信息等。

**个人信息处理者：** 在个人信息处理活动中自主决定处理目的、处理方式的组织、个人。

**受托人：** 接受委托处理个人信息的第三方（数据处理外包服务商）。

## 适用场景判断

在应用 playbook 前，判断所处理数据的性质：

| 场景 | 适用法规 | 核心义务 |
|---|---|---|
| 处理员工/用户个人信息 | PIPL | 告知+同意（或法定情形）|
| 委托处理（外包）| PIPL 第21条 | 签订委托合同，监督受托人 |
| 跨境传输（出境）| PIPL 第38-43条 | 安全评估/认证/SCC/其他 |
| 向第三方提供 | PIPL 第23条 | 告知+单独同意 |
| 使用个人信息进行自动化决策 | PIPL 第24条 | 告知+便捷拒绝权 |
| 处理不满14周岁未成年人信息 | PIPL 第31条 | 须取得监护人同意 |
| 涉及关键信息基础设施运营者 | PIPL 第40条 | 本地化存储+安全评估 |
| 涉及重要数据处理 | DSL | 数据分类分级、安全保障 |

## 角色判断

**我方是个人信息处理者（controller）还是受托处理者（processor）？**

- **处理者（Controller）：** 自己决定处理目的和方式——适用本 playbook 的 controller 侧
- **受托处理者（Processor）：** 按委托合同约定处理——适用本 playbook 的 processor 侧

## Playbook — Controller 侧（数据控制方）

### DPA — Controller 标准位置

| 条款 | 可接受标准 | 不可接受 |
|---|---|---|
| 适用法律 | 中国法律（PIPL/DSL/CSL） | 境外法律（除非有充分理由）|
| 争议解决 | 中国法院/仲裁 | 境外法院/仲裁 |
| 数据处理目的限制 | 仅用于约定的明确目的 | 泛化为"服务改进"等 |
| 数据保留 | 明确期限或触发条件 | 无期限或"按法律要求" |
| 数据安全 | 符合 GB/T 35273 或等效措施 | 无安全措施或描述模糊 |
| 子处理者 | 须通知并获得书面同意 | 允许自由转委托 |
| 数据主体权利 | 配合响应 PIPL 规定的权利请求 | 拒绝或推诿 |
| 违约通知 | 24小时内通知（重大事件）| 无通知义务或超长期限 |
| 跨境传输 | 须符合 PIPL 第38-43条规定 | 无出境合规机制 |

### Controller 标准义务清单

**处理前：**
- [ ] 建立个人信息处理清单（处理目的、类型、保存期限）
- [ ] 完成个人信息保护影响评估（PIA）
- [ ] 制定隐私政策并公开
- [ ] 获取必要同意（单独同意用于敏感信息、跨境）

**处理中：**
- [ ] 实施数据安全技术措施
- [ ] 监督受托人处理行为
- [ ] 记录处理活动（台账）

**变更时：**
- [ ] 重新评估影响（如处理目的、方式重大变更）
- [ ] 通知数据主体（如适用）

## Playbook — Processor 侧（受托处理方）

### DPA — Processor 标准位置

| 条款 | 可接受标准 | 不可接受 |
|---|---|---|
| 处理目的限制 | 仅按 controller 指示处理 | 用于其他目的包括"服务改进" |
| 处理范围 | 限于必要数据 | 要求提供超出必要的数据 |
| 子处理者 | 须获得 controller 事先书面同意 | 自由转委托 |
| 数据返还/删除 | 合同终止后返还或删除 | 保留数据无明确期限 |
| 安全措施 | 符合 GB/T 35273 等效要求 | 无具体安全要求 |
| 审计配合 | 配合 controller 审计 | 拒绝审计或仅提供自我声明 |
| 泄露通知 | 发现泄露后立即通知（不超过24小时）| 无通知义务 |
| 适用法律 | 中国法律 | 境外法律 |

## 个人信息主体权利响应

PIPL 第44-50条规定了数据主体的各项权利：

| 权利 | 响应时限 | 可拒绝情形 |
|---|---|---|
| 查询、查阅 | 15日内 | 涉及他人隐私、商业秘密 |
| 复制 | 15日内 | 技术上无法实现 |
| 更正、补充 | 15日内 | 信息准确无需更正 |
| 删除 | 15日内 | 法定保存义务、行使法定职权 |
| 撤回同意 | 即时生效 | — |
| 可携带 | 复杂（无细则）| 无实施细则暂不承诺 |
| 解释说明 | 15日内 | — |
| 对自动化决策拒绝 | 须提供关闭或调整选项 | — |

**DSAR（数据主体权利请求）响应流程：**
1. 验证请求人身份（防止冒用）
2. 确认请求属于数据主体权利范围
3. 在15日内响应（复杂情况可延长至45日）
4. 如拒绝，说明原因和救济途径

## 个人信息保护影响评估（PIA）

**须进行PIA的情形（PIPL第55-56条）：**
- 处理敏感个人信息
- 利用个人信息进行自动化决策
- 委托处理、共同处理、向第三方提供、跨境提供
- 其他对个人权益有重大影响的处理活动

**PIA评估内容：**
1. 个人信息处理目的、处理方式是否合法、正当、必要
2. 对个人权益的影响及安全风险
3. 所采取的保护措施是否合法、有效并与风险相适应

**PIA记录保存：** 不少于3年

## 跨境数据传输

### 合规路径

| 路径 | 适用条件 | 主管机关 |
|---|---|---|
| **安全评估**（网信办）| 关键信息基础设施运营者 / 处理百万人以上个人信息 / 累计向境外提供10万人个人信息或1万人敏感信息 | 国家网信办 |
| **个人信息保护认证** | 须通过专业机构认证 | 认证机构 |
| **标准合同（SCC）** | 非上述情形，且与境外接收方签订网信办标准格式合同 | 无需事先备案 |
| **其他** | 法律法规网信办规定的其他方式 | — |

### 标准合同要点

- 境外接收方须承诺接受中国法律管辖
- 须提供充分保障的证据
- 须在省级网信部门备案（合同签订后10个工作日内）
- 境外接收方须配合数据主体行使权利
