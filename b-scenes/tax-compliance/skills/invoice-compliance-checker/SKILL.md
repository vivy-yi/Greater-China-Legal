# /invoice-compliance-checker — 发票合规检查

## 加载上下文

**首次使用时：** 读取 `../CLAUDE.md` 获取虚开认定标准/善意取得条件。

## 工作流程

1. 读取 `../CLAUDE.md` → 获取虚开标准/善意取得条件
2. 检查发票基本信息
3. 核查三流合一（合同/发票/资金）
4. 判断是否虚开或善意取得
5. 输出合规评估

## 发票合规核心：三流合一

```
合同流：货物/服务合同与发票内容一致
发票流：发票开具内容与实际交易一致
资金流：付款方/收款方与合同一致

三流不一致 → 涉嫌虚开
```

## 虚开增值税发票认定

| 类型 | 行为 | 刑事责任 |
|---|---|---|
| 为他人虚开 | 无真实交易而开具 | 刑事责任 |
| 让他人为己虚开 | 接受虚开 | 刑事责任 |
| 介绍他人虚开 | 居间介绍 | 刑事责任 |
| 自己为自己虚开 | 无实际业务 | 刑事责任 |

## 善意取得判断

```
四要件（须同时满足）：
1. 购货方不知道取得的发票是虚开的
2. 销售方使用的是其所在省的发票
3. 购货方已按规定付款（付至销售方账户）
4. 购货方按发票载明的货物名称等申报抵扣进项

处理结果：
- 善意取得 → 不得抵扣进项，须转出
- 非善意取得 → 须补缴税款 + 滞纳金 + 罚款
```
