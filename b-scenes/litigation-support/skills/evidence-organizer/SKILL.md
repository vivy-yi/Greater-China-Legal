# /evidence-organizer — 证据组织整理

## 加载上下文

**首次使用时：** 读取 `../CLAUDE.md` 获取证据规则/举证责任分配。

## 工作流程

1. 读取 `../CLAUDE.md` → 获取证据规则/举证责任
2. 列出争议焦点
3. 收集和整理证据
4. 按焦点分组
5. 输出证据目录

## 证据目录格式

```
证据目录

原告/反诉被告提交：

证据1：[证据名称]
  证据类型：[书证/物证/视听资料等]
  证据来源：[自行制作/第三方机构]
  证明内容：[具体证明事实]
  页码：[P1-P5]

证据2：[...]
```

## 按案件类型组织证据

### 合同纠纷证据组织

| 争议焦点 | 所需证据 |
|---|---|
| 合同成立 | 合同原件/往来邮件/签约视频 |
| 合同履行 | 发货单/验收单/付款记录/沟通记录 |
| 违约事实 | 违约行为证据/损失计算依据 |
| 损害赔偿 | 损失明细/鉴定意见/市场数据 |

### 侵权纠纷证据组织

| 争议焦点 | 所需证据 |
|---|---|
| 侵权行为 | 侵权现场照片/视频/鉴定意见 |
| 损害结果 | 医疗票据/损失评估/鉴定意见 |
| 因果关系 | 鉴定意见/专家意见 |
| 免责事由 | 不可抗力证据/受害人过错证据 |

### 举证责任分配

| 案件类型 | 举证方 |
|---|---|
| 合同纠纷 | 主张合同成立/生效/履行方 |
| 侵权纠纷（一般） | 原告：侵权+损害+因果关系 |
| 侵权纠纷（特殊） | 被告：举证无过错/免责事由 |
| 劳动争议 | 用人单位：解除合法性 |

## 电子证据处理

```
微信/QQ记录：
- 须证明账号归属（实名认证页面截图）
- 须证明完整未删改（录屏+时间戳）

邮件：
- 须证明发送方和接收方
- 须证明邮件内容完整性

系统日志：
- 须证明系统正常运行
- 须证明日志未被篡改
```
