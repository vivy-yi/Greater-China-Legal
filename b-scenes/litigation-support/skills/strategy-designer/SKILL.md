# /strategy-designer — 诉讼策略制定

## 加载上下文

**首次使用时：** 读取 `../CLAUDE.md` 获取诉讼程序/策略选择规则。

## 工作流程

1. 读取 `../CLAUDE.md` → 获取程序规则/胜诉因素
2. 分析案件事实和证据
3. 确定诉讼地位（原告/被告）
4. 设计诉讼策略
5. 输出策略建议

## 原告 vs 被告策略差异

### 原告策略（攻击型）

```
核心目标：胜诉 + 获得最大赔偿/实现最大诉求

策略要点：
1. 诉讼请求设计：最大化但合理（避免被驳回）
2. 证据组织：完整证据链，避免证据突袭
3. 管辖选择：如有选择，选择对我方有利的法院
4. 诉讼时机：注意时效，及时起诉
5. 保全申请：及时申请财产保全，防止转移资产
```

### 被告策略（防御型）

```
核心目标：驳回/减少责任/反诉

策略要点：
1. 答辩要点：逐条反驳原告主张
2. 证据突袭：发现新证据后及时提交
3. 反诉：如存在反诉理由，考虑反诉
4. 时效抗辩：如诉讼时效届满，提出时效抗辩
5. 责任减轻：提出免责/减责事由
```

## 争议焦点分析

```
分析步骤：
1. 列出原告主张和被告抗辩
2. 找出核心争议焦点（通常1-3个）
3. 分析每个焦点：谁负举证责任
4. 评估每个焦点我方胜诉概率
5. 确定主要战场和次要战场
```

## 诉讼请求设计

### 合同纠纷诉讼请求

| 类型 | 内容 |
|---|---|
| 继续履行 | 要求继续履行合同 |
| 解除合同 | 要求解除合同 |
| 支付价款 | 要求支付货款/服务费 |
| 违约金 | 要求支付违约金 |
| 损害赔偿 | 要求赔偿实际损失 |

### 设计原则

```
1. 最大化但不虚高：诉讼请求须有事实和法律依据
2. 主次分明：核心请求+备选请求
3. 具体明确：金额须有计算依据
4. 可执行：须考虑对方执行能力
```
