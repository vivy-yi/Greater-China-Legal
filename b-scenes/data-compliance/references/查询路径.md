## 数据源优先级

| 优先级 | 数据源 | 说明 |
|---|---|---|
| 1 | [YD] 法律精灵 | PIPL/CAC规定+最新监管动态 |

## 常用查询场景

### 场景1：某业务是否构成个人信息处理

**查询语句：**
```
法律精灵："[数据类型] 是否构成个人信息 PIPL" 或 "[场景] 个人信息 认定"
```

**判断标准（PIPL第4条）：**
```
构成个人信息：可识别特定自然人的各种信息
- 直接识别：姓名、身份证号、手机号
- 间接识别：设备ID、IP地址、Cookie、浏览记录
- 特殊敏感：生物识别、宗教信仰、医疗健康、金融账户、行踪轨迹
```

---

### 场景2：敏感个人信息处理是否合规

**查询语句：**
```
法律精灵："[数据类型] 敏感个人信息 单独同意 PIPL" 或 "敏感个人信息处理规则"
```

**须单独同意的敏感信息（PIPL第28条）：**
```
- 生物识别（人脸/指纹）
- 宗教信仰
- 特定身份
- 医疗健康
- 金融账户
- 行踪轨迹
- 未满14周岁未成年人信息
```

---

### 场景3：数据出境合规路径选择

**查询语句：**
```
法律精灵："个人信息出境 安全评估 标准合同 条件" 或 "个人信息出境合规 路径"
```

**判断标准（见 pipl-assessment）：**
```
须安全评估：
- 处理个人信息>100万人
- 敏感个人信息出境
- 累计出境>10万人一般信息

可选择标准合同：
- 不满足安全评估条件
- 非敏感个人信息
- 出境人数<10万
```

---

### 场景4：CAC执法案例查询

**查询语句：**
```
CAC官网："[业务类型] 违规 处罚" 或 "APP超范围收集 个人信息 案例"
```

**常用查询入口：**
```
[GOV] CAC官网：https://www.cac.gov.cn/
[GOV] 网信办执法公告：http://www.cac.gov.cn/
[GOV] 全国信息安全标准化技术委员会：https://www.tc260.org.cn/
```

---

## 查询结果处理

### 情形一：查到明确的CAC执法案例
→ 直接对照处罚依据和金额

### 情形二：查到类似违规场景
→ 评估自身是否涉及类似问题

### 情形三：查到监管动态（征求意见稿）
→ 标注"尚未正式生效，关注进展"

### 情形四：查不到类似案例
→ 使用PIPL明确规定，注明"无直接违规先例"
