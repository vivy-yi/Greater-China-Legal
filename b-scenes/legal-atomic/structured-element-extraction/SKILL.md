# /structured-element-extraction — 结构化要素清单

## 核心概念

```
结构化要素 = 最小可判断单元

每个要素必须满足：
- 原子性：不可再进一步有意义地拆分
- 可判断性：可对其作出"满足/不满足/待查"的明确判断
- 独立性：各要素之间逻辑上互不包含（MECE原则）
- 可追溯性：每个要素可追溯到具体法律条文或法理依据
```

### 要素状态标注

| 状态 | 符号 | 含义 |
|---|---|---|
| 已满足 | ✅ | 现有材料可充分证明该要素成立 |
| 未满足 | ❌ | 现有材料明确表明该要素不成立 |
| 待查证 | ⚠️ | 现有材料不足以判断，需补充信息 |
| 不适用 | ➖ | 该要素在本案情形下不需要考察 |
| 存在争议 | 🔶 | 该要素的满足与否存在不同解释空间 |

### MECE原则

```
相互独立（Mutually Exclusive）：
  ❌ 错误：并列"过错"和"故意" → 包含关系
  ✅ 正确：将"过错"列为上位，"故意/过失"为其下位

完全穷尽（Collectively Exhaustive）：
  ❌ 错误：遗漏"因果关系"要素
  ✅ 正确：覆盖该法律问题的全部判断维度
```

## 工作流程

### Phase 1：识别问题类型与确定要素框架

```
步骤1.1：识别法律问题类型
  ├── 合同纠纷类（成立、效力、履行、违约、解除）
  ├── 侵权责任类（一般侵权、特殊侵权、产品责任）
  ├── 犯罪构成类（四要件/三阶层）
  ├── 行政行为类（合法性审查五要素）
  └── 程序事项类（管辖、时效、主体资格）

步骤1.2：调取对应标准要素框架
  - 优先使用法定构成要件
  - 法定要件不明确时，参照司法解释
  - 均无明确规定时，参照通说学理

步骤1.3：确认层级结构
  - 一级要素：法定构成要件（必须全部覆盖）
  - 二级要素：对一级要素的细化分解
  - 三级要素：具体事实判断点（可选，视复杂度决定）
```

### Phase 2：逐项提取与填充

```
步骤2.1：逐一对照要素框架，从原始材料提取
  - 每个要素必须独立提取，不可跳过
  - 直接引用原始材料中的关键语句
  - 无法提取的要素标注为"待查证"

步骤2.2：为每个要素标注状态（✅/❌/⚠️/➖/🔶）

步骤2.3：为每个要素记录证据/依据来源
  - 事实要素：标注来源于哪份材料的哪个部分
  - 法律要素：标注对应的法条编号
```

### Phase 3：三重完整性校验

```
第一重：框架完整性
  → 问：是否遗漏了标准框架中的任何一级要素？
  → 方法：将提取结果与标准框架逐一比对

第二重：事实覆盖度
  → 问：原始材料中是否有尚未被任何要素"消化"的重要事实？
  → 方法：重新通读原始材料，标记未被引用的事实片段

第三重：逻辑自洽性
  → 问：各要素之间是否存在逻辑矛盾？
  → 方法：检查要素间的逻辑关系
```

### Phase 4：输出与交接

```
步骤4.1：按标准模板输出结构化要素清单
步骤4.2：附加完整性声明（通过/未通过三重校验）
步骤4.3：标注整体置信度
步骤4.4：列出待查证事项清单
步骤4.5：确认可以进入下一阶段 / 需要补充信息后再继续
```

## 常见领域要素框架

### 合同纠纷

| 一级要素 | 法源 | 二级要素 |
|---|---|---|
| 合同成立 | 民法典469 | 要约+承诺 |
| 合同生效 | 民法典502 | 主体合格+意思表示真实+内容合法 |
| 合同履行 | 民法典509 | 按约定全面履行 |
| 违约责任 | 民法典577 | 不履行/不完全履行 |

### 侵权纠纷

| 一级要素 | 法源 | 二级要素 |
|---|---|---|
| 加害行为 | 民法典1165 | 行为违法性 |
| 损害事实 | 民法典1165 | 财产/人身损害 |
| 因果关系 | 民法典1165 | 相当因果关系 |
| 主观过错 | 民法典1165 | 故意/过失 |
