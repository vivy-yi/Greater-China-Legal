# /dispute-and-performance-risk — 识别争议与履约风险

## 核心概念

### 争议风险 vs 履约风险

| 维度 | 争议风险 | 履约风险 |
|---|---|---|
| 定义 | 可能引发当事人之间分歧、对抗的风险 | 一方可能无法按约定履行义务的风险 |
| 时间维度 | 签订前/履行中/履行后任何阶段 | 主要在履行阶段 |
| 表现形式 | 条款理解分歧、权利义务争议 | 迟延/不完全/不能/拒绝履行 |
| 后果 | 协商、调解、仲裁、诉讼 | 违约责任、解除、赔偿 |

### 三层风险模型

```
第一层：条款层风险（合同文本本身的缺陷）
  - 条款模糊/遗漏/冲突
  - 格式条款未合理提示
  - 效力瑕疵（无效/可撤销）

第二层：交易层风险（交易结构和对手方）
  - 交易结构缺陷
  - 对手方信用/履约能力
  - 利益失衡

第三层：环境层风险（外部变化）
  - 法律政策变化
  - 市场变动
  - 不可抗力
```

## 工作流程

### Phase 1：信息收集与预处理

```
步骤1.1：确认分析对象
  - 明确合同/协议/交易安排的范围
  - 确认当事人身份与角色
  - 确认用户立场（代表哪一方）
  - 确认分析侧重点（全面审查 vs 特定条款）

步骤1.2：收集背景信息
  - 合同签订背景与商业目的
  - 当事人基本情况（资质、信用、履约能力）
  - 合同履行当前状态
  - 已知的争议或分歧
  - 行业惯例与监管要求

步骤1.3：文本预处理
  - 通读全文，建立整体框架认知
  - 标注关键条款（核心权利义务、违约责任、争议解决）
  - 识别合同类型（买卖/租赁/服务/合作/投资）
  - 标注信息缺失或模糊之处
```

### Phase 2：三层风险识别

```
步骤2.1：条款层风险
  检查清单：
  □ 条款是否存在模糊之处？
  □ 是否有遗漏的重要条款？
  □ 条款之间是否存在冲突？
  □ 是否涉及格式条款？是否尽到提示说明义务？
  □ 合同是否存在效力瑕疵（违反强制性规定/违背公序良俗）？

步骤2.2：交易层风险
  检查清单：
  □ 交易结构是否存在缺陷？
  □ 对手方的信用和履约能力如何？
  □ 各方权利义务是否对等？
  □ 是否存在利益失衡的条款？

步骤2.3：环境层风险
  检查清单：
  □ 是否存在法律政策变化风险？
  □ 是否存在市场变动风险？
  □ 是否存在不可抗力条款？范围是否合理？
  □ 是否存在情势变更条款？
```

### Phase 3：风险评级与应对

```
步骤3.1：风险评级
  - 🔴 高风险：可能导致合同目的无法实现或重大损失
  - 🟠 中风险：可能导致争议或部分损失
  - 🟡 低风险：可能导致不便或轻微损失

步骤3.2：应对建议
  对每个风险点提出具体建议
```
