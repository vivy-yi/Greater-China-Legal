# /legal-document-summarization — 法律文书摘要

## 文书类型与摘要重点

| 文书类型 | 核心功能 | 摘要重点 |
|---|---|---|
| 判决书 | 解决实体争议 | 事实认定、法律适用、裁判理由、判决结果 |
| 裁定书 | 解决程序事项 | 程序背景、申请/异议内容、裁定依据 |
| 调解书 | 确认双方合意 | 争议事项、调解协议内容、履行安排 |
| 仲裁裁决书 | 仲裁庭终局裁决 | 仲裁请求、事实认定、仲裁庭意见、裁决结果 |
| 行政处罚决定书 | 行政制裁 | 违法事实、处罚依据、处罚种类和幅度 |

---

## 六要素提取框架

```
┌─────────────────────────────────────────┐
│  1. 案件信息：谁、何时、何地、何机构      │
│  2. 事实背景：与结论相关的事实           │
│  3. 争议焦点：核心法律问题               │
│  4. 法律依据：援引了哪些规范              │
│  5. 结论结果：最终如何处理               │
│  6. 说理过程：如何从事实+规范推导结论     │
└─────────────────────────────────────────┘
```

---

## 工作流程

### 步骤1：识别文书类型
```
根据文书类型选择摘要模板
```

### 步骤2：提取六要素
```
逐一提取，忠实原文，不添加
```

### 步骤3：生成结构化摘要
```
按文书类型对应的模板输出
```

### 步骤4：质量检查
```
- 是否忠实于原文？
- 是否客观中立？
- 是否突出核心？
- 是否区分了文书类型？
```

---

## 输出格式（判决书示例）

```
## 案件摘要

### 案件信息
- 法院：[名称]
- 案号：[案号]
- 日期：[日期]
- 当事人：[原告] vs [被告]

### 事实背景
[与裁判结论相关的核心事实，2-3句]

### 争议焦点
1. [焦点一]
2. [焦点二]

### 法律依据
- 《[法典]》第X条第X款
- [司法解释]

### 裁判结论
[判决主文]

### 说理过程
[法院如何从事实+规范推导出结论，1-2段]
```
