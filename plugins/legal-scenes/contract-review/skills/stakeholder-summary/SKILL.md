# /stakeholder-summary — 合同审查业务摘要生成

## 加载上下文

本技能通常在其他合同审查技能（如 vendor-agreement-review, saas-msa-review, nda-review, risk-clause-database）完成后调用。首次使用时，读取 `../CLAUDE.md` 获取公司基本信息、业务描述和风险等级定义。

## 核心概念

法律合同中90%的内容业务部门不需要深入了解。好的业务摘要应当回答三个问题：(1) 我能签吗？(2) 签了要承担什么核心义务？(3) 有哪些事必须注意/绝对不能做？本技能将专业法律审查结论转化为目标受众（采购/财务/业务负责人/管理层）可直接执行的行动指南。

## 工作流程

### 第一步：确认受众与信息层级

| 受众 | 关注重点 | 输出格式 | 详细程度 |
|------|---------|---------|---------|
| 采购/业务负责人 | 能不能签、价格、交付、违约责任 | 简洁表格+要点 | 中 |
| 财务/CFO | 付款条款、担保、赔偿责任上限、续期义务 | 金额敏感性分析 | 高 |
| 高管/决策层 | 整体风险评估、商业考量、是否批准 | 2分钟决策摘要 | 低 |
| 法务内部同事 | 详细条款分析、法律依据、修改建议 | 完整法律备忘录 | 最高 |

### 第二步：核心信息提取

从原始审查结果中提取以下核心信息：

```
1. 能不能签：GREEN（可签）/ YELLOW（建议修改） / RED（建议不签）
2. 核心义务：我方必须做什么
3. 核心权利：对方须为我方做什么
4. 金额/付款：多少钱、何时付、怎么付
5. 关键日期：签约日期、生效日期、终止日期、续期通知日期
6. 责任上限：如果出问题，我方最多赔多少
7. 特别红线：绝对不能做的行为或绝对不能接受的条款
8. 下一步：需要谁审批、多久完成
```

### 第三步：按目标受众定制输出

#### 格式一：采购/业务负责人摘要（2分钟版）

```
## [合同名称] — 业务摘要

**总体结论：[可签/建议修改后签/建议不签]**

### 我方核心义务
1. [义务一]
2. [义务二]

### 关键时间节点
| 事项 | 日期/期限 |
|------|---------|
| 合同生效 | [___] |
| 自动续期通知 | 须提前[___]天书面通知 |
| 合同到期 | [___] |

### 我方必须避免的
- [红线一：如逾期付款]
- [红线二：如超出授权范围使用]

### 如果出问题
- 我方责任上限：[金额或标准]
- 对方责任上限：[金额或标准]

### 下一步
- [ ] [___]审批
- [ ] [___]签署
- [ ] 关注截止日期：[___]
```

#### 格式二：管理层决策摘要（30秒版）

```
## [合同名称] — 决策摘要

**建议：** ✅ 批准 / ⚠️ 待谈判 / ❌ 拒绝

| 决策要点 | 说明 |
|---------|------|
| 交易对手 | [名称/背景] |
| 合同金额 | [___]元/年 |
| 合同期间 | [___]年 |
| 风险等级 | 低/中/高 |
| 核心风险 | [一句话描述] |
| 审批路径 | [业务负责人/CFO/董事会] |

### 法务建议
[3句话以内的核心建议]
```

#### 格式三：财务/CFO定向摘要

```
## [合同名称] — 财务关注要点

### 付款结构
| 阶段 | 金额 | 时间 | 条件 |
|------|------|------|------|
| 首期 | [___] | [___] | [___] |
| 后续 | [___] | [___] | [___] |

### 责任上限分析
- 我方赔偿责任上限：[___]
- 我方赔偿责任下限（如无上限）：[___]%合同金额
- 对方赔偿责任上限：[___]
- 保证金/保函：[___]

### 自动续期风险
- 当前合同期间：[___]
- 续期机制：[自动/手动]
- 通知窗口：[___]天
- 价格是否锁定：[是/否/涨幅限制___]

### 退出成本
- 提前终止费用：[___]
- 数据迁移费用：[___]
- 违约金：[___]
```

#### 格式四：法务内部备忘录（适用于移交决策）

```
## [合同名称] — 法务审查备忘录

**审查类型：** 供应商合同 / NDA / SaaS / 其他

### 关键条款偏离
| 条款 | 标准版本 | 当前版本 | 偏离影响 |
|------|---------|---------|---------|
| [___] | [___] | [___] | [___] |

### 谈判优先级
1. [强制性修改项]
2. [争取性修改项]
3. [可让步项]

### 风险等级与理由
- 风险：[GREEN/YELLOW/RED]
- 理由：[___]

### 路由建议
- [业务自行签署/法务内部签署/外部律师审核]
- 审批人：[___]
```

### 第四步：升级决策门

> "本技能生成的业务摘要依赖底层合同审查的质量。如果原始审查结果不完整或存在未识别的高风险条款，摘要可能产生误导。以下情形建议直接向业务部门发出风险警示并将完整审查备忘录抄送法务负责人：(1)摘要结论为RED（建议不签）但业务部门仍想推进；(2)涉及重大金额（超过500万元）的摘要须由法务负责人签字确认；(3)摘要中的风险等级与底层审查结果不一致时（须先确认底层审查的准确性）。本摘要为沟通工具，不构成法律意见。最终签署须以正式法律审查意见为准。"

## 输出格式

（输出格式即为上述四种受众定制格式之一，按需选择）

```
═══════════════════════════════════════
[合同名称] — 业务摘要
═══════════════════════════════════════
受众：[采购/业务/财务/管理层/法务]
生成日期：[___]
风险等级：[GREEN/YELLOW/RED]
═══════════════════════════════════════

[按上述对应格式输出]

---
*本摘要基于[合同审查时间]的合同审查报告生成，如有重大条款变更须重新审查。*
```

## 本技能不涵盖

- **原始合同审查** — 须先通过其他合同审查技能完成
- **代理合同纠纷诉讼**
- **特定行业的监管合规审查**
- **替代正式法律意见**
