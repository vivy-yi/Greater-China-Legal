## 0. 法规适用判断

```
产品/业务涉及用户群体 → 确定适用法规
  │
  ├─ 中国大陆用户为主 → PIPL 优先
  ├─ 同时有欧盟用户 → 加 GDPR 要求
  ├─ 同时有新加坡用户 → 加 PDPA 要求
  ├─ 同时有加州用户 → 加 CCPA 要求
  ├─ 同时有香港用户 → 加 PDPO 要求
  └─ 关键信息基础设施 → 加 NISL + CSL
```

**PIPL 是底线**——只要有中国大陆用户,PIPL 必须遵守。其他法域按用户来源叠加。

## 1. PIPL 核心规则速查

### 1.1 6 类合法性基础(PIPL 13 条)

| # | 基础 | 适用 |
|---|------|------|
| 1 | 同意 | 一般个人信息(须明确、具体、可撤回) |
| 2 | 合同必要 | 须与合同履行直接相关 |
| 3 | 法律义务 | 须为法定义务 |
| 4 | 紧急情况 | 须为保护生命财产 |
| 5 | 新闻报道 | 须为公益目的 |
| 6 | 学术研究 | 须去标识化 |
| 7 | 政府公开 | 须政府公开范围 |
| 8 | 其他法律法规规定 | 兜底条款 |

### 1.2 7 类敏感个人信息(PIPL 28 条)—— 须单独同意

| # | 类型 | 例子 | 额外要求 |
|---|------|------|---------|
| 1 | 生物识别 | 人脸/指纹/虹膜 | 单独告知+单独同意 |
| 2 | 宗教信仰 | 宗教信仰信息 | 同上 |
| 3 | 特定身份 | 公务员/党员信息 | 同上 |
| 4 | 医疗健康 | 病史/体检数据 | 同上 |
| 5 | 金融账户 | 银行账户/证券账户 | 同上 |
| 6 | 行踪轨迹 | 位置/行动记录 | 同上 |
| 7 | 未满 14 周岁 | 未成年人信息 | 须取得父母同意 |

### 1.3 5 模块 checklist(PIPL 评估)

```
1. 合法性基础
   - 6 类基础中选 1 类
2. 告知义务
   - 内容完整(目的/方式/范围/存储地)
   - 方式有效(不易忽视/易访问)
3. 同意管理
   - 撤回同意与给予同意同等便捷
   - 拒绝不得设置不合理条件
4. 数据安全
   - 分类分级保护
   - 必要安全措施
   - 泄露通知(72 小时内向 CAC 报告)
5. 主体权利
   - 查阅/复制(15 日内)
   - 更正/补充(15 日内)
   - 删除(法定情形)
   - 转移(特定情形)
   - 撤回同意(同等便捷)
```

### 1.4 主体权利响应时限

| 权利 | 时限 |
|------|------|
| 查阅/复制 | 15 日 |
| 更正/补充 | 15 日 |
| 删除(法定) | 立即 |
| 转移(特定) | 协商 |
| 撤回同意 | 立即 |
| 投诉 | 60 日内答复 |

## 2. 数据出境三路径(PIPL 38-40 条)

### 2.1 出境条件判断

| 条件 | 须走哪条路径 |
|------|----------|
| 个人信息 > 100 万人 | **必须安全评估** |
| 敏感个人信息(任意量) | **必须安全评估** |
| 累计出境 > 10 万人一般信息 | **必须安全评估** |
| 关键信息基础设施运营者(CIIO) | **必须安全评估** + 本地化 |
| 不满足以上 | 可选标准合同或认证 |

### 2.2 三种出境路径

| 路径 | 适用条件 | 主管机关 | 时效 |
|------|---------|---------|------|
| **安全评估** | 必评估情形 | 网信办(国家网信办) | 45 个工作日(可延 15 日) |
| **标准合同** | 自愿选择 | 无须审批,备案(10 个工作日) | 备案后生效 |
| **认证** | 自愿选择 | 认证机构 | 视机构 |

### 2.3 关键时间点(2023-2026 关键事件)

| 时间 | 事件 |
|------|------|
| 2023-06 | 《个人信息出境标准合同办法》正式生效 |
| 2023 | 《个人信息保护影响评估办法》生效 |
| 2024 | CAC 加强 APP 治理,大量下架违规 APP |
| 2025 | PIPL 实施 3 年,司法解释陆续出台 |
| 2026 | 数据要素流通试点扩大 |

## 3. 数据安全(DNPA)核心

### 3.1 数据分类分级

| 级别 | 描述 | 处理要求 |
|------|------|---------|
| 核心数据 | 国家安全/国计民生 | 最严格 |
| 重要数据 | 行业重要 | 严格(关基设施) |
| 一般数据 | 其他 | 标准 |

### 3.2 重要数据处理者义务

- 数据安全负责人
- 定期风险评估
- 重要数据处理者需向网信部门报送风险评估报告
- 数据出境:原则上不出境(确需出境——安全评估)

---

## 4. 关键信息基础设施(NISL + CSL)

### 4.1 识别标准

- 公共通信和信息服务
- 能源/交通/水利/金融/公共服务/电子政务
- 国防科技工业
- 其他一旦遭破坏/丧失功能/数据泄露可能严重危害国家安全/国计民生/公共利益

### 4.2 CIIO 义务

- 网络安全审查(采购网络产品和服务)
- 数据本地化(境内运营中收集和产生的个人信息和重要数据)
- 因业务需要确需出境的——安全评估
- 每年至少 1 次网络安全检测和风险评估

## 5. CAC 执法案例参照

### 5.1 常见违规场景

| 违规场景 | 处罚力度 | 典型案例 |
|---------|---------|---------|
| APP 超范围收集 | 警告/罚款/下架 | 某出行 APP 被处罚 |
| 未备案先上市 | 停业/罚款 | 某电商被处罚 |
| 数据泄露未报告 | 罚款 | 某社交平台被处罚 |
| 无同意处理敏感信息 | 高额罚款 | 某金融 APP 被处罚 |
| 未成年人保护不到位 | 罚款 | 某游戏 APP 被处罚 |
| 强制索权/不给用 | 通报 + 下架 | 某工具类 APP |

### 5.2 处罚力度

| 情形 | 罚款范围 |
|------|---------|
| 一般违规 | 100 万以下 |
| 严重违规 | 100 万-1000 万 |
| 特别严重(100 万+ 用户) | 5000 万以下或上一年度营业额 5% |
| 直接责任人 | 1-10 万 |

## 6. 关键期限速查

| 程序阶段 | 期限 | 法律依据 |
|---------|------|---------|
| 数据泄露通知 CAC | 72 小时 | PIPL 57 |
| 用户权利响应 | 15 日 | PIPL 50 |
| 同意撤回 | 立即 | PIPL 15 |
| PIPL 投诉 | 60 日内答复 | PIPL 65 |
| 安全评估 | 45 个工作日(+15) | 出境标准合同办法 |
| 标准合同备案 | 10 个工作日 | 出境标准合同办法 |
| APP 备案 | 上线前 | APP 治理通知 |
| 网络安全审查 | 45 个工作日(可延) | NISL |
| 主体权利 15 日未响应 | 投诉到网信办 | PIPL 65 |

## 7. 5 大程序对照(选择决策)

| 程序 | 触发 | 主管 |
|------|------|------|
| **PIPL** | 处理个人信息(中国大陆用户) | CAC + 各地网信办 |
| **DNPA** | 任何数据处理(中国境内) | CAC + 行业主管 |
| **NISL** | 关键信息基础设施运营 | 网信办 + 公安 |
| **APP 治理** | APP 上架 | 工信部 + 网信办 |
| **GDPR(域外)** | 欧盟用户 | 各国 DPA |
| **PDPA(域外)** | 新加坡用户 | PDPC |
| **CCPA(域外)** | 加州用户 | 加州司法部 |
