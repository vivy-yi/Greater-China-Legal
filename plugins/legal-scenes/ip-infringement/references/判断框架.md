# 知识产权侵权判断框架 — 中国大陆

> 本框架用于 ip-infringement 27 个 skill 的场景识别与判断逻辑，作为 IP 维权场景的统一依据。
> ip-infringement 是**知识产权侵权识别与维权**场景，覆盖商标/专利/著作权/商业秘密 4 大类型。
> 各 skill 内部已有判断逻辑（如侵权认定、风险评估），本文件描述跨 skill 的整体决策路由。

## 三、商标侵权判断（商标法第 57 条）

```
[节点 1] 是否有注册商标专用权？
├─ 是 → 进入侵权判断
└─ 否（未注册商标/驰名商标特殊保护）→ 商标法第 13/15/32 条另行判断

[节点 2] 商标使用方式？
├─ 同一种商品 + 相同商标 → 🔴 推定侵权（商标法第 57 条第 1 项）
├─ 同一种商品 + 近似商标 → 可能侵权（商标法第 57 条第 2 项）
├─ 类似商品 + 相同/近似商标 → 可能侵权（商标法第 57 条第 2 项）
└─ 不同商品 + 相同商标 → 一般不侵权（驰名商标除外）

[节点 3] 混淆可能性判断？
├─ 商标近似程度（音形义 + 整体印象）
├─ 商品类似程度（功能/用途/原料/消费对象）
├─ 商标显著性强弱
├─ 实际混淆证据
└─ 主观意图（恶意）

[节点 4] 例外情形？
├─ 善意先用 → 不侵权（商标法第 59 条）
├─ 描述性使用 → 不侵权
├─ 在先权利 → 限制注册商标权
└─ 权利穷竭 → 商品售出后不侵权

升级触发：
- 驰名商标认定 → 律师专项
- 涉外商标 → 律师专项
- 复杂商标近似判断 → 商标代理人介入
```

## 四、专利侵权判断

### 4.1 专利侵权判定步骤

```
Step 1：确定专利权保护范围
  - 权利要求书 + 说明书
  - 解释原则（说明书 + 权利要求 + 审查档案）
  - 禁止反悔原则

Step 2：确定被控侵权产品/方法
  - 技术特征比对
  - 全面覆盖原则（all-elements rule）

Step 3：等同原则判断
  - 方式/功能/效果三相同
  - 手段基本相同 + 功能基本相同 + 效果基本相同 + 领域普通技术人员易想到
  - 注意公知常识抗辩

Step 4：现有技术/现有设计抗辩
  - 专利申请日前公开
  - 抵触申请

Step 5：禁止反悔 + 捐献原则
  - 申请阶段放弃的内容不得再主张
  - 说明书记载但权利要求未要求的内容视为捐献
```

### 4.2 FTO（自由实施）判断

```
[节点 1] 现有专利权识别
├─ 中国专利 → CNIPA 检索
├─ 涉外专利 → 各国专利局检索
└─ PCT 申请 → 进入国家阶段后纳入

[节点 2] 技术特征比对
├─ 全面覆盖 → 🔴 侵权风险
├─ 等同覆盖 → 🟡 高风险
└─ 不覆盖 → 🟢 无侵权风险

[节点 3] 规避设计可能性
├─ 容易规避 → 🟢 低风险
├─ 需要重大改动 → 🟡 中风险
└─ 难以规避 → 🔴 高风险

升级触发：
- 涉外 FTO → 各国律师协作
- 标准必要专利 → 律师专项
```

### 4.3 专利无效宣告

```
[节点 1] 无效宣告理由（专利法第 22/26 条）
├─ 不符合新颖性/创造性/实用性 → 主要理由
├─ 公开不充分 → 重要理由
├─ 权利要求不清楚 → 一般理由
├─ 修改超范围 → 一般理由
└─ 违反法律/公序良俗 → 兜底理由

[节点 2] 证据准备
├─ 现有技术证据（专利申请日前公开）
├─ 公开方式（出版物/使用/网络）
├─ 时间证明
└─ 关联性证明

[节点 3] 程序
├─ 专利复审委员会受理
├─ 口头审理
├─ 无效决定
└─ 不服可起诉（北京知识产权法院）
```

## 五、著作权侵权判断

```
[节点 1] 是否有著作权？
├─ 是（独创性 + 可复制性 + 智力成果）→ 进入判断
└─ 否（公有领域/非作品）→ 不侵权

[节点 2] 著作权归属
├─ 原始归属（创作者）
├─ 继受归属（转让/继承）
├─ 法人作品（著作权法第 11 条）
├─ 职务作品
└─ 委托作品

[节点 3] 是否构成侵权
├─ 复制权（最常见）
├─ 发行权
├─ 展览权
├─ 表演权
├─ 放映权/广播权/信息网络传播权
├─ 摄制权/改编权/翻译权/汇编权
└─ 其他权利

[节点 4] 例外情形
├─ 合理使用（著作权法第 24 条）
│   ├─ 个人学习/研究/欣赏
│   ├─ 适当引用
│   ├─ 新闻报道
│   └─ 其他法定情形
└─ 法定许可

升级触发：
- 软件源代码比对 → 技术专家 + 律师
- 涉外著作权 → 律师专项
- 数字版权问题 → 互联网律师
```

## 六、商业秘密侵权判断

```
[节点 1] 是否构成商业秘密（反不正当竞争法第 9 条）
├─ 三性齐备：
│   ├─ 非公知性（不为公众所知悉）
│   ├─ 价值性（具有商业价值）
│   └─ 保密性（已采取保密措施）
└─ 不齐备 → 不构成商业秘密

[节点 2] 侵权方式
├─ 盗窃/贿赂/欺诈/胁迫/电子侵入
├─ 违反保密义务/违反权利人要求
├─ 教唆/帮助他人违反保密义务
└─ 第三人善意/恶意取得

[节点 3] 损害结果
├─ 实际损失（销售额减少等）
├─ 获利（侵权人获利）
├─ 许可费倍数
└─ 法定赔偿（500 万上限）

升级触发：
- 商业秘密构成 → 律师专项（举证困难）
- 员工跳槽争议 → 律师专项
- 大额赔偿 → 律师专项
```

## 七、维权路径选择（rights-protection-path）

```
[节点 1] 维权目标？
├─ 停止侵权（禁令）
├─ 赔偿损失
├─ 消除影响
├─ 道歉/恢复名誉
└─ 综合诉求

[节点 2] 维权路径
├─ 行政路径
│   ├─ 市场监管部门（商标/专利行政执法）
│   ├─ 版权局（著作权行政执法）
│   ├─ 公安（商业秘密/严重侵权）
│   └─ 海关（进出口侵权）
│
├─ 民事路径
│   ├─ 协商调解
│   ├─ 法院诉讼
│   │   ├─ 知识产权法院（北京/上海/广州）
│   │   ├─ 互联网法院（北京/杭州/广州）
│   │   └─ 中级人民法院（其他）
│   └─ 仲裁（如合同有约定）
│
├─ 刑事路径
│   ├─ 假冒注册商标罪（刑法第 213 条）
│   ├─ 销售假冒注册商标的商品罪（刑法第 214 条）
│   ├─ 侵犯著作权罪（刑法第 217 条）
│   ├─ 侵犯专利权罪（刑法第 216 条）
│   └─ 侵犯商业秘密罪（刑法第 219 条）
│
└─ 平台投诉
    ├─ 电商平台（淘宝/天猫/京东等）
    ├─ 内容平台（抖音/快手/B站等）
    ├─ 搜索引擎（百度/Google）
    └─ 社交平台（微信/微博）

[节点 3] 路径选择考量
├─ 时间效率：行政 > 平台投诉 > 民事
├─ 成本：平台投诉 < 行政 < 民事 < 刑事
├─ 证据要求：行政 < 平台投诉 < 民事 < 刑事
├─ 赔偿金额：刑事附带民事 > 民事 > 行政
└─ 威慑力：刑事 > 民事 > 行政
```

## 八、损害赔偿计算（damage-calculator）

### 8.1 计算方法（按优先顺序）

```
Step 1：实际损失（权利人因侵权所受损失）
  - 销量减少 × 单价 × 利润率
  - 或：销量减少 × 单位利润

Step 2：侵权获利（侵权人因侵权所获利益）
  - 侵权产品销量 × 单价 × 利润率
  - 注意扣除合理成本

Step 3：合理许可费倍数
  - 参照类似情形许可费
  - 1-3 倍（情节严重 5 倍）

Step 4：法定赔偿
  - 商标/专利/著作权：5 万 - 500 万
  - 商业秘密：500 万以下
  - 恶意侵权：1-5 倍惩罚性赔偿
```

### 8.2 赔偿数额影响因素

| 因素 | 加重 | 减轻 |
|---|---|---|
| 主观意图 | 恶意 | 善意 |
| 侵权规模 | 大规模 | 小规模 |
| 持续时间 | 长期 | 短期 |
| 侵权后果 | 严重 | 轻微 |
| 权利人损失 | 重大 | 轻微 |
| 补救措施 | 无补救 | 主动补救 |
| 重复侵权 | 是 | 否 |

## 九、证据保全指引（evidence-guide）

### 9.1 证据类型

```
[节点 1] 权利证据
├─ 商标注册证
├─ 专利证书
├─ 著作权登记证书
├─ 商业秘密载体（合同/制度/标识）
└─ 在先权利证据

[节点 2] 侵权证据
├─ 侵权产品/作品
├─ 销售凭证（发票/合同/收据）
├─ 网络证据（截图/录屏）
├─ 公证证据（公证处）
└─ 鉴定结论

[节点 3] 损害证据
├─ 权利人损失（财务报表）
├─ 侵权人获利（公开数据）
├─ 许可费证据
└─ 合理开支（律师费/公证费）
```

### 9.2 公证要点

```
- 时间：尽快（防止证据灭失）
- 地点：公证处或律师见证
- 方式：公证员现场操作 + 拍照/录像
- 范围：链接 + 购买过程 + 收货过程
- 费用：每件 500-3000 元
- 注意：网络证据公证需用公证处网络
```

## 十一、开源软件合规（oss-review）

### 11.1 开源许可证类型

```
[节点 1] 许可证分类
├─ 宽松型（Permissive）：MIT / Apache / BSD
│   ├─ 允许商用
│   ├─ 允许修改
│   ├─ 必须保留版权声明
│   └─ 限制小
│
├─ 弱传染型（Weak Copyleft）：LGPL / MPL
│   ├─ 修改部分必须开源
│   └─ 整体应用可闭源
│
└─ 强传染型（Strong Copyleft）：GPL / AGPL
    ├─ 修改部分必须开源
    ├─ 衍生作品必须开源
    └─ 网络分发也受 AGPL 约束

[节点 2] 合规评估
├─ 许可证识别（依赖扫描）
├─ 许可证兼容性
├─ 传染性分析
├─ 静态/动态链接情况
└─ 商业化策略

升级触发：
- AGPL/GPL 商用 → 律师专项
- 专利交叉许可 → 律师专项
- 出口管制（含美国 EAR）→ 律师专项
```

| 时效要求 | 中（侵权诉讼 3 年） | 中 | 高（仲裁 60 日） |
| 风险等级 | 高（影响业务 + 刑事风险） | 中-高 | 高 |
| 涉外因素 | 频繁（涉外 IP 占比高） | 偶尔 | 较少 |

**核心差异**：ip-infringement 是**侵权场景**（不是合同/劳动），有独特的取证、维权路径、赔偿计算方法。
