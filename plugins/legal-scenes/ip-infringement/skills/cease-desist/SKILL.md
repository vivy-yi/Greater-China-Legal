# /cease-desist — China Mainland

## 两种模式

- `--send` — 起草警告函（主张权利）
- `--receive` — 分析收到的侵权警告函（防御方）

---

## CN警告函特殊说明

**中国警告函的法律效力：**
- 警告函不等于法院裁定
- 收函方可提起确认不侵权之诉
- 恶意警告函可能构成不正当竞争（《反不正当竞争法》第11条）
- 警告函不得捏造虚假事实或夸大权利范围

**CN执法路径选择：**
| 路径 | 适用场景 |
|---|---|
| 律师函协商 | 首次侵权、金额较小 |
| 行政执法 | 商标侵权（市场监督管理局）|
| 民事诉讼 | 专利/著作权侵权、赔偿诉求 |
| 平台投诉 | 电商/APP侵权（通知-删除）|
| 刑事报案 | 严重侵权（违法所得≥3万元）|

## Send模式：起草警告函

### Step 1: 确认权利基础

询问：
- 权利类型：□ 商标  □ 专利  □ 著作权  □ 商业秘密
- 注册号/登记号：
- 权利人：[公司名称]
- 侵权行为：[描述]

### Step 2: 确认侵权事实

- 侵权产品/服务：[描述]
- 侵权证据：[截图/公证/购买]
- 发现渠道：[电商平台/展会/市场监测]

### Step 3: 起草警告函要点

**CN警告函须包含：**
1. 权利人信息 + 权利证明（注册号/登记号）
2. 侵权事实的具体描述
3. 法律依据（《商标法》第57条/《专利法》第11条/《著作权法》第52条/《反不正当竞争法》第9条）
4. 停止侵权要求（明确行为+期限）
5. 后续法律行动的明确警告

### Step 4: 审批前置条件

发出前须确认：
- 是否达到审批阈值（参考CLAUDE.md中的升级矩阵）
- 是否需要法务总监/外部律师审核
- 是否需要固定证据（公证）

## Receive模式：分析警告函

### 收到的警告函分析

**分析维度：**
1. **权利有效性** — 权利是否有效注册/登记？是否在有效期内？
2. **权利归属** — 警告函发出方是否为权利人？
3. **侵权指控是否成立** — 是否落入权利保护范围？
4. **管辖权** — CN主管机关是否有管辖权？
5. **抗辩理由** — 现有技术/无效宣告/合法来源/合理使用

**CN常见抗辩：**
- 商标：先用权、描述性使用、权利用尽
- 专利：权利用尽、先用权、无效宣告程序中
- 著作权：合理使用、法定许可
- 商业秘密：反向工程、独立开发

## 输出格式

### Send模式输出
```
## 警告函草稿 — [对方当事人]

[完整警告函内容]

---
审批状态：[待审批/已批准]
发送前确认：□ 证据已公证 □ 阈值已确认 □ 律师已审核
```

### Receive模式输出
```
## 警告函分析 — [发函方] vs [我方]

### 权利基础评估
[权利有效性分析]

### 侵权指控评估
[是否落入保护范围]

### 抗辩选项
[可用抗辩列表]

### 建议行动
🔴 高风险：[建议]
🟠 中风险：[建议]
🟡 低风险：[建议]
```
