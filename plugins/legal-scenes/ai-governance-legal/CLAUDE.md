### § A8.1 AI 治理特别注意 4 大块（算法备案 + 标识义务 + 内容审核 + 责任划分）

> **核心原则**：AI 治理的核心是**算法透明 + 内容安全 + 责任划分**。"技术中立" 不能作为法律免责依据。

#### 块 1：算法备案（《互联网信息服务算法推荐管理规定》2022）

**核心法规：**
- 《互联网信息服务算法推荐管理规定》（2022 年 3 月生效）
- 《生成式人工智能服务管理暂行办法》（2023 年 8 月生效）
- 《互联网信息服务深度合成管理规定》（2023 年 1 月生效）

**算法备案触发条件（任一）：**

| 类型 | 触发 | 备案机关 |
|------|------|---------|
| **算法推荐** | 推送 / 个性化排序 / 精选 / 过滤 | 网信办 |
| **生成式 AI** | 大模型 / 对话机器人 / 内容生成 | 网信办 + 主管部门 |
| **深度合成** | 换脸 / 语音克隆 / 视频生成 | 网信办 |
| **决策类算法** | 信贷 / 招聘 / 保险定价 | 网信办 + 行业主管部门 |

**备案材料：**

| 材料 | 必备性 |
|------|--------|
| 算法备案表 | 必备 |
| 算法安全评估报告 | 必备 |
| 算法机理说明 | 必备 |
| 用户投诉处理机制 | 必备 |
| 数据来源合法性 | 必备 |

**blocks（绝对禁止）**：
- 应备案未备案 → **责令改正 + 警告 + 1-10 万罚款**
- 推荐算法诱导用户沉迷 / 消费 → **责令改正 + 1-10 万罚款**
- 算法歧视（如大数据杀熟）→ **价格法 + 个保法 + 1-50 万罚款 + 责令停业**

**work but ships（可补救）**：
- 备案材料不齐 → 30 日内补正
- 算法变更未重新备案 → 60 日内补办

#### 块 2：深度合成标识义务（《深度合成管理规定》第 17 条）

**核心义务：**

| 场景 | 标识义务 |
|------|---------|
| **AI 生成 / 合成内容** | **显著标识**（不得隐藏 / 模糊） |
| **换脸 / 语音克隆** | 显著标识 + 用户单独同意 |
| **深度伪造新闻** | 显著标识 + 来源标注 |
| **AI 客服 / 助手** | 用户首次交互时告知 AI 身份 |
| **AI 生成广告** | 显著标识 + 广告法合规 |

**标识方式：**

| 方式 | 适用 |
|------|------|
| **显著文字标识** | "AI 生成" / "由 AI 创作" / "AI 合成" |
| **图标标识** | 通用 AI 标识 |
| **声音 / 视频提示** | AI 语音 / 视频前提示 |
| **元数据标识** | 内容元数据嵌入 |

**blocks（绝对禁止）**：
- 深度合成未标识 → **5-10 万罚款 + 责令改正**
- 虚假 AI 内容未标识 → **同上 + 网信办通报**
- 利用 AI 仿冒他人 → **民事侵权 + 治安处罚 + 刑责（伪造证件罪 / 诈骗罪）**

#### 块 3：生成式 AI 服务合规（《暂行办法》2023）

**核心义务：**

| 义务 | 具体标准 |
|------|---------|
| **预训练数据合法性** | 来源合法 + 知识产权合规 + 个人信息合规 |
| **内容安全** | 不得生成违法 + 不良信息 |
| **用户实名** | 提供服务前实名认证 |
| **未成年人保护** | 识别 + 拒绝 + 投诉渠道 |
| **投诉举报** | 便捷 + 及时处理 + 反馈 |
| **数据标注** | 标注规则 + 质量审核 |

**服务提供者责任：**

| 违法情形 | 处罚 |
|---------|------|
| 生成违法信息 | 责令改正 + 警告 + 停业整顿 |
| 未尽网络安全义务 | 《网安法》处罚 |
| 侵犯知识产权 | 民事赔偿 + 《暂行办法》第 12 条 |
| 违反个保法 | 个保法处罚（5000 万 / 5% 营业额） |

**blocks（绝对禁止）**：
- 生成危害国家安全 / 颠覆政权 / 恐怖主义内容 → **立即停业 + 刑责（寻衅滋事罪 / 煽动颠覆国家政权罪）**
- 生成淫秽色情 / 暴力 / 虚假信息 → **停业整顿 + 网信办通报**
- 未成年人不当内容 → **停业 + 5-10 万罚款**

#### 块 4：AI 法律责任划分（民事 / 行政 / 刑事）

**民事责任（《暂行办法》第 17 条）：**

| 主体 | 责任 |
|------|------|
| **服务提供者** | 内容侵权 + 知识产权 + 个人信息 |
| **使用者** | 内容侵权 + 知识产权 |
| **平台** | 知道或应当知道侵权的连带责任 |
| **用户** | 网络安全 + 个保法 + 知识产权 |

**算法歧视典型案例：**
- 大数据杀熟（老客户价格更高）→ 价格法 + 个保法
- 算法性别歧视（招聘 / 信贷）→ 妇女权益保障法 + 个保法
- 算法地域歧视（运费差异）→ 反不正当竞争法
- 算法年龄歧视（老年人价格更高）→ 老年人权益保障法

**blocks（绝对禁止）**：
- AI 内容侵犯著作权 → **民事赔偿 + 著作权法处罚**
- AI 训练数据非法 → **个保法 / 数据安全法处罚**
- AI 决策歧视 → **反不正当竞争 + 个保法**

**work but ships（可补救）**：
- AI 输出错误 → 立即修正 + 致歉
- 算法偏差 → 重新评估 + 调整
- 训练数据瑕疵 → 立即下线 + 清理

#### 块 5：4 大绝对禁止（含具体法条 + 后果分级）

| 禁止 | 法条 | 后果 |
|------|------|------|
| 1. **算法应备案未备案** | 《算法推荐管理规定》第 24 条 | 1-10 万 + 责令改正 |
| 2. **深度合成未标识** | 《深度合成管理规定》第 17/22 条 | 5-10 万 + 网信办通报 |
| 3. **生成违法内容** | 《暂行办法》第 4/17 条 | 停业整顿 + 刑责 |
| 4. **算法歧视** | 《个保法》第 24 条 + 《价格法》 | 1-50 万 + 责令停业 |

**关键差异（与 v3 标准一致）**：
- **绝对禁止（blocks）** → agent 看到这些**直接停止**——告诉用户"绝对不能做"
- **work but ships** → agent 提示整改 + 给出时限
- **FYI** → 记录不主动告知

- 输出责任划分（服务提供者 / 用户 / 平台）？（法律风险）
