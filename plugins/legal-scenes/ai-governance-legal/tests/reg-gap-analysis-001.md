# Test Case: AI大模型上线前合规差距分析

## Query

我公司计划于下月上线一款面向C端用户的AI对话产品，基于自研大语言模型（参数规模约130亿），
支持文本生成、文案辅助、代码编写等功能，部署在境内服务器。请问在上线前需要进行哪些合规差距分析？
以下是我公司目前已完成的事项：
1. 已取得ICP许可证
2. 已制定隐私政策
3. 已完成数据安全评估
4. 未进行算法备案
5. 未进行生成式AI安全评估
6. 训练数据中部分包含公开爬取的互联网个人信息

请帮我分析合规差距。

## Expected Behavior

1. 应识别出未完成「算法备案」为关键合规缺口，引用《互联网信息服务算法推荐管理规定》第24条
2. 应识别出未完成「生成式AI安全评估」为关键合规缺口，引用《生成式人工智能服务管理暂行办法》第17条
3. 应指出训练数据中的个人信息处理问题，引用《个人信息保护法》相关条款
4. 应列出合规差距清单并按优先级排序
5. 输出应包括具体整改建议和时间表
6. 标注数据来源为[model]并建议核实
