## 互联网贷款合规（141号文+2023年新规）

### 核心合规要求（141号文）

| 要求 | 说明 |
|---|---|
| 利率上限 | 年化36%以下（实际APR须<36%） |
| 收费透明度 | 不得收取砍头息/服务费不得前置扣除 |
| 联合贷款 | 联合贷款中出资方须持牌 |
| 贷款用途 | 不得流入股市/房市等限制领域 |
| 信息披露 | 须明示年化利率 |

### 2023年新规要点

```
互联网贷款新规（2023年征求意见稿/正式稿）：
- 联合贷款出资比例：商业银行须≥30%
- 单一借款人余额上限：≤注册资产10%且≤30万元
- 贷款催收规范：不得暴力催收/不得骚扰联系人
- 数据安全：不得滥用个人信息
```

## 消费者保护

### 监管要求（七项禁止）

| 禁止 | 说明 |
|---|---|
| 禁止虚假陈述 | 不得欺骗误导消费者 |
| 禁止捆绑销售 | 不得强制或诱导捆绑销售 |

### 追问规则（关键）

legal-element-extraction 的输出包含 `## 待补充事实` 节。如果该节非空：

1. **暂停当前分析流程**
2. 向用户逐一提问待补充事实，例如：
   > "请问合同中关于[知识产权归属/数据存储位置/价格调整机制]的条款是什么？这会影响后续判断。"
3. 用户补充后，**回到 Step 0 重新执行 legal-element-extraction**，将新信息并入结构化事实
4. 当待补充事实清空后，继续后续分析

**不得在待补充事实未清空的情况下输出最终结论。** 缺失关键事实的结论标注为「推定结论，须在事实补全后复核」。
