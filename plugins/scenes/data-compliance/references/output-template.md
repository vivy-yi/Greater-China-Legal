## 1. Reviewer note 5 行(Pattern 9)

> agent 必写——5 行内说清"为什么有风险/建议下一步/注意什么"

```
⚠️ Reviewer note

1. 核心风险:[本案最致命的一点 + 法条 + 风险等级 🔴/🟡/🟢]
2. 数据范围:[处理哪些个人信息 + 敏感程度 + 用户数]
3. 法律义务:[哪些 PIPL/GDPR 条文必查 + 是否触发 PIA 条件]
4. 升级建议:[是否必升 + 升给谁 + 触发条件(用户数/敏感/涉外)]
5. 下一步:[1-3 个具体动作 + 截止日 + 谁负责]
```

**5 行的深度要求**:
- 每行不超过 80 字
- 必须含法条引用(精确到条)
- 必须含具体动作(动词开头)
- 风险等级用三色(🟢/🟡/🔴)标注

## 2. Risk calibration 3 段表(Pattern 18)

> 来自 `upstream-kwp-design-analysis.md` 4g.7
> **关键洞见**——本项目当前没有"风险分级响应"机制——这是缺失的核心结构

### 2.1 data-compliance 专属 3 段表(初版)

| 段 | 模式 | 为什么 | 解决路径 |
|---|------|------|--------|
| **Usually blocks** | 未告知处理目的直接收集 | PIPL 13 条 | 立即停止 + 罚款 100 万以下 |
| | 未取得同意处理敏感 | PIPL 29 条 | 立即停止 + 直接责任人 1-10 万 |
| | 100 万+ 出境未走安全评估 | PIPL 38-40 条 | 立即停止 + 罚款 1000 万以下 |
| | 数据泄露 72 小时未报告 | PIPL 57 条 | 立即停止 + 罚款 100 万以下 |
| | 关基设施违反本地化要求 | NISL + CSL | 立即停止 + 暂停业务 |
| | 自动化决策导致算法歧视 | PIPL 24 条 | 立即停止 + 外部律师审查 |
| | 强制索权(不给权限不能用) | 个人信息保护法 5-6 条 | 立即停止 + APP 下架风险 |
| **Usually requires work but ships** | 隐私政策 1 年未更新 | PIPL 17 条 | 30 日内更新 |
| | 同意机制 6 类中选错 | PIPL 13-14 条 | 补充同意 |
| | PIA 报告不完整 | PIPL 55-56 条 | 30 日内补完 |
| | 用户权利响应超 15 日 | PIPL 50 条 | 立即响应 + 投诉风险 |
| | 同意记录不完整 | PIPL 14 条 | 补充记录 |
| | 数据分类分级不准确 | DNPA 21 条 | 重新分类 |
| | 出境后未做影响评估 | PIPL 55 条 | 60 日内补评估 |
| **Usually FYI** | 新司法解释生效但暂不适用 | 不在业务范围 | 关注扩展 |
| | 同行被处罚但本公司未涉 | 行业警示 | 修订内控 |
| | 行业自律规则更新 | 不具强制力 | 跟踪 |
| | 学者评论 | 不具约束力 | 参考 |
| | CAC 执法新案例 | 不直接适用 | 内部培训 |
| | 国标 GB/T 修订 | 建议而非强制 | 调整参考 |

### 2.2 使用规则

| 段 | agent 动作 |
|---|------|
| **blocks** | 立即停止 + 告知用户 + 给出解决路径——**不绕过** |
| **requires work but ships** | 提示 DPO + 给时间表 + 不阻塞——**不立即停止** |
| **FYI** | 记录到 matter.md 的"风险提示"段——**不主动告知** |

### 2.3 与 5 类必升的关系

| 必升触发器 | 落点 |
|---------|------|
| 100 万人+ 处理 | blocks 段(未评估可能违法) |
| 敏感个人信息 | blocks 段(未取得单独同意违法) |
| 关基设施 | blocks 段(本地化要求违法) |
| 涉外 | blocks 段(域外法适用) |
| 自动化决策 | blocks 段(算法审查要求) |

### 2.4 持续学习

- 每季度根据新司法解释/法规更新本表
- 律师反馈"agent 误判"→ 调整本表分类
- 跨 scene 通用 pattern 抽到 `learn-patterns.md` § 0

## 3. 完整 PIA 报告模板

```markdown
## PIA 报告：[功能/产品名] — [日期]

**密级:**[律师执业秘密 / 法务内部参考 / 参考资料]
**产品/功能:**[名称]
**评估日期:**[YYYY-MM-DD]
**风险等级:**[HIGH/MEDIUM/LOW]
**适用法规:**[PIPL/DNPA/NISL/GDPR/PDPA]
**DPO:**[姓名 + 联系方式]
**外部律师:**[姓名]
**更新日期:**[YYYY-MM-DD]

---

### ⚠️ Reviewer note(5 行)
1. 核心风险:[...]
2. 数据范围:[...]
3. 法律义务:[...]
4. 升级建议:[...]
5. 下一步:[...]

---

### 一、4 大绝对禁止检查(§ 5.3)
- [ ] 未告知收集 → 🔴 停止
- [ ] 未同意处理敏感 → 🔴 停止
- [ ] 100 万+ 出境未评估 → 🔴 停止
- [ ] 数据泄露 72 小时未报告 → 🔴 停止

### 二、5 类必升检查(§ 5.2)
- [ ] 100 万人+ 处理 → 🔴 升级 DPO + 外部律师
- [ ] 敏感个人信息 → 🔴 升级 DPO + 外部律师
- [ ] 关基设施 → 🔴 升级 DPO + 网信办
- [ ] 涉外(GDPR/PDPA) → 🔴 升级跨境律师 + 当地 DPO
- [ ] 自动化决策 → 🔴 外部律师 + 算法审查

### 三、Risk calibration 3 段检查(本文件 § 2)
- [ ] blocks 段命中 → 🔴 立即处理
- [ ] work but ships 段命中 → 🟡 提示 DPO
- [ ] FYI 段命中 → 🟢 记录

### 四、事实摘要

#### 4.1 业务/产品
- 名称/类型/功能
- 处理目的/方式/范围
- 用户规模(注册/日活/峰值)
- 涉及数据类型

#### 4.2 法律基础
- 6 类合法性基础中选哪类
- 同意机制(单独/一般/默示)
- 告知方式

#### 4.3 风险评估
- 对个人权益的影响
- 安全措施
- 跨境传输
- 自动化决策

### 五、风险等级判定

| 等级 | 条件 | 处理 |
|------|------|------|
| 🔴 HIGH | 涉及敏感信息处理/大规模用户数据/跨境传输 | 强制 DPO 审核+外部律师 |
| ⚠️ MEDIUM | 一般个人信息处理/新功能上线前 | 建议审核 |
| ✅ LOW | 内部管理数据/匿名化数据 | 快速评估 |

### 六、整改建议

#### 6.1 必做项
- 立即执行
- 截止日
- 负责人

#### 6.2 建议项
- 1 个月内完成
- 负责人

#### 6.3 跟踪项
- 3 个月内复评

### 七、Decision tree(Pattern 10)
> What next? Pick one:
> 1. [立即整改] — 列出必做项
> 2. [升级到 DPO/外部律师] — 草拟升级请求
> 3. [补事实] — 2-3 个开放问题
> 4. [归档到 DPO 文档库] — 进入日常跟踪
> 5. [别的] — [...]

### 八、附录
- A. 数据流图
- B. 法规清单
- C. CAC 执法案例
- D. 风险点对照表
```
