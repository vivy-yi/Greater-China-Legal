## 1. 5 大法域分层(Pattern 16)

### 1.1 5 大法域速查

| 法域 | 标识 | 主要法规 | 监管严度 | 默认权重 |
|------|------|---------|---------|---------|
| 中国大陆 | `cn-mainland` | PIPL + DNPA + NISL | 高 | 必填 |
| 香港 | `hk` | PDPO | 中 | 选填 |
| 澳门 | `mo` | 个资法 | 中 | 选填 |
| 台湾 | `tw` | 個資法 | 中 | 选填 |
| 新加坡 | `sg` | PDPA | 中 | 选填 |

### 1.2 域外法对照

| 法规 | 适用法域 | 关键特点 |
|------|---------|---------|
| **GDPR** | 欧盟 + 欧盟用户 | 域外效力(欧盟用户即适用) |
| **CCPA/CPRA** | 加州 | 销售/共享/删除权 |
| **PDPA(SG)** | 新加坡 | 相对宽松 |
| **PDPO(HK)** | 香港 | 6 类敏感 + 自愿 |
| **個資法(TW)** | 台湾 | 8 类敏感 + 强制告知 |
| **LGPD** | 巴西 | 类似 GDPR |
| **PIPL** | 中国大陆(对标 GDPR) | 严于 GDPR |

### 1.3 关键差异(中国 vs 欧盟)

| 维度 | 中国 PIPL | 欧盟 GDPR |
|------|----------|---------|
| 域外效力 | 域外(处理中国公民) | 域外(欧盟居民) |
| 同意 | 明确/默示 | 明确/可撤回 |
| 数据本地化 | 关基设施必须 | 否(但跨境需充分性) |
| 处罚 | 100 万-1000 万 | 全球营收 4% |
| 监管 | CAC | DPA(各国) |
| 跨境路径 | 安全评估/标准合同/认证 | 充分性/SCC/BCR |
| DPO | 鼓励 | 强制(某些情形) |

### 1.4 关键差异(中国 vs 美国)

| 维度 | 中国 PIPL | 美国 CCPA/CPRA |
|------|----------|--------------|
| 适用门槛 | 中国用户 | 加州居民 + 25 万用户 |
| 同意 | 明确(默认 opt-in) | opt-out |
| 处罚 | 100 万-1000 万 | $750/次 |
| 监管 | CAC | 加州司法部 |
| 数据本地化 | 关基必 | 否 |

## 2. 法域特定升级规则(Pattern 17)

### 2.1 升级规则总表

| 触发器 | cn-mainland | hk / sg(普通法系) | mo / tw(大陆法系) | 涉外(欧盟/美国等) |
|--------|------------|------------------|------------------|-----------------|
| **<1 万人处理** | DPO | DPO | DPO | 当地 DPO + 总部 |
| **1-100 万人** | GC | GC + 当地律师 | GC | 当地律师 + 跨境律师 |
| **>100 万人 / 敏感** | CEO + 董事会 | GC + 跨境律师 | GC | 当地律师主导 + 跨境律师 |
| **关基设施** | 网信办 + GC | GC + 跨境律师 | GC | 当地律师 + 跨境律师 |
| **数据泄露(严重)** | CEO + 董事会 + 公安 | 当地律师 + 跨境 | GC + 跨境 | 当地律师主导 |
| **GDPR 适用** | 跨境律师 + 当地律师 | 当地律师 | 当地律师 | 当地律师主导 |
| **CCPA 适用** | 跨境律师 + 加州律师 | 加州律师 | 加州律师 | 加州律师主导 |
| **跨境数据流(任意)** | 网信办备案 | 当地 DPA | 当地监管 | 当地监管 + 跨境律师 |
| **自动化决策/算法** | 外部律师 + 算法审查 | 当地律师 | 当地律师 | 当地律师 + 算法专家 |

### 2.2 cn-mainland 升级 4 档

| 档位 | 阈值 | 升给 | 通道 |
|------|------|------|------|
| junior | <1 万人 | dpo | 邮件 |
| dpo | 1-100 万人 | gc | 邮件+会议 |
| gc | >100 万人 / 敏感 / 涉外 | ceo | 邮件+董事会 |
| ceo | 上市公司 / 重大泄露 | board | 会议+文件 |

### 2.3 涉外合规清单(涉外案件必查)

| 维度 | 中国法 | 域外法 |
|------|------|------|
| **个人信息保护** | PIPL | GDPR / CCPA / PDPA |
| **数据安全** | DNPA | NIS Directive / NIST |
| **网络安全** | NISL | EU NIS2 / Executive Order 14028 |
| **数据本地化** | 关基必 | 一般不要求 |
| **跨境传输** | 安全评估/标准合同/认证 | 充分性/SCC/BCR |
| **同意机制** | 明确 | opt-in / opt-out |
| **处罚** | 100 万-1000 万 | 全球营收 4%(GDPR) |

### 2.4 涉外案件 4 步处理

1. **判断法域**——哪些用户/数据涉及域外?适用哪些法?
2. **选择律师**——当地 DPO + 跨境律师协同
3. **合规审查**——PIPL + 域外法 双线审查
4. **执行**——同时满足双方要求

## 3. GDPR 关键要求(中国公司涉外必读)

### 3.1 域外效力(GDPR 第 3 条)

- 欧盟境内设立机构 → 适用 GDPR
- 向欧盟数据主体提供商品/服务 → 适用 GDPR(无论机构在哪)
- 监控欧盟数据主体行为 → 适用 GDPR

### 3.2 关键权利

| 权利 | 描述 | 时限 |
|------|------|------|
| 知情权 | 处理目的/方式/法律基础 | 提供时 |
| 访问权 | 获取数据副本 | 30 日 |
| 更正权 | 更正不准确数据 | 30 日 |
| 删除权("被遗忘权") | 删除特定数据 | 30 日 |
| 限制处理权 | 限制处理但保留数据 | 30 日 |
| 数据可携权 | 结构化、通用、机器可读 | 30 日 |
| 拒绝权 | 反对处理/直接营销 | 任何时候 |

### 3.3 关键义务

| 义务 | 描述 |
|------|------|
| 透明度 | 隐私通知清晰易懂 |
| 数据保护影响评估(DPIA) | 高风险处理前必做 |
| 数据保护官(DPO) | 大规模/高风险必设 |
| 数据泄露通知 | 72 小时内通知 DPA |
| 数据最小化 | 仅收集必要数据 |
| 准确性 | 数据必须准确 |
| 存储限制 | 不得超过必要期限 |

### 3.4 跨境传输(GDPR 第 44-50 条)

- 充分性决定(adequacy decision)
- 标准合同条款(SCC)
- 约束性企业规则(BCR)
- 认证 + 行为准则
-  derogations(合同必要/重大公共利益等)

## 4. CCPA 关键要求(加州用户)

### 4.1 适用范围

- 年营收 > $25 million
- 持有 50,000+ 消费者/家庭/个人设备
- 50%+ 营收来自销售消费者个人信息

### 4.2 关键权利

| 权利 | 描述 |
|------|------|
| 知情权 | 收集/出售/披露信息 |
| 删除权 | 删除消费者个人信息 |
| 选择退出销售(opt-out) | "Do Not Sell My Personal Information" |
| 限制敏感信息使用 | Limit the Use of My Sensitive Personal Information |
| 不歧视 | 行使权利不被歧视 |

### 4.3 与 PIPL 差异

- PIPL 默认 opt-in(明确同意)
- CCPA 默认 opt-out(可选择退出)
- CCPA 处罚较轻($750/次)
- PIPL 处罚较重(100 万-1000 万)

## 5. 跨境数据流 3 条路径对比

| 维度 | 中国 PIPL 38-40 | 欧盟 GDPR 44-50 | 加州 CCPA |
|------|----------------|----------------|----------|
| 路径 | 安全评估/标准合同/认证 | 充分性/SCC/BCR | 不限制(只通知) |
| 主管 | 网信办 | 各国 DPA | 无 |
| 时效 | 45-60 工作日 | SCC 30 日 | 无 |
| 评估 | 强制(100 万+) | 强制(高风险) | 无 |
