# /sanctioned-entity-checker — 制裁实体筛查

## 核心概念

制裁实体筛查是对外贸易和跨境交易中最重要的合规防线之一。制裁名单通常分为三类：（1）全面制裁名单——禁止与名单上实体进行任何交易（如OFAC的SDN List）；（2）行业制裁名单——禁止在特定行业（如俄罗斯能源/军工/金融）进行交易；（3）出口管制名单——禁止向其出口特定管制物项（如BIS的实体清单Entity List、未经核实清单Unverified List）。违反制裁法规的法律后果极其严重：OFAC的民事罚款可达违法交易金额的两倍（或按次最高$356,579/次），故意规避则可触发刑事追诉和监禁。

中国法律体系中的制裁法规同样重要。《出口管制法》第18条授权国务院商务部建立"管控名单"——对于违反出口管制规定的进口商和最终用户，可列入管控名单，采取禁止交易、限制交易等措施。《反外国制裁法》第12条禁止中国境内的组织和个人执行或协助执行外国歧视性限制措施。因此，跨境贸易企业在筛查制裁名单时，必须同时考量中国法、美国法（长臂管辖）、联合国制裁决议和交易对手所在国的制裁法规。

## 工作流程

### 第一步：收集交易对手身份信息

- 收集交易对手全名（中英文）、注册地址、注册编号（如统一社会信用代码/LEI Code）、法定代表人姓名、实际控制人/最终受益人（UBO）
- 收集中间方/代理商/货运代理/运输公司等全链条参与主体的身份信息
- 收集最终用户（end-user）和最终用途（end-use）信息——出口管制项下比直接交易对手更重要
- 对交易对手的股权结构做穿透——识别拥有25%以上股权的自然人或实体（OFAC的50%规则：被制裁实体合计持股50%以上则视为被制裁）

### 第二步：对比制裁名单库

- 比对中国管控名单：（1）《出口管制法》管控名单（商务部公告）；（2）《不可靠实体清单》；（3）《反外国制裁法》反制清单
- 比对OFAC制裁名单：（1）SDN List（特别指定国民清单）；（2）CAPTA List（对应外国制裁规避者）；（3）NS-CMIC List（涉军企业清单）
- 比对BIS出口管制名单：（1）Entity List（实体清单）；（2）Unverified List（未经核实清单）；（3）Denied Persons List（拒绝人员清单）
- 比对欧盟制裁名单：EU Consolidated Sanctions List
- 比对联合国安理会制裁决议：UN Consolidated Sanctions List

### 第三步：模糊匹配与风险评分

- 对被制裁名单的匹配结果进行模糊比对——不仅仅是精确匹配，更是拼写变体(如"Mohammed"与"Mohamad")、音译差异、地址匹配
- 对匹配结果按风险等级评分：精确匹配（人名+地址+国籍均匹配）= 高风险；名称近似但其他信息不符 = 中风险；名称模糊匹配但信息完全不符 = 低风险
- 对中风险结果，通过商业情报数据库（如Dow Jones/Dun & Bradstreet）进一步核实身份信息
- 记录匹配详细信息：比对的名单名称、匹配字段、相似度百分比、风险评级

### 第四步：地理制裁和行业制裁核查

- 核查交易是否涉及受制裁的国家/地区：OFAC制裁的国家（伊朗/朝鲜/叙利亚/古巴/克里米亚）和部分制裁的国家（俄罗斯/委内瑞拉/缅甸/白俄罗斯）
- 核查交易是否涉及受制裁的行业：俄罗斯的军工/能源/金融/科技行业（OFAC Directive）、中国的半导体/人工智能领域（BIS新规）
- 核查交易货物是否受特定许可证要求的限制（即使交易对手不在制裁名单上）
- 如涉及中国对第三国的"长臂管辖"类反制措施，评估是否存在违反《反外国制裁法》第12条的风险

### 第五步：出具筛查结论与措施建议

- 出具《制裁筛查报告》：被筛查实体信息、比对结果、风险等级、合规结论
- 低风险（无匹配）：出具"可交易"结论，记录备查（保存5-10年）
- 中风险（模糊匹配）：出具"需进一步核实"结论，建议暂停交易并深入调查
- 高风险（精确匹配）：出具"禁止交易"结论，立即暂停所有交易，评估是否需要向OFAC/商务部提交自愿披露（voluntary disclosure）
- 如涉及中国制裁名单匹配，立即通知公司法务并遵循中国法律要求停止交易

## 升级决策门

出现以下情形须移交专业制裁合规律师处理：
- 被筛查实体在被制裁名单上精确匹配，且交易已经部分执行
- 被筛查实体为被制裁国家的政府/军方/情报机构关联主体
- 交易货物涉及军民两用技术或敏感物项，且交易对手位于受制裁国家
- 筛查涉及"中国法律与外国制裁法律冲突"的合规困境（如中国《反外国制裁法》第12条）
- 交易涉及金融机构（银行/支付机构）的内部制裁合规政策与法律要求不一致
- 需要向OFAC申请特定许可（Specific License）才能进行交易
