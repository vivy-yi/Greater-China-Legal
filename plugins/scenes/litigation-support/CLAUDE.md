## 诉讼程序概览

### 一审程序

```
起诉 → 立案（7日内）→ 送达（5日内）→ 答辩（15日内）
→ 举证（可申请延长）→ 庭前会议（可不开庭）→ 开庭审理
→ 判决（6个月内，简易程序3个月）

注：公告送达60日后视为送达
```

### 二审程序

```
上诉（判决书送达15日内）→ 上诉状提交 → 二审立案
→ 送达 → 审理（3个月内）→ 判决/发回重审
```

### 再审程序

```
申请（判决生效6个月内）→ 法院审查 → 裁定再审/驳回
→ 再审审理（按一审/二审程序）
```

## 诉讼时效

### 主要时效规定

| 案由 | 时效 | 起算 |
|---|---|---|
| 合同纠纷 | 3年 | 知道或应当知道权利受损 |
| 侵权纠纷 | 3年 | 知道或应当知道权利受损 |
| 人身损害 | 3年 | 知道或应当知道 |
| 产品质量 | 2年 | 知道或应当知道 |
| 劳动争议 | 1年（仲裁）/3年（诉讼） | 劳动关系终止 |
| 借款纠纷 | 3年 | 约定还款日期 |
| 物权请求权 | 一般无时效 | — |

### 时效届满的法律效果

```
时效届满：债务人获得时效抗辩权
→ 债务人可以拒绝履行
→ 但债务人放弃抗辩的，仍须履行
→ 诉讼中债务人提出时效抗辩，法院应予支持（除特殊情形）
```

## 证据规则

### 举证责任分配

| 情形 | 举证方 |
|---|---|
| 合同纠纷 | 主张合同成立/生效方 |
| 侵权纠纷 | 原告须证明侵权/损害/因果关系 |
| 劳动争议 | 用人单位对解除合法性负举证 |
| 知识产权侵权 | 权利人证明侵权+损害 |

### 证据种类

```
书证：合同/函件/发票/转账记录
物证：实物/现场照片
视听资料：录音/录像/监控
电子数据：微信记录/邮件/系统日志
证人证言：须出庭作证
鉴定意见：须有资质机构出具
勘验笔录：须法院或公证机构制作
当事人陈述：须结合其他证据
```

### 电子证据效力判断（2019年《电子数据规定》）

```
有效须满足：
1. 电子数据真实性须通过鉴定/公证等方式确认
2. 微信/QQ记录：须证明账号归属和完整未删改
3. 邮件：须证明发送方服务器和完整性
4. 建议：对关键证据提前做公证保全
```

## 诉讼策略

### 选择诉讼还是仲裁

| 对比 | 法院诉讼 | 仲裁 |
|---|---|---|
| 公开性 | 公开审理 | 不公开 |
| 速度 | 较慢 | 较快 |
| 费用 | 诉讼费 | 仲裁费较高 |
| 专家性 | 法官不一定懂技术 | 可选专家仲裁员 |
| 一裁终局 | 否，可上诉 | 是，无上诉 |
| 跨境执行 | 须双边司法协助 | 纽约公约92国可执行 |

### 和解/调解优先

```
建议优先和解/调解的情形：
1. 证据不充分
2. 诉讼时效临近
3. 对方无财产可执行
4. 维持商业关系
5. 成本效益不划算
```

## 推理原子能力调用流程

| 7 | `case-retrieval` | 需要检索类案时调用 |

### 追问规则（关键）

legal-element-extraction 的输出包含 `## 待补充事实` 节。如果该节非空：

1. **暂停当前分析流程**
2. 向用户逐一提问待补充事实，例如：
   > "请问合同中关于[知识产权归属/数据存储位置/价格调整机制]的条款是什么？这会影响后续判断。"
3. 用户补充后，**回到 Step 0 重新执行 legal-element-extraction**，将新信息并入结构化事实
4. 当待补充事实清空后，继续后续分析

**不得在待补充事实未清空的情况下输出最终结论。** 缺失关键事实的结论标注为「推定结论，须在事实补全后复核」。
