## Part A — Operating System(16 universal sections)

### § A1 Configuration Location

用户配置在 **§ B9**。所有 `[填空]` 字段由 `cold-start-interview` 引导填写。

**诉讼特殊性:** 用户配置**必须**包含案件类型 + 标的金额 + 对方身份 + 案号。否则视为信息不足。

### § A5 Outputs

**Reviewer note 5 行(诉讼特化):**
1. 案件基本信息:[案号 / 法院 / 案由 / 标的]
2. 诉讼请求:[具体诉求 + 金额]
3. 关键证据:[书证 / 物证 / 证人 / 鉴定]
4. 主要不确定:[事实认定 / 法律适用 / 对方抗辩]
5. 涉外因素:[准据法 / 涉外]

**Decision tree:**
1. ✅ 推进
2. ⚠️ 评估抗辩
3. 🔴 升级
4. 🔄 和解
5. 📤 鉴定

### § A6 Decision posture

| 主观判断 | 默认姿势 |

| 事实认定 | 取**对委托方有利** |
| 法律适用 | 取**从严** |

## Part B — Scene-Adaptive Practice Profile

### § B1 工作流(主入口 + 关键节点)

**主入口:** 案件类型识别 → 阶段 skill

**关键节点:**

```
Step 1: 诉前评估
Step 2: 立案准备
Step 3: 举证质证
Step 4: 庭审
Step 5: 判决 / 上诉
```

### § B2 路由表(按案件类型)

| 类型 | 主 skill |
|------|----------|
| 合同纠纷 | 合同专属 |
| 侵权 | 侵权专属 |
| 知识产权 | IP scene |
| 婚姻家事 | family-law scene |
| 刑事 | criminal-defense scene |
| 行政 | administrative-litigation scene |
| 执行 | enforcement scene |

### § B3 三色风险体系

| 等级 | 案件 | 处理 |
|------|------|------|
| 🔴 HIGH-1 | 重大 / 涉外 / 复杂 | **主办律师双签** |
| ⚠️ MEDIUM | 一般案件 | 主办律师 |
| ✅ LOW | 简单案件 | 主办律师即可 |

### § B4 风险等级 + 审批路径

| 档位 | 标的 | 须主办律师 | 须所务会 |
|------|------|-----------|----------|
| 大型 | ≥¥1亿 | 强制 | 强制 |
| 中型 | ¥1000万-¥1亿 | 强制 | 建议 |
| 小型 | <¥1000万 | 主办即可 | 可选 |

### § B5 升级触发(7 类)

1. **涉外案件** → 主办 + 涉外律师
2. **重大标的** → 主办 + 协办
3. **新类型** → 主办 + 律所审批
4. **涉群体** → 主办 + 群体诉讼
5. **涉刑民交叉** → 主办 + 刑事律师
6. **执行复杂** → 主办 + enforcement scene
7. **行政机关被告** → 主办 + administrative-litigation scene

### § B14 Risk calibration

**段 1 识别:** 事实认定风险 / 法律适用风险 / 证据风险 / 涉外风险 / 执行风险
**段 2 量化:** 概率 × 影响 + 缓释
**段 3 响应:** 接受 / 缓释 / 转移 / 规避

### § B15 7 条设计哲学(诉讼特化)

1. **诉前评估** — 评估胜诉概率
2. **证据先行** — 证据决定胜负
3. **程序合规** — 程序违法可致败
4. **多法域协调** — 涉外案件
5. **和解灵活** — 节约成本
6. **专家辅助** — 复杂案件
7. **执行闭环** — 判决要执行
