## 对赌协议判断规则

### 效力判断（九民纪要2019第5条）

| 情形 | 效力 | 说明 |
|---|---|---|
| 目标公司与投资方对赌 | ✅ 有效 | 约定业绩目标，通常有效 |
| 目标公司股东与投资方对赌 | ✅ 通常有效 | 实践中一般认定有效 |
| 固定收益回报（名投资实借贷） | ❌ 可能无效 | 可能被认定为借贷关系 |
| 违反法律强制性规定 | ❌ 无效 | 按具体条款判断 |

### 调整机制建议

```
建议：设置估值调整上限（如"补偿不超过投资额的X倍"）
避免：固定回报回购条款（可能被认定无效）
```

## 外商投资准入判断

### 负面清单查询

```
[GOV] 国家发改委：http://www.ndrc.gov.cn/
[GOV] 商务部：https://www.mofcom.gov.cn/
关键词："外商投资准入特别管理措施（负面清单）"
```

### 行业分类

| 分类 | 说明 | VIE是否可行 |
|---|---|---|
| 禁止类 | 绝对禁止外资 | VIE为实践做法（但法律上存在风险） |
| 限制类 | 外资比例受限或须合资 | 可考虑合资或VIE |
| 鼓励类 | 一般无限制 | 直接外资即可 |
| 未列入负面清单 | 允许外资 | 直接外资即可 |

## ODI备案流程

### 须办理ODI备案的情形

| 情形 | 是否须备案 |
|---|---|
| 境内企业向境外设立公司 | ✅ 须备案 |
| 境内企业向境外汇出投资款 | ✅ 须外汇登记 |
| 37号文登记 | 须（个人境外投资）或可不（企业境外投资） |

### 主管机关

| 机关 | 职责 |
|---|---|
| 商务部/发改委 | ODI备案/核准 |
| 外管局 | 外汇登记 |
| 36号文登记（个人） | 个人境外投资备案 |

## 股权激励规则

### 中国法律下的股权激励

| 类型 | 法律依据 | 说明 |
|---|---|---|
| 员工期权 | 无直接法律依据 | 通常通过持股平台或代持实现 |
| 限制性股票 | 无直接法律依据 | 实践中有各种变通做法 |
| 虚拟股 | 无直接法律依据 | 内部激励，非真正股权 |

### 税务考虑

```
行权时：按"工资薪金所得"计征个人所得税
转让时：按"财产转让所得"计征个人所得税
建议：咨询专业税务律师
```

## 推理原子能力调用流程

本场景的工作流程中，按以下顺序调用 `legal-atomic` 原子能力：

| 顺序 | 原子 Skill | 调用时机 |

| 1 | `legal-norm-validity-check` | 在任何法条引用前调用，验证法条是否现行有效 |
| 2 | `deductive-reasoning` | 在分析阶段，将待判断的问题转化为 P-F-C 三段论格式 |
| 3 | `conflict-resolution` | 发现多个法条或请求权可能竞合时调用 |
| 4 | `evidence-argument-chain` | 需要组织证据与主张对应关系时调用 |
| 5 | `argument-strength-evaluation` | 输出结论前，标注论证强度（强/中/弱/存疑） |
| 6 | `legal-risk-assessment` | 在风险分级判断时调用 |
| 7 | `case-retrieval` | 需要检索类案时调用 |

### 追问规则（关键）

legal-element-extraction 的输出包含 `## 待补充事实` 节。如果该节非空：

1. **暂停当前分析流程**
2. 向用户逐一提问待补充事实，例如：
   > "请问合同中关于[知识产权归属/数据存储位置/价格调整机制]的条款是什么？这会影响后续判断。"
3. 用户补充后，**回到 Step 0 重新执行 legal-element-extraction**，将新信息并入结构化事实
4. 当待补充事实清空后，继续后续分析

**不得在待补充事实未清空的情况下输出最终结论。** 缺失关键事实的结论标注为「推定结论，须在事实补全后复核」。
