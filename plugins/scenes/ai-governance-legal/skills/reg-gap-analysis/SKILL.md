# /reg-gap-analysis — China Mainland

## CN AI合规差距分析框架 v2.0

本框架用于系统性地分析AI系统与中国大陆监管要求之间的差距，适用于算法备案前自查、安全评估前准备、以及定期合规审计。

## 第二部分：具体执行步骤

### Step 1：系统画像构建

**1.1 收集基础信息**

| 信息项 | 获取方式 | 必填 |
|--------|----------|------|
| 系统名称 | 用户输入 | 是 |
| 系统类型 | 技术文档审阅 | 是 |
| 用户规模 | 运营数据 | 是 |
| 服务范围 | 产品文档 | 是 |
| 数据来源 | 数据流梳理 | 是 |
| 模型架构 | 技术文档 | 部分 |
| 境外运营情况 | 法务确认 | 是 |

**1.2 系统分类判定**

根据判断树输出系统所属分类：

- A类：算法推荐服务（需备案+透明度说明）
- B类：生成式AI服务（需安全评估+备案+标注合规）
- C类：深度合成服务（需安全评估+备案+溯源机制）
- D类：行业AI应用（行业专项+基础合规）
- E类：基础AI服务（备案+基础合规）

### Step 3：现状调研与差距识别

**3.1 调研检查清单**

针对每一项监管要求，逐一确认：

- [ ] **现状**：目前是否已满足该要求？
- [ ] **证据**：提供相关证明材料
- [ ] **差距**：如不满足，差距在哪里？
- [ ] **风险等级**：重大/中等/轻微
- [ ] **补救方案**：具体改进措施
- [ ] **预计完成时间**：可设定目标日期

**3.2 常见差距模式**

| 模式 | 描述 | 优先级 |
|------|------|--------|
| 完全缺失 | 对应措施/机制根本不存在 | 🔴 重大 |
| 部分覆盖 | 有措施但未覆盖全部要求 | 🟠 中等 |
| 形式合规 | 有文件但未实际执行 | 🟠 中等 |
| 证据不足 | 已执行但缺乏书面记录 | 🟡 轻微 |
| 时效过期 | 备案/评估报告已过期 | 🔴 重大 |

### Step 4：差距分析

**4.1 风险矩阵**

| 影响维度 | 说明 |
|----------|------|
| 备案影响 | 是否会导致备案被拒或延误 |
| 处罚风险 | 可能面临的行政处罚 |
| 业务影响 | 对服务连续性的影响 |
| 声誉影响 | 对企业形象的影响 |

**4.2 优先级排序**

根据以下规则确定补救优先级：

1. **P0-紧急**：影响备案/安全评估通过，必须立即处理
2. **P1-高**：面临处罚风险，需30日内处理
3. **P2-中**：存在隐患，需60日内处理
4. **P3-低**：优化项，90日内处理
