## 1. Reviewer note 5 行(Pattern 9)

> agent 必写——5 行内说清"为什么有风险/建议下一步/注意什么"

```
⚠️ Reviewer note

1. 核心风险:[本案最致命的一点 + 法条 + 风险等级 🔴/🟡/🟢]
2. 证据缺口:[缺失的关键证据 + 补救路径 + 截止日]
3. 程序风险:[民主程序/公示/工会通知等程序瑕疵 + 法条]
4. 升级建议:[是否必升 + 升给谁 + 触发条件(人数/经济性裁员/上市公司)]
5. 下一步:[1-3 个具体动作 + 截止日 + 谁负责]
```

**5 行的深度要求**:
- 每行不超过 80 字
- 必须含法条引用(精确到条)
- 必须含具体动作(动词开头)
- 风险等级用三色(🟢/🟡/🔴)标注

## 2. Risk calibration 3 段表(Pattern 18)

> 来自 `upstream-kwp-design-analysis.md` 4g.7
> **关键洞见**——本项目当前没有"风险分级响应"机制——这是缺失的核心结构
> **3 段**:
> 1. **Usually blocks** —— 真正阻挡的事
> 2. **Usually requires work but ships** —— 妥协空间
> 3. **Usually FYI** —— 通知但不动作

| | 学者评论 | 不具约束力 | 参考 |

| | 当地最低工资标准调整 | 影响所有员工 | 调整薪酬 |

### 2.2 使用规则

| 段 | agent 动作 |
|---|------|
| **blocks** | 立即停止 + 告知用户 + 给出解决路径——**不绕过** |
| **requires work but ships** | 提示律师 + 给时间表 + 不阻塞——**不立即停止** |
| **FYI** | 记录到 matter.md 的"风险提示"段——**不主动告知** |

### 2.3 与 11 大必升的关系

| 必升触发器 | 落点 |
|---------|------|
| 3 期/医疗期/工伤/老员工/工会 | **blocks 段**(都是绝对禁止) |
| 经济性裁员 20+ 人 | blocks 段(程序违法) |
| 规章制度解除 | blocks 段(未经民主程序) |
| 试用期违法 | blocks 段 |
| 扣押证件 | blocks 段 |

### 2.4 持续学习

- 每季度根据新司法解释/法规更新本表
- 律师反馈"agent 误判"→ 调整本表分类
- 跨 scene 通用 pattern(数据跨境/未披露事项)抽到 `learn-patterns.md` § 0
