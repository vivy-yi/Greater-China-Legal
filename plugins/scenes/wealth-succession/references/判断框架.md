## 二、复杂度评估框架

### 2.1 案件复杂度评级
| 评级 | 标准 | 处置方式 |
|-----|------|---------|
| 🟢 基础 | 单一日标、单一资产类型、无跨境因素、无争议 | 本skill直接处理 |
| 🟡 进阶 | 多项目标、多类型资产、无跨境但可能有争议 | 本skill处理，升级门严格审查 |
| 🔴 专业 | 跨境、多继承人争议、特殊资产类型、涉及诉讼 | 升级至专业律师 |

### 2.2 升级决策门判断要素
```
触发升级的五个核心维度：
┌────────────────────────────────────────────────────────────┐
│ 1. 跨境因素（境外财产/身份/法律适用）                      │
│    → 升级至跨境税务/法律专业律师                           │
│                                                            │
│ 2. 争议强度（已产生或可能产生诉讼）                        │
│    → 升级至诉讼代理律师                                    │
│                                                            │
│ 3. 资产复杂度（须专业估值/特殊资产类型）                   │
│    → 升级至资产处置专业律师+评估机构                        │
│                                                            │
│ 4. 主体特殊性（无行为能力人/外籍/上市公司相关）            │
│    → 升级至对应领域专业律师                                │
│                                                            │
│ 5. 金额门槛（遗产/资产规模超过1亿元）                      │
│    → 升级至高净值法律服务团队                              │
└────────────────────────────────────────────────────────────┘
```

## 三、交叉领域判断规则

### 3.1 Skill交叉重叠区域
| 交叉情形 | 判断规则 | 处置 |
|---------|---------|-----|
| 遗嘱+信托 | 遗嘱中提及信托，以遗嘱效力审查为入口 | testament-drafting-checker优先 |
| 遗嘱+税务 | 遗产继承税务，以程序为入口 | probate-procedure-guide优先 |
| 婚前协议+家族治理 | 涉及家族企业股权的婚前协议 | family-governance-advisor优先 |
| 信托+税务 | 信托设立税务规划 | trust-structure-advisor为入口，升级时联动estate-tax-planner |
| 家族治理+遗嘱 | 家族企业传承安排 | family-governance-advisor为入口 |

### 3.2 案件性质判断流程
```
输入：用户描述的需求
      │
      ▼
┌──────────────────────────────────┐
│ 是否涉及财产传承/安排？          │
├──────────────────────────────────┤
│ 是                               │
│   ↓                              │
│ 财富传承需求分类 → 确定主Skill  │
│   ↓                              │
│ 评估复杂度 → 确定是否触发升级门  │
│   ↓                              │
│ 输出处理方案                     │
└──────────────────────────────────┘
```

## 四、时效性判断框架

### 4.1 法律变更影响评估
| 变更类型 | 对本领域影响 | 响应级别 |
|---------|------------|---------|
| 《民法典》司法解释修订 | 可能影响继承、婚姻相关规定 | 30日内更新相关skill |
| 税务法规调整 | 影响税务规划方案 | 15日内更新estate-tax-planner |
| 信托监管政策 | 影响信托架构设计 | 30日内更新trust-structure-advisor |
| 外管/外汇政策 | 影响跨境资产安排 | 15日内发出风险提示 |

### 4.2 更新触发机制
```
监控到法规变动
      │
      ▼
┌────────────────────────────────┐
│ 评估影响范围：                  │
│ - 涉及哪些skill                │
│ - 影响哪些具体条款             │
│ - 是否影响核心判断逻辑         │
└────────────────────────────────┘
      │
      ▼
┌────────────────────────────────┐
│ 影响≥2个skill → 启动联合评审   │
│ 影响≥4个skill → 发布整体更新   │
│ 核心逻辑改变 → 发布重大公告   │
└────────────────────────────────┘
```

## 五、质量控制判断标准

### 5.1 答案质量自检清单
- [ ] 是否明确了适用的法律法规
- [ ] 是否区分了法律规定与实践惯例
- [ ] 是否提示了时效性和地域差异
- [ ] 是否在升级决策门处正确截断
- [ ] 是否引用了真实的法条编号
- [ ] 是否避免了预测性法律意见

### 5.2 输出分级标准
| 级别 | 输出内容 | 适用场景 |
|-----|---------|---------|
| L1-信息提供 | 法律框架、条文引用 | 基础了解需求 |
| L2-分析建议 | 框架分析、风险提示 | 进阶规划需求 |
| L3-方案输出 | 具体建议方案（含限制性声明） | 明确规划需求 |
| L4-升级 | 提示专业介入必要性，提供查询路径 | 触发升级门情形 |
