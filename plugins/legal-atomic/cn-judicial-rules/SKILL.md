
## 使用方式

### 1. 命令行（gcl CLI）

```bash
# 查询相关规则
gcl rules <关键词>

# 按案由查询
gcl rules-by-case <案由名>

# 按规则 ID 查询
gcl rule <rule_id>
```

### 2. 在 SKILL.md 中调用

```yaml
---
data_source: cn-judicial-rules
# 引用方式见具体场景 skill
---
```

### 3. AI 推理时使用

```
输入：用户问题
  → 调用 legal-element-extraction 提取关键事实
  → 在裁判规则库中关键词匹配
  → 找到相关规则后输出
    - 审查顺序（来自"审理方法论"）
    - 风险等级（来自"律师实操要点.风险提示"）
    - 法条依据（来自"法条依据"）
  → 标注 [data: cn-judicial-rules]
```

## 重要边界

**本数据集是离线 ground truth，不是实时权威**：
- ❌ 不替代付费法律数据库（北大法宝/元典）
- ❌ 不能保证所有规则与最新司法解释完全一致
- ❌ 实际案件中需结合具体事实由律师判断

**使用场景**：
- ✅ AI 推理时的离线 ground truth
- ✅ 审查顺序和判断标准的参考
- ✅ 风险等级（致命/重要/一般）的量化依据
- ✅ 客户紧急问题、不便连外网时的备选

## 更新机制
