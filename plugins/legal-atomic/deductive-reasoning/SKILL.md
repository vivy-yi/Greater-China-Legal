# 演绎推理（P-F-C三段论）

## 定位

将隐含的法律推理转化为显式的P-F-C结构：

```
大前提(P) = 法律规则（法条）
小前提(F) = 法律事实（法条对案件事实的定性）
结论(C)   = 法律效果（权利/责任/义务）
中项(M)   = 连接大前提和小前提的桥梁概念
```

```
大前提(P): 《民法典》第577条 — 违约方应承担继续履行/补救/赔偿损失等违约责任
小前提(F): 甲公司未按约定时间交付货物 → 构成违约
结论(C):   甲公司应向乙公司赔偿损失
中项(M):   "不履行合同义务"
```

## 三段论类型

### 直言三段论（构成要件型）

```
所有满足构成要件X的行为 → 都有法律后果Y
当前行为满足构成要件X
∴ 当前行为具有法律后果Y
```

### 假言三段论（条件关系型）

**肯定前件式（最常用）：**
```
如果X，则Y（X → Y）
X成立
∴ Y成立
```

**否定后件式（用于抗辩）：**
```
如果X，则Y（X → Y）
Y不成立
∴ X不成立
```

## 完整推理流程

### 第一步：输入分类

将案件材料分类到三个初始框：

```
【大前提 P】：法律规则（法条），如无明文规定需补充隐含前提
【小前提 F】：法律事实（法律对自然事实的定性）
【小前提 f】：自然事实（当事人陈述/事件描述）
【结论 C】：假设性结论（待验证）
```

### 第二步：推理（循环交替执行直到逻辑闭环）

#### A. 完善各部分逻辑结构

```
P层：大前提梳理
  - 辨别类型：直言三段论（构成要件）or 假言三段论（条件关系）
  - 如无明文规定，补充隐含大前提

F层：小前提梳理
  - 理清事件性质/先后顺序/因果关系
  - 厘清各主体间的法律关系
  - 标注缺失或模糊的法律关系

C层：结论排序
  - 对多个结论排序
  - 识别中间结论
```

#### B. 链接关系与识别中项

```
f → F（事实归入）
  将自然事实链接到法律定性
  例："没付尾款" → "根本违约"

F → P（寻找规范）
  将法律小前提链接到法条
  例："根本违约" → 《民法典》第577条

P + F → C（推导结论）
  由法条+法律事实推理到结论
  写成三段论格式，验证中项是否正确推导
```

### 第三步：有效性验证

| 谬误类型 | 错误形式 | 后果 |
|---|---|---|
| **肯定后件谬误** | X→Y, Y成立, ∴X成立 | 把结果当原因 |
| **否定前件谬误** | X→Y, ¬X成立, ∴¬Y成立 | 把无因当无果 |
| **四词谬误** | A→B, C→D, ∴A→D | 中项未连接/偷换概念 |

**有效性只取决于形式，不取决于内容。** 即使大前提/小前题为假，只要形式正确，推理仍判定为"有效"。

## 输出格式

### 1. P-F-C结构图

```
## 推理链条

### C1：[最终结论]
  P：[法条] — 所有X → Y
  F：[法律事实] — 当前事实满足X
  C：当前事实 → Y
  中项(M)：[连接P和F的概念]

### C1.1：[中间结论]
  P：[法条]
  F：[法律事实]
  C：[中间结论]
```

### 2. 三段论详表

```
| 自然事实(f) | 法律小前提(F) | 触发法条(P) | 中项(M) |
|---|---|---|---|
| f1:... | F1:... | P1:... | X |
| f2:... | F2:... | P2:... | Y |
```

### 3. 有效性报告

```
## 推理有效性

| 结论 | 形式有效性 | 中项状态 | 谬误类型 | 说明 |
|---|---|---|---|---|
| C1 | ✅有效/❌无效 | 存在/缺失 | 无/肯定后件/否定前件/四词 | ... |
| C2 | ... | ... | ... | ... |

### 谬误说明
- [如有谬误，详细说明]
```

## 各场景应用

### labor-arbitration（解除合法性判断）

```
判断公司解除劳动合同是否违法：

P层：《劳动合同法》第39条（过错解除）/ 第40条（非过错解除）
F层：员工是否存在第39条情形 / 是否提前30日通知
C层：解除是否合法

中项示例："严重违反规章制度"、"不能胜任工作"
```

### contract-review（违约责任分析）

```
判断违约责任是否成立：

P层：《民法典》第577条（违约责任）
F层：合同义务是否未履行/不符合约定
C层：违约责任是否成立

中项示例："不履行合同义务"、"履行不符合约定"
```

### litigation-support（诉讼策略）

```
设计诉讼请求的论证链：

P层：请求权基础（合同/侵权/不当得利）
F层：案件事实是否满足构成要件
C层：诉讼请求能否得到支持

中项示例："合同成立"、"损害后果"、"因果关系"
```
