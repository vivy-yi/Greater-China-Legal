# /legal-article-retrieval — 法律条文检索

## 五阶段检索流程

```
准备阶段 → 思维阶段 → 执行阶段 → 复核阶段 → 成果阶段
    ↓         ↓         ↓         ↓         ↓
明确目的   法源位阶   体系检索   交叉验证   报告呈现
工具配置   法律关系   关键词法   涵摄检验   结论形成
          请求权思维  案例倒查   检索循环   风险提示
          效力甄别
          冲突处理
```

---

## 检索前必须确认的信息

| 信息类别 | 必须确认内容 |
|---------|------------|
| 争议类型 | 劳动争议/合同纠纷/侵权责任/婚姻家庭/行政 |
| 主体信息 | 我方 vs 对方 |
| 核心事实 | 时间、地点、行为 |
| 争议焦点 | 我方诉求是什么 |
| 地域信息 | 涉及省份/城市 |
| 法律关系 | 是否签订合同 |

**严格禁止：**
- ❌ 用户未确认事实前调用检索工具
- ❌ 基于假设进行推断式法律分析
- ❌ 事实总结中出现法律结论

## 法源位阶体系

| 效力层级 | 渊源 | 说明 |
|---------|------|------|
| 高 | 宪法 | 最高法律效力 |
| ↑ | 法律（全国人大/常委会） | 基本法律/普通法律 |
| | 行政法规（国务院） | 总理签署国务院令 |
| | 地方性法规 | 省级人大及其常委会 |
| | 部门规章 | 国务院各部委 |
| 低 | 地方政府规章 | 省级政府 |

**非正式渊源（参考）：**
- 司法解释（最高法/最高检）
- 指导性案例
- 会议纪要（九民纪要等）
- 学术通说

## 请求权思维

```
步骤1：识别请求权基础
  用户想要什么？（赔偿/解除/确认/变更）

步骤2：定位请求权类型
  合同请求权 / 侵权请求权 / 不当得利请求权 / 无因管理请求权

步骤3：检索构成要件
  《民法典》第X条：
  ① 要件A：...
  ② 要件B：...
  ③ 要件C：...

步骤4：涵摄检验
  案件事实 → 是否满足全部要件？
```

---

## 关键词检索法

```
步骤1：确定核心术语
  法律概念的标准表达

步骤2：构建检索词组合
  AND：同时包含（概念+地域）
  OR：同义词扩展（甲方/发包方）

步骤3：交叉验证
  同一法条在不同数据库中交叉核对
```

---

## 复核阶段

```
QC1：法律是否现行有效？
  - 检查修订历史
  - 检查是否已被废止

QC2：适用范围是否匹配？
  - 主体：自然人/法人/特殊主体
  - 地域：全国/地方
  - 时间：生效/失效

QC3：冲突如何处理？
  - 上位法优于下位法
  - 新法优于旧法
  - 特别法优于一般法
```
