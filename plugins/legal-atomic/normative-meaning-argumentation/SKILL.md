# /normative-meaning-argumentation — 规范意义论证

## 核心问题

不仅要说明法条"什么意思"，还要论证：
1. 该法条为何能评价特定事实
2. 评价到何种程度
3. 事实是否落入规范所欲调整的范围

---

## 工作流程

### Step 1：确定待适用法条

```
明确：
- 具体条文/规则/原则
- 构成要件
- 法律后果
- 制度位置
```

### Step 2：识别规范目的

```
分析该规范：
- 保护何种法益
- 防止何种风险
- 维持何种法律秩序/交易秩序

区分：
- 主要目的
- 附随目的
- 制度背景
```

### Step 3：提炼规范评价对象

```
该规范真正评价的是什么？
（不是抽象的"权益保护"，而是具体的"何种行为/风险/利益状态"）
```

### Step 4：筛选规范相关事实

```
从复杂案件事实中选出：
✓ 与规范目的直接相关的事实 → 进入法律评价
✗ 背景事实/情绪性事实 → 排除
```

### Step 5：建立对应关系

```
规范目的 ←→ 事实特征

"案件事实为何触及该规范所关注的风险/利益/秩序"
例如：某行为虽形式不完全符合典型样态，但实质上制造了规范所欲防止的风险
```

### Step 6：判断涵摄边界

```
是否应被纳入规范适用范围？

纳入 → 说明理由
不纳入 → 说明为何超出（过度扩张/破坏体系协调）
```

### Step 7：形成结论

```
在本案中：
- 该规范应如何理解
- 如何评价事实
- 是否支持最终法律结论
```

## 输出格式

```
## 规范意义论证报告

### 规范确定
《X法》第X条：[构成要件] → [法律后果]

### 规范目的
- 主要目的：[法益/风险/秩序]
- 附随目的：[...]
- 制度背景：[...]

### 规范评价对象
[该规范真正评价的是什么]

### 案件事实的规范相关性
✓ 相关事实：[清单]
✗ 不相关事实：[清单]

### 规范目的—事实特征对应
[建立对应关系，说明为何触及]

### 涵摄边界判断
[纳入/不纳入] + 理由

### 结论
- 规范理解：[如何理解]
- 事实评价：[如何评价]
- 法律结论：[是否支持]

### 置信度：🟢高 / 🟡中 / 🔴低
```
