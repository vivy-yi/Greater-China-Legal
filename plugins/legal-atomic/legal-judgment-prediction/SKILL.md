# /legal-judgment-prediction — 法律判决预测

## 8步流程

```
输入：案件事实描述
  │
  ▼ S1: 要素提取（structured-element-extraction）
  │
  ▼ S2: 概念理解（legal-concept-comprehension）
  │
  ▼ S3: 争议焦点识别（dispute-issue-identification）
  │    └── QC-1: 要素完整性与定性一致性
  ▼ S4: 法条检索（legal-article-retrieval）
  │    └── QC-2: 法条适配性与案例相关性
  ▼ S5: 案例检索（case-retrieval）
  │    └── QC-3: 推理逻辑自洽性
  ▼ S6: 演绎推理（deductive-reasoning）
  ▼ S7: 格式适用（legal-document-formatting）
  ▼ S8: 术语规范（legal-terminology）
  │
  ▼
输出：结构化判决预测报告（含置信度）
```

## QC质量检查点

```
QC-1: 要素完整性
  - 主体/行为/客体/结果/因果/情节是否完整？
  - 如不完整 → 回溯S1/S2

QC-2: 法条适配性
  - 所引法条是否适用于本案事实？
  - 如不适配 → 回溯S3重新识别

QC-3: 推理自洽性
  - 演绎推理逻辑是否自洽？
  - 如不自洽 → 回溯S4补充法条
```
