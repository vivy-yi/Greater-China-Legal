# /case-retrieval — 案例检索方法论

## 检索前准备

### 确定检索需求

```
来源一：从案件材料中提炼
  - 案由/纠纷类型
  - 核心法律关系
  - 主要争议焦点
  - 适用法律条文

来源二：从检索目的反推
  - 支撑论点：找支持己方的裁判先例
  - 预判风险：了解同类案件胜诉率
  - 反驳对方：找否定对方论点的案例
  - 填补空白：法律未明确规定时的司法态度
```

### 检索范围

```
时间范围：优先近三年（指导性案例除外）
地域范围：上级法院及本院 > 省市高院/中院近三年
案例层级：指导性案例 > 公报案例 > 典型案例 > 一般案例
```

## 检索方法

### 四种检索路径

| 路径 | 适用场景 | 方法 |
|---|---|---|
| **法条找案例** | 法律适用明确 | 在数据库找到目标法条，点击"引用本法条"的案例 |
| **案例找案例** | 领域陌生 | 先找种子案例→精读提取关键词→扩大检索 |
| **关键词检索** | 最常用 | 规范性关键词+非规范性关键词组合 |
| **主体找案例** | 特定企业 | 在当事人字段输入公司名，检索其所有判决 |

### 关键词提取

```
规范性关键词：法律概念/术语
  例：案由、罪名、自首、国家工作人员、非法占有目的

非规范性关键词：自然事实/通俗用语
  例：App、代持股、干股、逾期完工

提取方法：
1. 从法条中拆分（如《公司法》第63条→一人公司/股东/连带责任）
2. 从案件材料中提取行业术语
3. 从案件事实中归纳
```

## 类案相似度评估

### 相似度五维评估

| 维度 | 权重 | 评估内容 |
|---|---|---|
| 法律关系一致 | 30% | 案由/当事人地位/权利义务结构/基础法律关系 |
| 争议焦点对应 | 25% | 核心法律问题是否同一/涉及法条是否相同 |
| 关键事实相似 | 25% | 要件事实是否类似/决定性情节是否一致 |
| 裁判规则参照 | 15% | 裁判要旨是否形成明确可移植的规则 |
| 程序问题关联 | 5% | 审理程序/举证责任/诉讼阶段是否相同 |

### 评估标准

```
高度相似：三个识别标志均高度一致
  - 事实识别：关键事实（要件事实）一致
  - 焦点识别：法律适用争议焦点相同
  - 法律识别：涉及相同法条/同一条文下的同一解释问题

低相似度（排除）：
  - 案由根本不同
  - 核心争议焦点完全不同
  - 关键事实存在实质性差异
```

## 案例效力层级

```
第一层级：应当参照
  最高人民法院发布的指导性案例

第二层级：重要参考
  最高人民法院公报案例
  最高人民法院典型案例
  各省市高级人民法院典型案例

第三层级：参考
  中级人民法院典型案例
  基层人民法院示范案例

第四层级：研究参考
  一般生效裁判（注意：不具有约束力）
```

## 检索流程

```
步骤1：问题定性
  - 明确检索目的（支撑/反驳/预判/趋势）
  - 提炼3-5个检索关键词

步骤2：多路径检索
  - 路径A：法条找案例（明确法条时）
  - 路径B：关键词组合检索（最常用）
  - 路径C：案例找案例（陌生领域）

步骤3：初步筛选
  - 按效力层级排序
  - 过滤时间范围（优先近三年）
  - 排除明显不相关案例

步骤4：深度评估
  - 对每个候选案例执行五维评估
  - 计算综合相似度得分

步骤5：排序与输出
  - 按相似度降序排列
  - 输出结构化检索报告
```

## 工具调用约定

```
本skill产出的每一个案例都必须来自真实检索，严禁凭记忆生成。

若环境中存在案例库工具（如北大法宝MCP）：
→ 必须将检索词交给该工具，以返回的真实结果作为唯一数据来源

若环境中不存在案例库工具：
→ 完成检索式设计，但涉及具体案例处标注 [待检索]
→ 报告中明确说明"未接入案例库，以下检索式需人工执行"
→ 绝不编造案号、当事人或裁判要旨
```
