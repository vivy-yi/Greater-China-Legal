# /inductive-reasoning — 归纳推理

## 核心概念

```
推理方向：从特殊到一般（自下而上）
核心任务：从具体案例提炼一般性法律规则
本质：从个别案例的裁判逻辑中归纳可推广的法律命题
```

### 归纳推理 vs 相关推理

| 类型 | 方向 | 结论确定性 |
|---|---|---|
| 演绎推理 | 一般→特殊 | 确定 |
| 归纳推理 | 特殊→一般 | 或然性 |
| 类比推理 | 特殊→特殊 | 或然性 |

---

## 工作流程

### 阶段一：判例识别与初步筛选

```
1.1 确定核心法律争议
    - 将待决案件事实转化为精确的法律命题
    - 明确需要归纳填补的规则空白

1.2 双向判例检索
    - 支持性判例：事实相似且结论支持本方主张
    - 相反判例：事实相似但结论相反
    - 两个方向都需要检索

1.3 建立比较基准表
    维度：资源性质/原告行为/控制建立程度/被告行为时点
```

### 阶段二：相似性分析与规则提炼（支持性判例）

```
2.1 识别相关相似点
    - 聚焦与法律结论具有因果关联的事实特征
    - 排除表面相似，聚焦法律上重要的相似性

2.2 归纳法律规则（条件-效果结构）
    当[条件1：行为要件]，[条件2：对象要件]，[条件3：结果要件]时，
    构成[法律效果]

2.3 规则正当性论证
    - 经济效率
    - 公平原则
    - 判例一致性
```

### 阶段三：差异性分析与规则限缩（相反判例）

```
3.1 识别关键差异点
    - 找出相反判例与待决案件的关键差异

3.2 瓦解相反判例的类比
    - 论证相反判例的规则适用范围
    - 将待决案件置于相反判例规则之外

3.3 精细化关键概念
    - 提出概念的层级结构
    - 论证适用只需达到某一级即可
```

### 阶段四：论证构建

```
标准格式：
大前提：[归纳所得法律规则]
  1. [条件1]
  2. [条件2]
  3. [条件3]

Rule-Based Explanation:
  4. 相似性建立：[案例]与本案在[维度]具有相关相似性
  5. 差异性排除：[相反判例]与本案存在关键差异，不适用
  6. 规则适用：本案事实满足规则第X条件，故应产生[法律效果]
```

### 阶段五：批判性检验

```
5.1 反向检验：构造形式相同但结论荒谬的类比，检验推理合理性
5.2 差异性再审视：确认差异是否足以推翻类比
5.3 规则冲突检查：验证归纳规则是否与上位法/强制性规定冲突
5.4 结论可接受性评估：是否符合法律正义和公平原则
```

## 输出格式

```
## 归纳推理报告

### 归纳目标
- 从[数量]个案例归纳[法律问题]的裁判规则
- 样本案例：[案例清单]

### 支持性判例分析
- 案例1：[案名] — 裁判要旨：[要旨]
- 案例2：[案名] — 裁判要旨：[要旨]

### 相反判例分析
- 案例X：[案名] — 与支持性判例的关键差异：[差异]

### 归纳法律规则
```
当[条件1：行为要件]，
    [条件2：对象要件]，
    [条件3：结果要件]
则构成[法律效果]
```

### 规则正当性
- 经济效率：[论证]
- 公平原则：[论证]
- 判例一致性：[论证]

### 相反判例的适用范围
- [相反判例]的规则仅适用于：[适用范围描述]
- 本案：[不在/在]该适用范围

### 置信度：🟢高 / 🟡中 / 🔴低
- 理由：[判断理由]
```
