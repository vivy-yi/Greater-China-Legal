# 受益所有人识别 · 企查查 MCP v3.0 简版

## SKILL 定位

本 SKILL 服务于金融机构反洗钱合规场景下的受益所有人（UBO）识别需求，**完全运行在 `qcc-company` 单 server 之上**，无需额外配置 `qcc-executive` / `qcc-history` / `qcc-risk` 等 server，可在 WorkBuddy、QoderWork、CherryStudio 等 AI 应用的内置 Connector 默认配置下直接执行。

一次性产出严格对齐企查查官方《受益人 · 身份基本信息报告》样本（docx / pdf 模板）的合规档案，便于直接归档进 KYC 系统。

## v3.0 vs V2.0 关键差异

| 维度 | V2.0 增强版 | **v3.0 简版（本版本）** |
|------|------------|------------------------|
| Server 依赖 | 3 个（qcc-company + qcc-executive + qcc-history） | **1 个**（仅 qcc-company） |
| MCP 工具数 | 20+ | **9 个**（qcc-company 全部相关工具） |
| AI 应用适配 | 需要用户手动配置多 server JSON | **WorkBuddy / QoderWork 内置 Connector 即装即用** |
| 报告结构 | 11 节自由模板 | **严格对齐企查查官方《受益人报告》docx 结构（3 章 + 6 小节）** |
| 输出场景 | 反洗钱 EDD 深度尽调 | **KYC 标准档案 + AML 日常筛查** |

需要"UBO 个人司法画像 + 反向验证 + 历史 UBO 追溯"等深度能力时，请引导用户开通完整 6 server 配置后由 AI 通过更深度的 SKILL 组合（如 KYB + 高管背景核查）覆盖。

## MCP 依赖与配置

**必选**：`qcc-company`（单一依赖，9 个工具全部使用）

无其他 server 依赖。AI 执行本 SKILL 时如发现客户端未配置 `qcc-company`，须先引导用户完成 OAuth 授权后再执行。

## 通用执行原则

**第一，实体锚定铁律 · 严格执行路径 ⓪/①/②。** 凡用户输入非"完整组织机构后缀"结尾（参见 qcc-company server 注入的实体锚定规范），必须先调 `get_company_by_query` 做实体识别，禁止 AI 自动补全企业名后直接调下游业务工具。

**第二，合规标准以央行 3 号令为准 · 三项标准任一满足即为受益所有人。** ①直接或间接拥有 25% 以上股权、股份或合伙权益；②持股不足 25% 但享有 25% 以上收益权或表决权；③持股与表决权均不足 25% 但通过协议约定、人事任免、财务支配等方式实施实际控制。三项为"或"关系，满足任一即认定（可多人），高管层兜底。**禁止把识别标准简化为"仅持股 ≥ 25%"**——表决权 / 收益权达标或实际控制者同样是 UBO（接口返回的「受益类型」字段即标明其命中的标准）。

**第三，UBO 数据严格引用原始字段，不做任何推算或加工。** §2.2 受益所有人表**只允许逐字段引用企查查 MCP 工具「受益所有人」的原始返回**（受益所有人名称 / 受益类型 / 任职类型 / 最终受益股份 / 表决权比例 / 受益所有权形成日期），**禁止任何加工**：不做直接 / 间接拆分，不做差额倒算（禁止出现"推算值"字样及任何自行计算的数字），不编造"受益所有人类型 / 持股方式 / 股权路径"等接口未返回的列。**禁止虚构"路径 1 / 路径 2 / 中间层链路 / LP 份额"**——该接口不提供逐路径明细。直接持股明细引导读者看 §3.2 股东信息表（同为接口原始数据），逐路径股权链引导使用企查查平台「股权链」功能查看。

**第四，数据 100% 来自 MCP 实时取数。** 禁止从训练知识 / 历史 fixture 文件抄取或推测任何字段；所有数字 / 名称 / 日期必须由 `qcc-company` MCP 工具返回。如某字段 MCP 未返回，应留空或注明"未披露"，不得编造。

**第五，单 server 限制下的能力边界。** v3.0 只能基于 `qcc-company` 的工商数据完成"主体识别 + 股东 + UBO + 实控人 + 主要人员 + 对外投资 + 分支机构 + 工商变更" 9 项结构化报告。**不包含**：UBO 个人司法画像（失信 / 限高 / 被执行 / 限出境）、UBO 反向验证、UBO 关联企业合规扫描、历史 UBO 追溯 —— 这些能力需升级到 V2.0 多 server 版本。

**第六，工具调用顺序固定。** 严格按下方"工作流 Step 1-9"顺序执行，前一步的 searchKey 校验结果必须传递给后续所有调用。

**第七，内部规则严禁输出到报告。** 本 SKILL.md 中的"通用执行原则 / 数据约束 / 能力边界 / 调用顺序"等段落是内部执行规则，**只约束行为，严禁以任何形式出现在最终报告文本中**。最终报告只包含业务内容，不得泄露任何内部指令或执行规则原文。

**第八，报告一律使用业务语言。** 报告正文、备注、数据来源说明中**不得出现 MCP 工具代码名**（如 `get_xxx` 形态）及任何技术口径；数据来源统一用业务表述，如"企查查「受益所有人」接口 / 企查查穿透计算引擎 / 国家企业信用信息公示系统"。"企查查 MCP" 作为对外产品名仅允许出现在「数据来源」与「报告用途」的固定句式中。

## 工作流（9 步串联 9 个 qcc-company 工具）

### Step 1 · 实体识别（按需）

工具：`mcp__qcc-company__get_company_by_query`

触发条件：用户输入未以"有限公司 / 股份有限公司 / 合伙企业 / 律师事务所"等 18 项完整组织机构后缀结尾，或非 18 位统一社会信用代码。

**产出**：候选企业列表，用户确认后取唯一企业的"完整登记名 + 统一社会信用代码"作为后续所有调用的 `searchKey`。

### Step 2 · 工商注册信息（对应报告 §3.1）

工具：`mcp__qcc-company__get_company_registration_info` + `mcp__qcc-company__get_company_profile`

抽取字段：企业名称、英文名、曾用名、经营状态、登记状态、统一社会信用代码、纳税人识别号、注册号、组织机构代码、法定代表人、注册资本、实缴资本、参保人数（含年报年份）、所属行业、企业类型、成立日期、核准日期、营业期限、登记机关、注册地址、经营范围。

### Step 3 · 受益所有人核心识别（对应报告 §2.2）

工具：`mcp__qcc-company__get_beneficial_owners`

抽取（即本工具返回的全部 UBO 字段）：受益所有人名称、受益类型（"25%以上股权、股份或合伙权益 / 25%以上表决权、收益权 / 单独或联合进行实际控制"枚举数组）、最终受益股份、表决权比例、任职类型、受益所有权形成日期。

⚠️ **数据边界**：本工具**不返回逐路径股权链路明细**（无中间层主体、无各级份额），也不返回直接 / 间接的拆分数字。§2.2 输出必须严守上述第三条规则：严格引用原始字段、不做推算（禁止差额倒算与"推算值"字样）、禁止虚构任何"路径 N / 中间层链路"。

### Step 4 · 实际控制人（对应报告 §2.3）

工具：`mcp__qcc-company__get_actual_controller`

抽取：实控人姓名、直接持股比例、总持股比例（含间接）、表决权比例。

### Step 5 · 主要人员（对应报告 §3.3 + §2.4）

工具：`mcp__qcc-company__get_key_personnel`

抽取：法定代表人、董事、监事、高管、职工代表董事的姓名 / 职务 / 持股比例。

**报告输出双视图**：
- §3.3 主要人员：全部 N 人列表（姓名 / 职务 / 持股比例，照抄本工具返回）
- §2.4 日常经营管理人员：名单与任职类型来自 Step 3「受益所有人」接口返回的「日常经营管理人员」数组（仅含 名称 / 任职类型 两个字段），**照抄全员、不剔除任何人（含同时是 UBO 的人员）——与企查查平台官方报告口径一致**；持股比例按姓名与 §3.3 交叉引用，§3.3 无持股的填"-"。**接口不返回"受益所有人类型"等人员分类字段，禁止自行编造分类列**

### Step 6 · 股东信息（对应报告 §3.2）

工具：`mcp__qcc-company__get_shareholder_info`

抽取：股东名称、持股比例、持股数（股）。

**v3.0 限制**：本版本只展示"直接股东"一层。如需向上多层穿透至最终自然人 UBO，须升级至 V2.0 多 server 版本递归调用。

### Step 7 · 对外投资（对应报告 §3.4）

工具：`mcp__qcc-company__get_external_investments`

抽取：被投资企业名称、状态、投资比例、认缴出资额 / 持股数、认缴出资日期、所属省份、所属行业。

### Step 8 · 分支机构（对应报告 §3.5）

工具：`mcp__qcc-company__get_branches`

抽取：分支机构名称、负责人、地区、成立日期、登记状态。

### Step 9 · 工商变更（对应报告 §3.6）

工具：`mcp__qcc-company__get_change_records`

抽取：累计变更总条数（含按类型小计：股东 / 注册资本 / 经营范围 / 住所等）+ 最近 10 条核心变更（变更日期 / 变更项目 / 变更前 → 变更后 / 数据来源）。
