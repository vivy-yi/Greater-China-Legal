# /customize — 通用配置调整

## 原理

cold-start 完成场景 CLAUDE.md 的初次填充，customize 用于后续调整特定字段而无需重新运行完整向导。自动识别当前场景 CLAUDE.md 中所有可配置项。

## 工作流程

### Step 1：读取当前配置

读取当前场景的 `CLAUDE.md`。如果文件中仍包含 `[填空]` 标记，提示用户先运行 cold-start：

> "当前 CLAUDE.md 仍有未配置的 `[填空]` 字段。建议先运行 `cold-start-interview` 完成基础配置，再用 customize 调整特定项。"

### Step 2：展示可调整项

读取 CLAUDE.md 各节标题，按层级展示调整地图：

```
📋 当前配置可调整项 — [场景名]

## 公司基本信息
  - 公司名称：[当前值]
  - 业务描述：[当前值]
  - 外部律师：[当前值]

## 数据源配置
  - 优先级 1 (yuandian MCP)
  - 优先级 2 (pkulaw MCP)
  - 降级规则

## 风险等级
  - 🔴 HIGH 阈值
  - ⚠️ MEDIUM 阈值

## 升级决策门
  - 升级条件列表

> 输入要修改的节标题或具体字段名，或输入"完成"退出。
```

### Step 3：修改指定项

用户指定要修改的节后，逐项展示当前值并询问新值：

```
系统：公司名称当前为 "ABC科技有限公司"，要修改吗？(Y/N)
用户：Y
系统：请输入新的公司名称：
用户：XYZ科技有限公司
系统：✅ 已更新为公司名称：XYZ科技有限公司
```

### Step 4：写入并确认

修改完成后，展示 diff 摘要：

```
✅ 修改完成

已修改：
- 公司名称：ABC → XYZ

文件已更新：[场景路径]/CLAUDE.md

运行 customize 可继续修改其他项。
```

## 高阶用法

### 按关键词直接修改

用户输入包含关键词时，直接定位到对应配置项：

| 关键词 | 定位项 |
|--------|--------|
| "公司"、"名称" | 公司基本信息 → 公司名称 |
| "律师"、"外部" | 外部律师联系方式 |
| "金额"、"阈值" | 合同金额阈值 |
| "风险" | 风险等级定义 |
| "数据源"、"MCP" | 数据源配置 |
| "升级"、"决策" | 升级决策门 |

### 批量修改（YAML 导入）

用户可粘贴 YAML 格式的批量修改：

```yaml
公司名称: XYZ科技有限公司
外部律师: 李律师 139xxxx
```

系统解析后逐项确认再写入。

## 本技能不做什么

- 不重新运行完整冷启动。只修改指定项。
- 不验证配置内容的法律准确性。
- 不修改 CLAUDE.md 的结构或节标题——只改变字段值。
