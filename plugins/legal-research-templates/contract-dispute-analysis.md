# Template: 合同纠纷分析

## 一、触发条件

律师输入包含：
- "合同纠纷" / "合同争议" / "合同违约"
- "分析这份合同" / "评估这个条款"
- "这份合同能不能解除" / "对方违约了我怎么办"

适用场景：合同效力争议、合同履行争议、违约责任承担、合同解除/终止。

## 二、工作流（5 步）

### 步骤 1：法律要素提取
**调 atomic**：`legal-element-extraction`
- **输入**：合同全文 + 律师陈述
- **输出**：结构化要素（主体 / 标的 / 履行 / 违约 / 救济）
- **预期耗时**：< 30 秒

### 步骤 2：法条检索
**调 atomic**：`legal-article-retrieval`
- **输入**：步骤 1 输出的法律要素
- **输出**：相关法条列表（合同法 / 民法典合同编 / 司法解释）
- **预期耗时**：< 20 秒

### 步骤 3：案例检索
**调 atomic**：`case-retrieval`
- **输入**：步骤 1 的争议焦点 + 步骤 2 的法条
- **输出**：3-5 个类似判例
- **预期耗时**：< 30 秒

### 步骤 4：演绎推理
**调 atomic**：`deductive-reasoning`
- **输入**：步骤 1 + 2 + 3 的全部输出
- **输出**：法条 → 事实 → 结论 的推理链
- **预期耗时**：< 60 秒

### 步骤 5：论证强度评估
**调 atomic**：`argument-strength-evaluation`
- **输入**：步骤 4 的推理链
- **输出**：论证强度评分 + 弱点提示
- **预期耗时**：< 30 秒

## 三、输入契约

律师需提供：
- **必需**：
  - 合同全文（或关键条款）
  - 一句话描述争议点
- **可选**：
  - 对方当事人信息
  - 已发生的履行情况
  - 类似案例（如有）

## 四、输出契约

最终给律师：
```markdown
## 合同纠纷分析报告

### 一、要素提取
[主体 / 标的 / 履行 / 违约 / 救济]

### 二、相关法条
[法条列表 + 适用条款]

### 三、参考判例
[3-5 个类似判例 + 关键裁判要点]

### 四、推理链
[大前提 → 小前提 → 结论]

### 五、论证强度
[评分 1-10 + 弱点 + 改进建议]
```

## 五、失败 fallback

| 失败位置 | fallback 动作 |
|---|---|
| 步骤 1（要素提取）失败 | 提示律师补充更详细的合同描述 |
| 步骤 2（法条检索）失败 | 跳过法条检索，直接进入步骤 4（基于律师陈述推理） |
| 步骤 3（案例检索）失败 | 跳过案例检索，继续推理（标记"无案例参考"） |
| 步骤 4（演绎推理）失败 | 改用 `legal-abductive-reasoning`（溯因推理） |
| 步骤 5（论证评估）失败 | 跳过评估，直接输出推理结果（标记"未评估"） |
| 整链失败 | 报错让人工核对，给律师应急建议：咨询专业律师 |

## 六、关联 skill

- **跨场景协作**：合同审查完毕 → 律师要求"生成可外发版" → `legal-document-redaction` 脱敏
- **案件存档**：分析完成后 → `matter-workspace new` 创建案件
- **写作**：律师要书面意见 → `legal-document-formatting` 输出法律意见书
