# Template: 劳动争议

## 一、触发条件

律师输入包含：
- "劳动争议" / "劳动仲裁" / "劳动纠纷"
- "解除劳动合同" / "拖欠工资" / "经济补偿金" / "工伤"
- "员工要告公司" / "公司要开除员工"

适用场景：劳动合同解除争议、工资拖欠、工伤赔偿、社保纠纷、竞业限制。

## 二、工作流（4 步）

### 步骤 1：要素提取
**调 atomic**：`legal-element-extraction`
- **输入**：劳动争议事实 + 律师陈述
- **输出**：结构化要素（用人单位 / 劳动者 / 劳动关系 / 争议焦点）

### 步骤 2：法条检索
**调 atomic**：`legal-article-retrieval`
- **输入**：劳动法 + 劳动合同法 + 社保法 + 工伤保险条例 + 司法解释
- **输出**：相关法条列表

### 步骤 3：案例检索
**调 atomic**：`case-retrieval`
- **输入**：争议焦点 + 法条
- **输出**：5-10 个类似劳动仲裁/判决案例

### 步骤 4：反事实推理
**调 atomic**：`counterfactual-reasoning`
- **输入**：要素 + 法条 + 案例
- **输出**：多种可能的处理路径 + 各路径后果

## 三、输入契约

律师需提供：
- **必需**：
  - 劳动关系事实（入职 / 离职 / 工资 / 岗位）
  - 争议焦点
- **可选**：
  - 劳动合同文本
  - 工资条 / 考勤记录
  - 双方协商记录

## 四、输出契约

```markdown
## 劳动争议分析

### 一、案件要素
[结构化要素]

### 二、相关法条
[劳动法 / 劳动合同法 / 社保法 / 司法解释]

### 三、参考案例
[5-10 个类似案例 + 裁判要点]

### 四、可能路径
- 路径 A：[劳动仲裁 → 一审 → 二审]，预期 [胜诉率 / 时长 / 成本]
- 路径 B：[调解]，预期 [结果 / 时长]
- 路径 C：[直接诉讼]，预期 [结果 / 时长]
```

## 五、失败 fallback

| 失败位置 | fallback 动作 |
|---|---|
| 步骤 2 失败 | 跳过法条检索，基于常识判断 |
| 步骤 3 失败 | 跳过案例检索 |
| 步骤 4 失败 | 输出单路径（仲裁）建议 |
| 整链失败 | 提示律师人工分析 + 给出最低限度建议 |

## 六、特别提示

劳动争议常见 1 年仲裁时效。涉及：
- 工资拖欠 → 《劳动法》第 50 条 + 劳动合同法第 30 条
- 解除 → 劳动合同法第 39-40 条（过错性解除）/ 41 条（经济性裁员）
- 工伤 → 工伤保险条例 + 民事赔偿

agent 输出**仅供初步参考**，具体维权路径**应咨询当地劳动仲裁机构**。
