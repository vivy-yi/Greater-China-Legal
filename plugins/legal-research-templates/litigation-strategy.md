# Template: 诉讼策略

## 一、触发条件

律师输入包含：
- "诉讼策略" / "怎么起诉" / "怎么应诉"
- "庭审准备" / "这个案子怎么打"
- "胜诉可能性多大" / "起诉流程"

适用场景：诉前评估、起诉准备、应诉答辩、庭审策略、上诉策略。

## 二、工作流（4 步）

### 步骤 1：要素提取
**调 atomic**：`legal-element-extraction`
- **输入**：案件事实 + 律师陈述
- **输出**：结构化要素

### 步骤 2：判例检索 + 类比
**调 atomic**：
- `case-retrieval` → 找 5-10 个类似判决
- `analogical-reasoning` → 抽取类比规则

### 步骤 3：判决预测
**调 atomic**：`legal-judgment-prediction`
- **输入**：要素 + 类比规则
- **输出**：胜诉概率预测 + 关键变量

### 步骤 4：战略风险优先级
**调 atomic**：`strategic-risk-prioritization`
- **输入**：预测结果 + 律师资源约束
- **输出**：行动优先级（起诉/和解/调解）

## 三、输入契约

律师需提供：
- **必需**：
  - 案件事实（时间 / 人物 / 行为 / 结果）
  - 现有证据状态
- **可选**：
  - 对方当事人态度
  - 律师资源（时间 / 预算）
  - 商业关系考量

## 四、输出契约

```markdown
## 诉讼策略建议

### 一、案件要素
[结构化要素]

### 二、判例参考
[5-10 个类似判决 + 关键裁判要点]

### 三、胜诉概率
[百分比预测 + 关键变量]

### 四、行动建议
- 优先级 1：[诉讼路径 A]（预期 X% / 风险 Y）
- 优先级 2：[和解方案 B]（预期 X% / 风险 Y）
- 优先级 3：[调解方案 C]
```

## 五、失败 fallback

| 失败位置 | fallback 动作 |
|---|---|
| 步骤 1 失败 | 提示律师补充事实 |
| 步骤 2 失败 | 跳过类比，直接步骤 3 |
| 步骤 3 失败 | 输出"无法预测胜诉概率"+ 强调需专业评估 |
| 步骤 4 失败 | 输出"请咨询专业诉讼律师"+ 列出关键考量维度 |
| 整链失败 | **强制标记 [高风险] + 建议律师面谈客户** |

## 六、特别提示

诉讼策略是高风险决策，agent 输出**仅供初步参考**，最终决策**必须由执业律师根据具体案情做出**。
