# Template: 跨境交易

## 一、触发条件

律师输入包含：
- "跨境交易" / "跨境支付" / "跨境数据传输"
- "跨境合规" / "出海" / "GDPR"
- "跨境投资" / "境外上市"

适用场景：跨境贸易、跨境支付、跨境数据出境、跨境投资、境外上市。

## 二、工作流（4 步）

### 步骤 1：要素提取
**调 atomic**：`legal-element-extraction`
- **输入**：跨境交易事实 + 涉及法域
- **输出**：结构化要素（交易类型 / 当事方 / 资金流 / 数据流）

### 步骤 2：多法域规则检索
**调 atomic**：`cn-judicial-rules`
- **输入**：涉及法域列表（cn / hk / sg / eu 等）
- **输出**：相关法域规则 + 互斥/冲突点

### 步骤 3：法律解释
**调 atomic**：`legal-interpretation-argument`
- **输入**：多法域规则
- **输出**：法律解释 + 适用优先级

### 步骤 4：冲突解决
**调 atomic**：`conflict-resolution`
- **输入**：解释 + 冲突点
- **输出**：合规路径 + 风险等级

## 三、输入契约

律师需提供：
- **必需**：
  - 跨境交易类型
  - 涉及法域（中/外）
  - 资金流 / 数据流方向
- **可选**：
  - 双方主体性质（国企 / 民企 / 外资）
  - 涉及监管机构
  - 制裁清单核查需求

## 四、输出契约

```markdown
## 跨境交易分析

### 一、交易要素
[结构化要素 + 资金 / 数据流图]

### 二、各法域规则
- cn-mainland: [PIPL / 数据出境 / 外汇管理 / 制裁]
- hk: [数据保护条例 / 制裁]
- sg: [PDPA / 制裁]
- eu: [GDPR / 数字服务法 / 制裁]

### 三、法律适用优先级
[冲突规则 + 适用顺序]

### 四、合规路径
- 数据出境备案：[PIPL § 38 安全评估 / 标准合同 / 认证]
- 外汇合规：[直接投资 / 跨境融资]
- 制裁规避：[OFAC / EU sanctions / UN sanctions]
```

## 五、失败 fallback

| 失败位置 | fallback 动作 |
|---|---|
| 步骤 2 失败 | 仅按 cn-mainland 评估 + 提示"未完整覆盖目标法域" |
| 步骤 3 失败 | 输出多法域规则原文 + 提示"需专业解释" |
| 步骤 4 失败 | 列出所有冲突点 + 建议"逐项咨询各法域律师" |
| 整链失败 | **强制标记 [高风险] + 强烈建议人工审查** |

## 六、特别提示

跨境交易涉及**多个法域**和**多个监管机构**，是高风险场景：
- 数据出境 → 国家网信办安全评估
- 反制裁 → 商务部 / 央行
- 反洗钱 → 央行 / 外管局
- 反垄断 → 市场监管总局 / 商务部

**任何跨境合规输出仅供初步参考，必须由专业律师 + 当地法律顾问双重审查**。
