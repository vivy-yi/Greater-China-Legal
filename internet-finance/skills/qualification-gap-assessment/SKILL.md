# /qualification-gap-assessment — 牌照资质差距评估

## 加载上下文

**首次使用时：** 读取 `../CLAUDE.md` 获取牌照申请条件/差距分析框架。

## 工作流程

1. 读取 `../CLAUDE.md` → 获取申请条件/评估维度
2. 对照目标牌照要求
3. 评估现有资质
4. 识别差距
5. 输出整改建议

## 评估五维度

| 维度 | 内容 |
|---|---|
| 注册资本 | 实缴货币资本是否达标 |
| 风控体系 | 风险管理制度/模型/团队 |
| 信息系统 | 系统安全等级保护/数据安全 |
| 高管资质 | 董事/高管是否有相关任职资格 |
| 股东资质 | 股东背景/出资能力/关联关系 |

## 常见差距

| 差距类型 | 说明 |
|---|---|
| 注册资本不足 | 网络小贷须实缴到位 |
| 股东背景不符 | 须中资背景/无重大违规记录 |
| 信息系统未备案 | 等保三级须先完成 |
| 风控体系缺失 | 须建立完整风控模型 |
| 高管资质不足 | 须有金融从业经验 |

## 整改建议

| 整改项 | 优先级 | 预计时间 |
|---|---|---|
| 注册资本增资 | P0 | 约2-3月 |
| 股东结构重组 | P0 | 约3-6月 |
| 信息系统等保 | P1 | 约2-4月 |
| 风控体系搭建 | P1 | 约3-6月 |
| 高管引进 | P2 | 按需 |
