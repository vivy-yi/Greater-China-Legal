# 法律诊所数据源清单 — 中国大陆

> 本清单整理法律诊所场景常用数据源，按 [YD]/[WKL]/[GOV]/[BD]/[model] 五级标注分类。
> legal-clinic 是**涉及实际当事人**的场景，对数据源准确性要求高于学习场景。
> 与 employment-legal 的核心差异：**督导审核机制** + **当事人沟通档案**。
> 实际检索入口见 `references/查询路径.md`。

## 七、来源标注规范（法援场景特化）

每个结论/数字必须挂载来源标签：

| 标注 | 含义 | 法援示例 |
|---|---|---|
| `[YD]` | 元典检索结果 | `[YD] 民法典第 1062 条` |
| `[WKL]` | 无合力检索结果 | `[WKL] 类案 XX 号` |
| `[BD]` | 北达/内部档案 | `[BD] 督导审核意见 2026-06-XX` |
| `[GOV]` | 政府平台数据 | `[GOV] 司法部法律援助法（2022.01.01 施行）` |
| `[web]` | 网络搜索结果 | `[web] 北京市最低工资标准 2024` |
| `[model]` | 模型知识（须核实） | 不允许直接引用，须经督导审核 |
| `[user]` | 当事人提供的信息 | `[user] 当事人陈述：工资每月 8000 元` |
| `[settled — last confirmed YYYY-MM-DD]` | 已验证的稳定引用 | 已督导审核的法条引用 |

**法援场景特殊规则**：
- 当事人陈述必须标注 `[user]`，且**明确区分事实陈述与法律评价**
- 督导审核意见标注 `[BD] supervisor + 日期`
- 不允许未经验证的 `[model]` 引用——所有结论须可追溯
- 内部档案优先于外部数据源

## 九、当事人隐私保护

法援场景涉及真实当事人，**数据保护要求最高**：

### 9.1 隐私分级

| 数据类型 | 保护级别 | 处理方式 |
|---|---|---|
| 身份证号 | 高 | 脱敏处理（仅保留前 6 位 + 后 4 位） |
| 联系方式 | 高 | 仅律师/学生/督导可见 |
| 银行账号 | 高 | 仅律师/学生/督导可见 |
| 病历/诊断 | 高 | 仅律师/督导可见 |
| 案件事实 | 中 | 内部可见 |
| 法律分析 | 低 | 公开讨论 |

### 9.2 数据存储

```
- 案件档案存储在 BD 内部系统
- 不上传到公网或公共云
- 案件结案后归档保留 10 年
- 涉及未成年人案件保留 20 年
```

### 9.3 数据共享

```
- 与督导共享：须明确授权
- 与对方律师/法院：仅必要信息
- 与媒体：禁止
- 与学术研究：脱敏后可用
```

## 十、法援场景风险等级说明

legal-clinic 风险等级**主要为 medium**，部分场景为 high：

| 场景 | 风险等级 | 原因 |
|---|---|---|
| C1 劳动法援 | medium-high | 影响当事人经济权益 + 时效紧迫 |
| C2 消费者维权 | medium | 影响金额 + 群体性可能 |
| C3 婚姻家事 | medium-high | 影响家庭关系 + 未成年人保护 |
| C4 行政诉讼援助 | high | 民告官 + 被告为政府 + 法律专业度高 |

**所有场景必须督导复核**，不可由学生独立处理。
