# 法律诊所判断框架 — 中国大陆

> 本框架用于 legal-clinic 16 个 skill 的场景识别与判断逻辑，作为法援场景的统一依据。
> legal-clinic 是**学生办案+督导审核**模式，与 employment-legal（律师）和 law-student（学习）的核心差异在此。
> 各 skill 内部已有场景识别段，本文件描述跨 skill 的整体决策路由。

---

## 一、法援场景识别（4 场景簇）

```
当事人案件类型？
├── 劳动争议（C1 劳动法援）
│   ├─ 劳动合同解除/工资拖欠/工伤待遇
│   ├─ skill：client-intake / research-start / memo / draft / supervisor-review-queue / status / deadlines
│   └─ 关键判断：仲裁前置 + 60 日时效 + 经济困难认定
│
├── 消费者维权（C2 消费者维权）
│   ├─ 消费欺诈/质量纠纷/格式合同
│   ├─ skill：client-intake / client-letter / client-comms-log / form-generation
│   └─ 关键判断：消保法 + 小额诉讼 + 调解优先
│
├── 婚姻家事（C3 婚姻家事）
│   ├─ 离婚/抚养/继承/家事调解
│   ├─ skill：client-intake / memo / draft / plain-language-letters
│   └─ 关键判断：婚姻法 + 未成年人保护 + 避免刺激性表述
│
└── 行政诉讼援助（C4 行政诉讼援助）
    ├─ 民告官/复议前置/国家赔偿
    ├─ skill：client-intake / research-start / memo / deadlines / supervisor-review-queue
    └─ 关键判断：复议前置 + 被告主体 + 6 个月起诉时效
```

## 二、场景入口路由（当事人问什么？）

| 当事人表述 | 识别场景 | 推荐 skill |
|---|---|---|
| "老板欠我工资 / 被违法解除 / 工伤不管" | C1 | client-intake → research-start → memo → draft |
| "买的东西是假的 / 商家欺诈" | C2 | client-intake → client-letter |
| "要离婚 / 争抚养权 / 继承纠纷" | C3 | client-intake → memo → draft |
| "政府违法处罚 / 强拆 / 不作为" | C4 | client-intake → research-start → memo |
| "我经济困难 / 没钱请律师" | 任意 | 触发 client-intake 经济困难审核 |
| "我是农民工 / 残疾人 / 老年人" | 任意 | 触发特殊群体免审查通道 |

**特殊通道：**
- 经济困难认定 + 特殊群体（残疾人/老年人/妇女/儿童/农民工）→ 优先
- 刑事案件：可能判处死刑/无期/未成年犯罪 → 公设辩护全覆盖
- 见义勇为 + 受害者 → 部分地区免审查

## 三、CN 法律援助体系核心判断节点

### 3.1 经济困难认定

```
[节点 1] 申请人是否符合经济困难标准？
├─ 是 → 进入法援范围
└─ 否 → 拒绝 + 推荐其他渠道（一般律师/咨询）

判断标准：
  - 各省市最低生活保障 1.5-2 倍
  - 低收入户证明 / 五保户证明 / 失业证明
  - 农民工凭务工证明 + 收入证明
```

### 3.2 案件类型匹配

```
[节点 1] 案件类型？
├─ 劳动争议 → C1 路径
├─ 消费者权益 → C2 路径
├─ 婚姻家庭/继承 → C3 路径
└─ 行政诉讼 → C4 路径

[节点 2] 是否涉及特殊群体？
├─ 残疾人/老年人/妇女/儿童 → 免经济困难审查
├─ 农民工 → 部分地区简化
├─ 死刑/无期/未成年被告 → 公设辩护全覆盖
└─ 见义勇为者 → 部分地区优先

[节点 3] 是否涉及禁止代理情形？
├─ 对方是法律援助机构律师 → 不代理
├─ 案件已结案 → 不代理
└─ 其他不适宜代理情形 → 转介
```

### 3.3 案件分流

```
Step 1：案件受理（client-intake）
  ├─ 经济困难认定
  ├─ 案件类型识别
  ├─ 资格审核
  └─ 创建 client/[id]/ 文件夹

Step 2：案件研究（research-start / memo）
  ├─ 管辖法院
  ├─ 适用法律
  ├─ 时效紧迫度
  └─ 案件策略

Step 3：文书起草（draft / form-generation）
  ├─ 起诉状/答辩状/申请书
  ├─ 调解协议/律师函
  └─ 提交督导审核

Step 4：督导复核（supervisor-review-queue）
  ├─ 案件复杂程度（简单/中等/复杂）
  ├─ 风险等级（低/中/高）
  ├─ 时效紧迫（普通/紧急/特急）
  └─ 复核通过 → 提交

Step 5：当事人沟通（client-comms-log / client-letter / plain-language-letters）
  ├─ 受理通知
  ├─ 进度更新
  ├─ 法律意见（避免承诺）
  └─ 结案报告

Step 6：案件状态跟踪（status / deadlines）
  ├─ 立案前/审限中/上诉期/执行期
  ├─ 时效管理（关键时点）
  └─ 学期交接（semester-handoff）
```

## 四、4 场景核心判断

### 4.1 C1 劳动法援（最高频）

```
[节点 1] 是否劳动争议？
├─ 是 → 进入 C1
└─ 否 → 转其他场景

[节点 2] 解除类型？
├─ 用人单位单方解除 → 合法性判断（§36/§39/§40/§41 劳动合同法）
├─ 劳动者主动辞职 → 工资/补偿结算
└─ 协商解除 → 经济补偿 N

[节点 3] 仲裁时效？
├─ 已过 60 日时效 → 审查时效中止/中断
├─ 在时效内 → 准备仲裁申请
└─ 临近时效 → 加急处理

[节点 4] 是否涉及特殊群体？
├─ 三期女职工/工伤员工/医疗期员工 → 第 42 条禁止解除
├─ 高龄员工（连续工作满 15 年，距退休不足 5 年）→ 第 42 条
└─ 工会代表维权 → 第 43 条

升级条件：触及高风险情形必须督导复核（详见 §六）
```

### 4.2 C2 消费者维权

```
[节点 1] 是否消费纠纷？
├─ 是 → 进入 C2
└─ 否（合同/侵权）→ 转其他场景

[节点 2] 经营者类型？
├─ 商家（个体户/企业）→ 消保法
├─ 平台电商 → 消保法 + 电子商务法
└─ 预付款消费（健身/美容）→ 单用途商业预付卡管理办法

[节点 3] 争议金额？
├─ 简易程序（小额诉讼）→ 基层法院
│   标的额：各省市不同（通常 5 万以下，部分省份 10 万以下）
├─ 普通程序 → 标的额法院管辖
└─ 集体诉讼（众多受害者）→ 消协支持起诉

[节点 4] 是否存在欺诈？
├─ 是 → 消保法第 55 条 3 倍赔偿
├─ 质量不合格 → 退换 + 赔偿损失
└─ 虚假宣传 → 消保法 + 广告法
```

### 4.3 C3 婚姻家事

```
[节点 1] 案件类型？
├─ 离婚 → 婚姻法 + 民法典婚姻家庭编
├─ 抚养/赡养 → 民法典 + 未成年人保护法
├─ 继承 → 民法典继承编 + 遗嘱
└─ 家暴 → 反家庭暴力法 + 人身安全保护令

[节点 2] 是否涉及未成年人？
├─ 是 → 特别保护
│   ├─ 抚养权归属（最有利于未成年人原则）
│   ├─ 财产代管
│   └─ 心理疏导
└─ 否 → 普通处理

[节点 3] 是否调解前置？
├─ 离婚案件 → 法院调解优先（民法典婚姻家庭编）
├─ 继承纠纷 → 调解可行
└─ 家暴 → 不适用调解（保护令可直接申请）

[节点 4] 当事人情感状态？
├─ 冷静 → 标准处理
├─ 激动 → 情绪安抚 + 避免刺激性表述
└─ 悲伤 → 心理支持 + 避免归责性语言
```

### 4.4 C4 行政诉讼援助

```
[节点 1] 是否行政争议？
├─ 是 → 进入 C4
└─ 否 → 转其他场景

[节点 2] 是否复议前置？
├─ 自然资源权属（土地/矿产）→ 复议前置
├─ 纳税争议 → 复议前置
├─ 工伤认定 → 部分地区复议前置
├─ 其他可复议可诉讼 → 自由选择
└─ 复议机关逾期不决定 → 起诉复议机关

[节点 3] 被告主体？
├─ 作出具体行政行为的行政机关 → 被告
├─ 委托执法的组织 → 委托行政机关为被告
├─ 复议机关改变原行为的 → 复议机关为被告
└─ 多部门联合执法 → 共同被告

[节点 4] 起诉时效？
├─ 知道作出行为之日起 6 个月
├─ 复议后不服 → 收到复议决定书 15 日
└─ 最长保护期：5 年（不动产）/ 20 年（其他）

[节点 5] 案件类型？
├─ 撤销之诉（行政行为违法）
├─ 确认违法之诉
├─ 变更之诉（显失公正的行政处罚）
├─ 履行之诉（不作为）
└─ 国家赔偿（单独或一并）
```

## 五、督导审核判断树

```
[节点 1] 案件复杂程度？
├─ 简单（标准情形，事实清楚）→ 自查 + 1 审
├─ 中等（涉及地方差异或新法）→ 督导 1 审
└─ 复杂（金额大/影响广/涉及新领域）→ 督导 2 审 + 律师

[节点 2] 风险等级？
├─ 低（金额小/影响小）→ 1 审
├─ 中（标准情形）→ 督导 1 审
└─ 高（特殊群体/重大利益/涉外）→ 督导 2 审 + 律师

[节点 3] 时效紧迫？
├─ 普通 → 标准流程
├─ 紧急（30 日内）→ 加急处理
└─ 特急（7 日内）→ 督导实时审核

升级条件（详见 §六）：
- 经济补偿金计算 / 违法解除认定 → 复核
- 被告主体认定 → 复核
- 见效会见 / 阅卷 / 辩护意见 → 复核
- 复议前置要件 → 复核
- 涉外/涉港澳台 → 律师主导
```

## 六、升级决策门（强制督导/律师介入）

下列情形**必须**升级至督导或律师，不可由学生直接处理：

### 6.1 通用升级条件

- 涉及具体案件的法律意见
- 案件复杂程度超过学生能力
- 时效紧迫 + 金额重大（劳动争议标的 ≥ 10 万）
- 跨学科/跨法域问题
- 涉外或涉港澳台案件

### 6.2 刑事案件升级条件

- 会见（须律师在场或授权）
- 阅卷（须律师执业证）
- 辩护意见（须律师签字）
- 取保候审申请（须律师代理）
- 上诉/申诉（须律师代理）

### 6.3 行政诉讼升级条件

- 复议前置要件认定
- 被告主体认定
- 国家赔偿计算
- 跨区域行政诉讼
- 涉外行政诉讼

### 6.4 婚姻家事升级条件

- 家暴认定 + 人身安全保护令
- 涉外婚姻/继承
- 涉及未成年人权益重大事项
- 财产标的 ≥ 100 万
- 案件社会影响较大

### 6.5 消费者维权升级条件

- 集体诉讼/群体性消费纠纷
- 涉及上市公司/大型企业
- 跨境消费纠纷
- 标的额 ≥ 50 万

## 七、案件全生命周期管理

### 7.1 案件阶段判断

```
Stage 1：受理
├─ 审核经济困难 + 案件类型
├─ 创建 client/[id]/ 文件夹
├─ 输出：intake.md + 立案通知
└─ 时长：即时-7 日

Stage 2：研究
├─ 法律研究（research-start）
├─ 案件备忘录（memo）
├─ 督导初步审核
└─ 时长：7-30 日

Stage 3：调解
├─ 内部调解（如适用）
├─ 庭外调解
├─ 调解协议（plain-language-letters）
└─ 时长：可变

Stage 4：起诉/仲裁
├─ 文书起草（draft）
├─ 督导复核（supervisor-review-queue）
├─ 提交
└─ 时长：7-30 日

Stage 5：审理
├─ 跟进（status）
├─ 时效管理（deadlines）
├─ 出庭/出庭准备
└─ 时长：劳动仲裁 45 日 + 15 日；民事 6/9 个月

Stage 6：结案
├─ 结案报告
├─ 归档
├─ 学期交接（如需要，semester-handoff）
└─ 当事人回访
```

### 7.2 时效管理（关键时点）

| 案件类型 | 关键时效 |
|---|---|
| 劳动仲裁申请 | 60 日（特殊 1 年） |
| 民事起诉 | 3 年 |
| 行政起诉 | 6 个月（复议后 15 日） |
| 上诉（民事） | 判决 15 日 / 裁定 10 日 |
| 上诉（行政） | 15 日 |
| 复议申请 | 60 日 |
| 国家赔偿请求 | 2 年 |

## 八、与当事人沟通规范

### 8.1 沟通原则

```
- 避免承诺胜诉
- 避免评价其他律师
- 避免刺激性表述（C3 家事）
- 避免归责性语言
- 关键事项须书面回执
- 当日补记口头沟通（client-comms-log）
```

### 8.2 沟通分层

| 对象 | 语言 | 格式 |
|---|---|---|
| 当事人对外 | 通俗语言 | 书信/口头 |
| 督导对内 | 专业术语 | 备忘录 |
| 法院对外 | 规范格式 | 法律文书 |

## 十、与其他 plugin 的差异

| 维度 | legal-clinic | employment-legal | law-student |
|---|---|---|---|
| 目标用户 | 法援学生/律师/当事人 | 律师/HR | 学生 |
| 输出类型 | 法律服务 + 案件档案 | 法律意见 | 知识传递 |
| 数据需求 | 案件档案 + 当事人信息 + 法规 | 法规 + 实务案例 + 地方差异 | 教材 + 真题 + 案例库 |
| 时效要求 | 极高（仲裁时效 60 日） | 高（案件时效） | 中（教材稳定） |
| 风险等级 | 中-高（影响当事人） | 高（影响实际案件） | 低（学习无后果） |
| 督导机制 | 必须（supervisor-review-queue） | 无（律师自主） | 无（学生自主） |

**核心差异**：legal-clinic 的所有输出**必须经过督导复核**，这是与 employment-legal / law-student 的根本区别。
