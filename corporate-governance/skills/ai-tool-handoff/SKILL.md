# /ai-tool-handoff — China Mainland

## CN常用法律AI工具

| 工具 | 适用场景 | 擅长条款类型 |
|---|---|---|
| 幂律（MetaLaw）| 中文合同审查 | 控制权变更、转让限制、违约金 |
| 秘塔AI（MetaTation）| 中文合同审查 | 合同要素提取、风险识别 |
| 通义法务（AliLegal）| 中文合同审查 | 劳动人事、知识产权 |
| 华为云合同分析 | 批量合同分析 | 标准合同比对 |

## 工作流程

### Step 1: 判断是否适合批量处理

**适合批量处理的情形：**
- 合同数量 > 20份
- 提取目标是工具擅长的条款类型（控制权变更、转让限制、竞业限制、自动续期）
- 合同格式相对统一（标准模板合同，而非逐条谈判的复杂合同）

**不适合批量处理的情形：**
- 合同数量 < 20份（逐份阅读更快）
- 条款需要上下文判断（如"关联交易认定"）
- 合同高度定制化
- 侧函、谅解备忘录（须结合主合同判断）

---

### Step 2: 准备批次

**CN合同批次准备：**

| 检查项 | 说明 |
|---|---|
| 文档格式统一 | PDF/Word/图片，建议纯文本优先 |
| 合同命名规范 | `[序号]_[相对方]_[合同类型]_[年份]` |
| 批次大小 | 每批次不超过200份 |
| 分类标记 | 标注合同类型（采购/销售/租赁/劳动）|

**批次上传指引：**

```markdown
## 批量审查批次说明

**批次编号：** Batch-[序号]
**合同数量：** N份
**合同类型：** [采购/销售/租赁/服务/劳动/其他]
**审查工具：** [幂律/秘塔AI/通义法务/其他]
**提取目标：**
- 控制权变更条款
- 转让限制条款
- 自动续期条款
- 违约金上限
- 争议解决条款

**过滤条件：** 仅提取含目标条款的合同，无需对每份合同输出"未找到XX条款"

**返回截止：** YYYY-MM-DD
```

---

### Step 3: 质量核验（QA）

工具返回结果后，按以下标准核验：

**"Use as-is"（直接使用）：** 仅当 CLAUDE.md 明确说明时——极少使用。

**"Spot-check"（抽样核验）：**
- 随机抽取10%的输出结果
- 每份：原文引用 → 结论 → 核对是否一致
- 错误率 > 5% → 重新审查整批

**"Full re-review"（全面重审）：**
- 全部输出均需人工核对
- 适用于涉及重大风险的合同

##信任级别设置

在 `../CLAUDE.md` 中设置AI工具信任级别：

```yaml
ai_review_tools:
  - name: "[工具名称]"
    used_for: "[控制权变更/转让限制/全部]"
    trust_level: "spot-check"  # use-as-is / spot-check / full-re-review
    handoff_process:
      loader: "[谁上传]"
      qa_owner: "[谁核验]"
```

## 本技能不做什么

- 不代替律师判断。只提取条款线索。
- 不保证提取准确性。每格须人工核验。
- 不评估业务影响。只描述法律现象。
