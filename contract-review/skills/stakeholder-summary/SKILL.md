# Stakeholder Summary — China Mainland

## Purpose

提出这份合同的业务负责人不想要法律备忘录。他们想知道：能签吗，有什么坑，我需要做什么。本技能将完成的审查转化为他们需要的内容。

## 受众校准

读取 `../CLAUDE.md` → `## House style` → 利益相关方摘要发送给谁、多长。如未指定，默认：采购负责人或部门负责人，最多两段，不使用法律术语。

**不同受众需要不同摘要：**

| 受众 | 关注点 | 不关心的 |
|---|---|---|
| **采购** | 价格、续期机制、审批路由 | 赔偿上限结构 |
| **部门负责人（预算负责人）** | 团队能否使用、出问题怎么办、成本 | 赔偿范围 |
| **财务** | 总拥有成本、续期价格风险、表外承诺 | 适用法律 |
| **安全/IT** | 数据处理、子处理者、等保认证、数据存储位置 | 其他 |
| **高管发起人** | 是否会让我们尴尬、法律是否阻止 | 细节 |

如不明确，询问发送给谁。

---

## 摘要模板

**长度限制——严格执行：**

- **一段：** 结论和这是什么（商业条款，用通俗语言）
- **一段：** 风险——如果现在没人告诉他们，业务负责人以后会惊讶的事
- **2-3项清单：** 业务负责人实际需要做什么（最多三项）
- **一行收尾：** 审批时间

**总计不超过200字。**

---

## 输出格式

```markdown
## 合同摘要——[合同名称]

**[结论，可签/需修改/暂不签]**

[对手方]向您提供[产品/服务名称]。合同期[期限]，年费约¥[金额]。如不采取行动，将于[日期]自动续期。

**风险提示：**
[如不采取行动，业务负责人以后会惊讶的一件事]

**您需要做的：**
- [ ] [具体行动1，如"在本月15日前回复是否续期"]
- [ ] [具体行动2，如"通知法务批准修订条款"]
- [ ] [具体行动3，如"获取财务预算确认"]

**下一步：** [审批人]需在[日期]前确认。
```

---

## 本技能不做什么

- 不发送摘要。只草拟并展示。
- 不替代法律备忘录。摘要面向非法律背景人员，备忘录面向律师。
