# /escalation-flagger — China Mainland

## Purpose

根据 `../CLAUDE.md` 中的升级矩阵，为合同问题命名审批人，并草拟升级请求

## Instructions

1. **加载 `../CLAUDE.md`** → `## Escalation` 章节。如缺失，告知——需要编辑 practice profile。

2. **定性问题：** 金额阈值 / 条款偏离 / 自动触发 / 业务决策

3. **匹配矩阵，指定审批人。** 具体到人名或角色，而非"法务领导"。

4. **草拟请求：** 按模板输出：合同说什么、playbook说什么、选项及建议、决策截止日

5. **不发送。** 草拟后展示，由律师发送。

## 中国合同审批流程

**典型升级矩阵：**

| 审批人 | 无需升级的阈值 | 升级至 | 方式 |
|---|---|---|---|
| 法务专员 | 标准条款内的修改 | 法务总监 | 企业微信/邮件 |
| 法务总监 | 超过标准条款但非关键条款 | CEO/CFO | 邮件 |
| CEO/CFO | 关键条款、无限责任、IP全转让 | — | — |

**自动升级触发（无论金额）：**
- 无限责任条款
- 知识产权全转让给对方
- 适用美国法律或境外法院管辖
- 违约金超过合同总价50%

**金额触发升级：**
- 单笔合同金额超过人民币500万元
- 年度合同累计金额超过人民币1000万元

## 升级请求模板

```markdown
## 合同审批请求

**审批人：** [姓名/角色]
**截止日期：** [决策截止日]
**紧急程度：** [🔴高 / 🟡中 / 🟢低]

### 问题摘要

[一句话描述问题]

### 合同条款

> "[原文引用]"

### playbook 位置

**标准说：** [来自 `../CLAUDE.md`]
**实际说：** [合同实际条款]

### 风险评估

- **法律风险：** [🔴关键 / 🟠高 / 🟡中 / 🟢低]
- **业务影响：** [🔴阻止交易 / 🟠延迟交易 / 🟡需关注 / 🟢无感]

### 选项

**选项A：** [建议方案，如"接受当前条款"]
- 优势：[...]
- 风险：[...]

**选项B：** [备选方案，如"要求删除该条款"]
- 优势：[...]
- 风险：[...]

### 建议

**[建议选项]** — 原因：[...]

**需要您决策的问题：**
- [具体问题1]
- [具体问题2]
```

---

## 本技能不做什么

- 不发送升级消息。只草拟并展示。
- 不做决策。只将问题和建议呈现给审批人。
