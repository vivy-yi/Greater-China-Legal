# /trial-scheduling-and-deadline-monitoring — 庭审排期与程序时限监控

## 触发条件

满足任一即可：
- 提到"上诉期限"/"证据提交期限"/"送达期限"/"保全期限"
- 增加、修改、询问案件期限
- 要求交付程序时限监控一览表

## 排除情形

- ❌ 管理案件关键时间地点一览表（用 case-lifecycle-planning）
- ❌ 管理律师/助理/实习生工作时长
- ❌ 与案件期限排期无关的编程任务

---

## 核心法定期限（民事诉讼）

| 期限类型 | 法律依据 | 时长 | 起算点 |
|---------|---------|------|-------|
| 答辩期 | 民诉法130条 | 15日 | 送达后 |
| 举证期 | 民诉法68条 | 30日（可申请延长） | 送达后 |
| 上诉期 | 民诉法171条 | 15日 | 送达后 |
| 执行申请 | 民诉法239条 | 2年 | 判决生效后 |
| 保全续保 | 民诉法100条 | 届满前 | 保全期限届满前 |
| 公告送达 | 民诉法95条 | 30日 | 公告发出日 |

## 工作流程

### 情形1：增加法定期限

```
步骤1：创建/选择案件
  - 新增案件名称
  - 或通过案件名称选择已有案件

步骤2：输入已知时间起算点
  - 起诉材料送达法院时间
  - 立案时间
  - 裁判文书送达时间
  - 开庭时间
  - 保全时间

步骤3：计算法定期限节点
  根据《民事诉讼法》及相关司法解释计算

步骤4：确认计算结果
  向用户确认期限截止日期是否有误

步骤5：输出经确认的监控一览表
```

### 情形2：调取期限一览表

```
步骤1：检索涉及该期限的所有案件
  仅筛选截止日期在今日之后的案件

步骤2：检查是否缺失起始日期
  如有，向用户补充信息

步骤3：输出一览表
```

### 情形3：近日期限提醒

```
步骤1：询问筛选期限范围
  - 明确期限（如15日）
  - 或明确截止日期（如3月2日）

步骤2：以当前日期为节点筛选

步骤3：输出近日期限一览表
```

### 情形4：期限提醒

```
期限前5天/3天自动提醒
格式："注意：xx案件将于X日后（xx年xx月xx日）到达xx期限。"
```
