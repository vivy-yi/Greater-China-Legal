# /teleological-interpretation — 目的解释

## 触发条件

满足任一即可：
1. 文义解释存在两种以上可能，无法择一
2. 文义解释结论导致明显不合理结果
3. 条文含开放性评价概念（"合理""正当""必要""重大"）
4. 条文较旧，面对新型事实时文义覆盖不足

---

## 工作流程

### Phase 1：接收文义解释结果

```
从文义解释skill获取：
- 解释对象：具体条文/概念/句子
- 文义可能含义：[列表]
- 文义边界：文字射程的上下限
- 文义不足原因：歧义/抽象/不合理/新型事实
```

### Phase 2：识别目的

#### Step 1：寻找主观目的（立法者意图）

```
数据源优先级：
L1. 法条本身 → 用语/结构/位置/宗旨条款/章节标题
L2. 立法说明材料 → 草案说明/审议报告/修法说明
L3. 立法背景 → 当时要解决的现实问题/旧规则不足

输出格式：
"该条主观目的在于通过[手段]，防止/实现[具体效果]"
禁止口号式表述（"保护权益""维护秩序"）
```

#### Step 2：寻找客观目的（当下合理目的）

```
维度：
- 制度功能：该法条在整个法律制度中的角色
- 法律价值：诚实信用/公平/效率/安全
- 现实需求：当下社会实际需求
- 权威裁判：司法实践中的主流理解

判断一致/互补/冲突关系：
- 一致：叠加强化
- 互补：不同维度互补
- 冲突：分配权重
```

### Phase 3：目的转化为含义判断

```
将抽象目的转化为具体的条文含义判断
选择最符合目的的解释结论
```

### Phase 4：结论检验

```
- 文义限制：解释结论是否在文义射程内？
- 体系协调：是否与其他法条协调？
- 上位法原则：是否符合上位法的基本原则？
- 公开说服力：解释结论能否为当事人和社会接受？
```
