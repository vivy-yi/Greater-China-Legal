# /case-lifecycle-planning — 案件全周期规划

## 触发条件

满足任一即可：
- 提到"案件准备"/"案件准备规划"/"案件准备时间线"/"规划案件准备节奏"/"排开庭"
- 需要生成案件准备时间表或诉讼路线图
- 新增/询问/修改/删除案件的准备进度、开庭时间、庭前会议时间、案件进程
- 询问某月有多少案件处于准备阶段/开庭/执行

## 排除情形

- 不处理案件分配给律师、律师助理、实习生
- 不处理与案件管理无关的一般性编程任务

## 工作流程

```
步骤1：为案件命名
  包括当事人双方姓名及案由

步骤2：上传案件文件
  传票、仲裁通知书等，指定关注重点
  或直接说明案件的开庭时间地点或庭前会议的时间地点

步骤3：补充案件关键时间
  完整说明案件名，包括：
  - 立案时间
  - 庭前会议时间地点
  - 开庭时间地点
  - 强制执行时间等

步骤4：生成输出
  - 本案诉讼路线图
  - 本案关键时间一览表
  - 本月诉讼路线图
  - 本月关键时间一览表
```
