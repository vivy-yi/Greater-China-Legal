## 2. evidence-evaluation / 证据评估

### GCL版本
- `litigation-support/evidence-organizer`：按争议焦点组织证据，准备证据目录，输出格式规范。有举证责任分配（合同纠纷/侵权纠纷/劳动争议）。

### THUYRan版本
- evidence-evaluation：三性判断（真实性/合法性/关联性）+ 证明力评估（直接/间接/原始/传来）+ 证明标准（高度盖然性/排除合理怀疑/明显优势）+ 非法证据排除规则。

### 对比结论

| 维度 | GCL | THUYRan | 哪个更好 |
|---|---|---|---|
| 证据组织 | ✅ 按争议焦点分组，格式规范 | ❌ 无 | **GCL** |
| 举证责任 | ✅ 合同/侵权/劳动 | ❌ 无 | **GCL** |
| 证据三性评估 | ❌ 无 | ✅ 完整 | **THUYRan** |
| 证明力评估 | ❌ 无 | ✅ 直接/间接/原始/传来 | **THUYRan** |
| 证明标准 | ❌ 无 | ✅ 三种标准 | **THUYRan** |
| 非法证据排除 | ❌ 无 | ✅ 有 | **THUYRan** |

**融合方案：**
证据-organizer 升级为 evidence-assessment，保留组织功能，增加三性评估和证明力判断：

```markdown
## 证据三性评估（融入THUYRan）

### 真实性
- 书证：原件 vs 复印件 vs 复制件
- 物证：是否保持原始状态
- 证人证言：证人身份是否可信

### 合法性
- 取得方式是否合法（不得以严重侵害他人合法权益的方式取证）
- 形式是否合法（须公证/鉴定的是否有）
- 证人证言：须出庭作证

### 关联性
- 与待证事实是否有逻辑关联
- 关联程度：直接证明 vs 间接证明

## 证明力评估（融入THUYRan）

| 证据类型 | 证明力 |
|---|---|
| 原始证据 | 强于传来证据 |
| 直接证据 | 强于间接证据（但间接证据可形成锁链） |
| 公证文书 | 强于普通书证 |
| 鉴定意见 | 须质证后才能作为定案依据 |

## 证明标准检查（融入THUYRan）

- 民事案件：高度盖然性（优势证据）
- 刑事案件：排除合理怀疑
- 行政案件：明显优势标准

## 证据缺口识别（融入THUYRan）

识别现有证据链的缺口，标注：
- ⚠️ 缺失证据：[什么证据]
- 补充方式：[怎么补]
- 替代方案：[无法补时的替代]
```

| 论证强度评估 | ❌ 无 | ✅ 强/中/弱/存疑 | **THUYRan** |
| 反驳预判 | ❌ 无 | ✅ 有 | **THUYRan** |

**融合方案：**
strategy-designer 增加论证链构建段：

```markdown
## 论证链构建（融入THUYRan）

### 论证链格式
1. **主张**：我方核心诉求是什么
2. **前提**：支撑主张的大前提（法条）和事实基础
3. **支撑**：证据如何支撑事实，事实如何支撑法条适用
4. **反驳回应**：如果对方反驳，回应是什么
5. **结论**：综合论证后的最终结论

### 论证强度自检
- 强：大前提清晰 + 事实充分 + 证据链完整
- 中：大前提清晰但事实/证据有缺口
- 弱：大前提有争议或事实/证据严重不足
- 存疑：无法确定，需要进一步证据

### 薄弱环节识别
标注论证链中最可能被攻击的部分，并准备备用论证
```

**融合方案：**

```markdown
## 演绎推理格式（融入THUYRan）

在每个合法性判断中，显式展示P-F-C链：

**协商解除合法性判断**

- **大前提(P)**：[劳动法第36条/劳动合同法第36条]
  "协商解除须双方自愿协商一致"
  
- **小前提(F)**：[法律事实]
  "2024年1月15日，公司提出协商解除，员工于同月18日签署解除协议"
  "法律定性：双方协商一致解除"
  
- **结论(C)**：[法律效果]
  "协商解除有效，员工不享有违法解除赔偿金请求权"
  "补偿金额=N（按实际工作年限计算）"

**抗辩分析（否定后件式）**：
- 如果员工主张"协议是被胁迫签署的"
- **小前提(F')**："员工提交录音证据证明公司在签署前威胁'不签就没有离职证明'"
- **否定后件**："协商一致须双方自愿，胁迫下达成的'一致'不符合自愿要件"
- **结论(C')**："该协议可撤销，单位须承担违法解除赔偿责任"
```

### 三层事实结构
