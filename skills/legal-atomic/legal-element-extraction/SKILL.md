# /legal-element-extraction — 法律要素提取

> 场景通用原子skill
> 本skill是所有9个场景的输入预处理。任何场景收到非结构化叙述时，先调用本skill提取法律事实，再进入场景判断。

## 加载上下文

本skill为纯方法论skill，不依赖法域配置。
如需针对特定场景（如劳动仲裁/合同纠纷），在调用前读取对应场景的CLAUDE.md获取场景上下文。

## 一、三层事实结构

```
原始叙述（生活叙事）
    ↓ 提取与筛选
法律相关事实（具有规范意义的事实单元）
    ↓ 类型化与对应
要件事实（可直接对应法律构成要件的事实）
```

| 层次 | 特征 | 示例 |
|---|---|---|
| 原始叙述 | 口语化、情绪化、叙事性强 | "他一直拖着不还，这种人简直不讲信用" |
| 法律相关事实 | 与法律判断直接关联，剔除情绪和评价 | 借款5万元、约定1个月、逾期3个月、删除微信 |
| 要件事实 | 直接对应特定请求权或抗辩的构成要件 | 借贷合意成立→履行期限届满→未返还（违约） |

## 二、九类事实单元

所有法律事实均可归入以下九类：

| 事实类型 | 核心问题 | 典型内容 |
|---|---|---|
| 主体事实 | 何人？ | 行为人、受害人、合同当事人、代理人 |
| 行为事实 | 做了什么？ | 签约、支付、交付、通知、拒绝、催告 |
| 时间事实 | 何时？ | 合同成立、履行期限、损害发生、催告时间 |
| 地点事实 | 何地？ | 合同成立地、履行地、管辖连接点 |
| 对象事实 | 指向什么？ | 款项、货物、数据、作品、隐私信息 |
| 结果事实 | 产生什么后果？ | 未履行、瑕疵交付、损害发生、财产减少 |
| 因果关系事实 | 行为与结果有无联系？ | 是否导致某一结果，介入因素 |
| 主观事实 | 主观状态？ | 故意、过失、明知、应知、善意/恶意 |
| 程序事实 | 采取了什么程序？ | 催告、协商、申诉、报案、起诉 |

## 三、事实属性四分法

| 属性 | 特征 | 识别标志 | 处理方法 |
|---|---|---|---|
| 客观事实 | 可通过外部证据描述和证明 | 陈述具体行为、时间、地点、对象 | 直接提取，标注证据来源 |
| 主观判断 | 当事人基于自身感受的判断 | "我觉得""我认为""显然""肯定" | 剥离情绪，追问客观事实 |
| 法律评价 | 已带有规范归类或责任判断 | "构成欺诈""属于违约""存在恶意" | 还原为客观事实基础 |
| 背景信息 | 与法律判断无直接关联 | 当事人性格、无关的前因后果 | 剔除，仅保留必要背景 |

**关键原则：** "欺骗""恶意""明显违法"等评价性表述不能直接作为分析前提，必须还原为可被规范识别的客观事实。

## 四、工作流程

### 步骤1：切分最小事实单元

将原始叙述拆解为可独立识别、独立判断的最小事实单元。

- 按句子拆分复合叙事
- 每个单元应包含：谁（主体）+ 做了什么（行为）+ 何时（时间）+ 指向什么（对象）+ 产生什么（结果）
- 缺少要素的单元，标注"待补充"

### 步骤2：事实类型化归类

将切分出的单元按九类事实归类。

### 步骤3：属性辨析与净化

| 原始表述 | 属性判断 | 净化结果 |
|---|---|---|
| "他一直拖着不还" | 主观判断+法律评价 | 行为事实：履行期限届满后未履行；程序事实：债权人多次催告 |
| "卖家明明说是新的" | 主观判断（"明明"） | 行为事实：出卖人作出"商品为全新"表示；主观事实：出卖人明知或应知商品非全新 |
| "平台把我的信息泄露了" | 法律评价（"泄露"） | 行为事实：平台向第三方提供用户个人信息；对象事实：用户个人信息；结果事实：信息被第三方获取 |
| "这种人简直不讲信用" | 主观判断+情绪 | 背景信息 → 剔除 |

### 步骤4：生活语言→法律语言转化

| 生活语言 | 法律语言 |
|---|---|
| "他一直拖着不还" | "债务履行期限届满后，债务人未履行还款义务" |
| "卖家说是全新的" | "出卖人曾就标的物品质作出特定表示（标的物为全新）" |
| "平台把我的信息泄露了" | "信息处理者实施了向第三方提供个人信息的行为，该提供行为未经信息主体同意" |
| "每次问他都说再等等" | "债权人多次催告，债务人多次承诺履行但未实际履行" |
| "被公司开除了" | "用人单位单方解除劳动合同" |
| "公司没给我交社保" | "用人单位未依法缴纳社会保险" |

### 步骤5：对应法律构成要件

1. 识别案件涉及的可能法律关系（合同/侵权/不当得利/劳动争议）
2. 列出该法律关系的构成要件
3. 将事实逐一映射到各要件
4. 标注：✅满足 / ⚠️部分满足 / ❌缺乏

## 五、输出格式

### 形式A：结构化法律要素清单（默认）

```
[1-2句话概括核心内容]

### 九类事实提取

#### 主体事实
- [事实1]
- [事实2]

#### 行为事实
- [事实1]

#### 时间事实
- [事实1]

#### 对象事实
- [事实1]

#### 结果事实
- [事实1]

#### 因果关系事实
- [如有]

#### 主观事实
- [如有]

#### 程序事实
- [如有]

### 待补充事实
- [缺失但对法律分析重要的事实]

### 争议事实
- [叙述中存在矛盾或需证据证实的事实]
```

### 形式B：要件映射分析（当用户指定法律关系时）

```
## 要件映射分析

**法律关系**：[如：借款合同纠纷]
**请求权基础**：[如：《民法典》第675条]

| 构成要件 | 对应事实 | 满足状态 |
|---|---|---|
| 借贷合意 | 双方就5万元借款达成一致 | ✅ 满足 |
| 款项交付 | 出借人向借款人转账5万元 | ✅ 满足 |
| 履行期限届满 | 约定1个月，已逾期3个月 | ✅ 满足 |
| 未返还借款 | 借款人未返还任何款项 | ✅ 满足 |
```

### 形式C：时间轴（复杂案件）

```
## 法律时间轴

| 时间 | 行为主体 | 行为内容 | 涉及对象 | 关键证据 | 法律意义 |
|---|---|---|---|---|---|
| [日期] | [主体] | [行为] | [对象] | [证据] | [意义] |
```

## 六、示例

**输入：**
> "我去年借给朋友五万元，他当时说一个月后一定还我。后来过了三个月他都没还，每次问他都说再等等，最后把我微信删了。"

**输出：**
```
## 法律要素提取报告

### 原始叙述摘要
出借人向借款人出借5万元，约定1个月归还，逾期3个月未还，借款人删除出借人微信。

### 九类事实提取

#### 主体事实
- 出借人（叙述者）与借款人（朋友）成立借款关系
- 双方为自然人之间的借贷

#### 行为事实
- 双方就5万元借款达成合意
- 出借人向借款人交付借款（金额5万元）
- 借款人承诺1个月后归还
- 借款人在履行期限届满后未归还借款
- 出借人多次向借款人催告还款
- 借款人多次以"再等等"推诿
- 借款人删除出借人微信（断绝联系）

#### 时间事实
- 借款时间："去年"（需确认具体日期）
- 约定还款期限：1个月
- 逾期时间：3个月
- 诉讼时效起算点：约定还款期限届满之日

#### 对象事实
- 借款标的：人民币5万元

#### 结果事实
- 借款人未按约定期限返还借款
- 借款人与出借人失去联系

#### 因果关系事实
- 借款人未履行还款义务 → 出借人债权未实现

#### 主观事实
- 借款人明知还款义务（曾承诺还款）
- 借款人有履行能力但拒不履行（多次推诿后断绝联系）

#### 程序事实
- 出借人多次催告（可构成时效中断）
- 未提起诉讼或仲裁

### 待补充事实
1. 借款具体日期（影响诉讼时效计算）
2. 借款交付方式（现金/转账，影响证据证明力）
3. 是否有借条、欠条或借款合同
4. 借款人身份信息（起诉需要明确的被告）
5. 借款人是否有财产线索（影响执行）

### 争议事实
- 借款人删除微信的行为是否构成逃废债的恶意证据（需结合其他证据判断）

### 要件映射

**法律关系：** 自然人借贷纠纷
**请求权基础：** 《民法典》第675条（返还借款请求权）

| 构成要件 | 对应事实 | 满足状态 |
|---|---|---|
| 借贷合意 | 双方就5万元借款达成一致 | ✅ 满足 |
| 款项交付 | 出借人向借款人转账5万元 | ⚠️ 待补充（需确认转账记录） |
| 履行期限届满 | 约定1个月，已逾期3个月 | ✅ 满足 |
| 未返还借款 | 借款人未返还任何款项 | ✅ 满足 |
```

## 七、质量检查清单

- [ ] 原始叙述是否已切分为最小事实单元？
- [ ] 每个事实单元是否已明确行为主体？
- [ ] 全部事实是否已按九类事实类型归类？
- [ ] 客观事实、主观判断、法律评价、背景信息是否已区分？
- [ ] 法律评价性表述是否已还原为客观事实基础？
- [ ] 主观情绪词是否已剥离？
- [ ] 生活语言是否已转化为法律语言？
- [ ] 提取的事实是否已对应到具体法律构成要件？
- [ ] 待补充事实和争议事实是否已列明？
- [ ] 关键时间节点是否已提取？
