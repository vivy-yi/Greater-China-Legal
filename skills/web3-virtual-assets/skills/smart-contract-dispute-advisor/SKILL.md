# 智能合约纠纷咨询

## 一、基本信息

### 适用场景

本 skill 适用于以下智能合约相关纠纷：
- **合约漏洞导致资产损失**：闪电贷攻击、重入攻击、预言机操纵等
- **跨链桥/跨链协议攻击**：跨链桥被盗后的资产追索
- **Rug Pull 纠纷**：项目方撤走流动性或跑路
- **合约执行争议**：智能合约条款约定不明或未按预期执行
- **DAO 治理攻击**：利用治理机制漏洞通过的恶意提案

### 客户类型

- DeFi 协议用户/投资者
- 协议开发团队
- 项目方（遭遇攻击的）
- 保险公司（加密资产保险理赔）

## 二、分析框架

### 2.1 管辖权重叠

| 因素 | 影响 |
|------|------|
| 开发团队所在地 | 确定主要管辖法域 |
| 用户所在地 | 中国居民的救济路径受限 |
| 合约部署链 | 公链无单一管辖，仲裁条款可能内置 |
| 服务器/前端所在地 | 受当地数据保护与金融监管 |

### 2.2 中国居民的救济路径

| 路径 | 可行性 | 主要障碍 |
|------|--------|---------|
| 中国法院诉讼 | 低 | 虚拟货币交易不受法律保护（9·4公告） |
| 香港法院诉讼 | 中 | 须能确定被告在香港有资产 |
| 国际仲裁 | 中-高 | 须合约内置仲裁条款 |
| 链上取证+报警 | 低 | 中国警方对 DeFi 案件接纳度低 |

### 2.3 合约漏洞举证

```
举证责任：
  受害方须证明：
  1. 合约存在漏洞（代码审计报告或攻击交易分析）
  2. 损失由漏洞直接导致
  3. 损失金额可量化（链上交易记录）

证据类型：
  - 链上交易哈希（TXID）
  - 合约地址及源码（已验证的 Etherscan 源码）
  - 攻击交易分析报告（如 SlowMist/Trail of Bits 报告）
  - 时间戳公证（可选用中国法律认可的可信时间戳）
```

### 2.4 常见攻击类型与法律定性

| 攻击类型 | 技术描述 | 潜在法律定性 |
|---------|---------|-------------|
| 重入攻击 | 递归调用提取资金 | 盗窃/不当得利 |
| 闪电贷攻击 | 无抵押借贷操纵价格 | 市场操纵/不当得利 |
| 预言机操纵 | 操纵价格预言机数据 | 欺诈/市场操纵 |
| Rug Pull | 撤走流动性或 mint 大量代币 | 诈骗罪（刑法第266条）|

## 三、操作流程

### 第一步：事实采集

```
收集以下信息：
- 交易哈希（TXID）
- 合约地址
- 损失金额（USD 及对应的加密资产数量）
- 攻击时间线
- 用户所在地及身份
```

### 第二步：技术分析（建议委托专业审计公司）

```
- 攻击交易逆向分析
- 合约源码审计
- 攻击者地址追踪（Chainalysis/慢雾 MistTrack）
```

### 第三步：法律评估

```
技术分析结果 + 用户信息
  → 是否有可识别的攻击方？
    ├─ 是 → 评估诉讼/仲裁可行性
    └─ 否 → 评估链上治理恢复/保险理赔
  → 用户是中国居民？
    ├─ 是 → 救济路径极窄，重点放在链上恢复
    └─ 否 → 评估当地法院的加密案件受理态度
```

### 第四步：输出报告

```
## 智能合约纠纷评估报告

事件概述：[攻击类型/时间/损失金额]

救济路径分析：
1. [路径A] — [可行性评估]
2. [路径B] — [可行性评估]

推荐方案：[具体建议]

升级事项：[如需律师介入]
```

## 四、升级决策门

出现以下情形，将案件移交专业律师：
- 涉及刑事犯罪（诈骗/盗窃）
- 涉案金额巨大（>500万人民币）
- 涉及跨境诉讼/仲裁
- 用户试图通过中国法院诉讼追索加密资产
- 涉及与监管机构的正式沟通
- 涉及保险公司理赔争议
