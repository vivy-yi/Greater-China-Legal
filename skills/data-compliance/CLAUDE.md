## 覆盖的数据保护法规

### 主要法规

| 法规 | 适用场景 | 说明 |
|---|---|---|
| PIPL（个人信息保护法） | 中国大陆用户 | 2021年11月生效 |
| DNPA（数据安全法） | 中国大陆数据处理活动 | 2021年9月生效 |
| NISL（网络安全法） | 网络运营者 | 2017年6月生效 |
| GDPR | 欧盟用户 | 如有欧盟用户 |
| PDPA | 新加坡用户 | 如有新加坡用户 |

### 适用判断

```
产品面向群体 → 确定适用法规
中国大陆用户为主 → PIPL优先
同时有欧盟用户 → 加GDPR要求
同时有新加坡用户 → 加PDPA要求
```

## PIPL合规评估维度

### 五模块checklist

```
1. 合法性基础
   - 同意（须明确、具体、可撤回）
   - 合同必要（须与合同履行直接相关）
   - 法律义务（须为法定义务）
   - 紧急情况（须为保护生命财产）
   - 新闻报道（须为公益目的）
   - 学术研究（须去标识化）
   - 政府公开（须政府公开范围）

2. 告知义务
   - 告知内容完整（处理目的/方式/范围/存储地点）
   - 告知方式有效（不易忽视/易于访问）

3. 同意管理
   - 撤回同意的途径（与给予同意同等便捷）
   - 拒绝的同等便利（不得设置不合理条件）

4. 数据安全
   - 分类分级保护
   - 必要安全措施
   - 数据泄露通知（72小时内向CAC报告）

5. 主体权利
   - 查阅/复制（15日内响应）
   - 更正/补充（15日内响应）
   - 删除（法定情形下）
   - 转移（特定情形下）
   - 撤回同意（须同等便捷）
```

## 敏感个人信息处理规则

### 须单独同意的敏感类型（PIPL第28条）

| 类型 | 示例 | 额外要求 |
|---|---|---|
| 生物识别 | 人脸/指纹/虹膜 | 单独告知+单独同意 |
| 宗教信仰 | 宗教信仰信息 | 单独告知+单独同意 |
| 特定身份 | 公务员/党员信息 | 单独告知+单独同意 |
| 医疗健康 | 病史/体检数据 | 单独告知+单独同意 |
| 金融账户 | 银行账户/证券账户 | 单独告知+单独同意 |
| 行踪轨迹 | 位置/行动记录 | 单独告知+单独同意 |
| 未满14周岁 | 未成年人信息 | 须取得父母同意 |

## 数据出境合规

### 出境条件判断

| 条件 | 是否须出境合规 |
|---|---|---|
| 处理个人信息>100万人 | ✅ 须安全评估 |
| 敏感个人信息出境 | ✅ 须安全评估 |
| 累计出境>10万人一般信息 | ✅ 须安全评估 |
| 不满足以上 | 可选标准合同或认证 |

### 三种出境路径

| 路径 | 适用条件 | 主管机关 |
|---|---|---|
| 安全评估 | 须评估情形 | 网信办 |
| 标准合同 | 自愿选择 | 无须审批，备案 |
| 认证 | 自愿选择 | 认证机构 |
