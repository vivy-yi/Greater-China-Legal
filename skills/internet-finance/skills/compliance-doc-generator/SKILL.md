# /compliance-doc-generator — 金融合规文书生成

## 加载上下文

**首次使用时：** 读取 `../CLAUDE.md` 获取报告格式/监管要求。

## 工作流程

1. 读取 `../CLAUDE.md` → 获取文书格式/注意事项
2. 收集合规信息
3. 按模板生成文书
4. 输出报告

## 合规自查报告要素

```
一、基本信息
  - 公司名称/牌照/自查期间

二、合规治理情况
  - 合规部门设置
  - 合规负责人履职情况

三、业务合规情况
  - 牌照合规（是否超范围经营）
  - 利率合规（是否超过36%）
  - 信息披露（是否合规）
  - 催收合规（是否有投诉）

四、风险管控情况
  - 风控模型有效性
  - 数据安全措施
  - 消费者保护措施

五、发现问题
  - 已发现问题清单
  - 整改措施

六、整改计划
  - 整改措施 + 时间表
```

## 投诉处理报告要素

```
一、投诉概况
  - 总投诉量/同比/环比
  - 主要投诉类型

二、投诉分析
  - 重点问题分布
  - 根本原因分析

三、处理情况
  - 已处理投诉量
  - 平均处理时间
  - 调解成功率

四、整改措施
  - 产品层面整改
  - 流程层面整改
  - 人员培训
```
