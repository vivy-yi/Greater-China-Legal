# /infringement-detector — IP侵权类型判断

## 加载上下文

**首次使用时：** 读取 `../CLAUDE.md` 获取覆盖的IP类型/数据源/风险等级。

## 工作流程

1. 读取 `../CLAUDE.md` → 获取IP类型覆盖/维权路径规则
2. 分析涉嫌侵权的产品/行为
3. 识别IP类型（商标/专利/著作权/商业秘密）
4. 输出侵权类型判断和初步法律分析

## 侵权类型判断树

### 第一步：识别权利基础

```
询问：对方主张的是哪种权利？
→ 商标权 → 进入商标侵权判断
→ 专利权（发明/实用新型/外观设计）→ 进入专利侵权判断
→ 著作权 → 进入著作权侵权判断
→ 商业秘密（技术/经营信息）→ 进入商业秘密侵权判断
→ 不明确 → 列出可能的权利类型，建议进一步调查
```

### 第二步：商标侵权判断（商标法第57条）

```
同一种商品 + 相同商标 → 假冒注册商标（直接侵权）
类似商品 + 相同商标 → 容易导致混淆 → 侵权
销售假冒注册商标商品 → 销售侵权（不知者免责条件）
伪造/擅自制造他人注册商标 → 制造侵权
```

### 第三步：专利侵权判断

```
发明/实用新型：
  技术特征落入权利要求保护范围
  全面覆盖原则：所有权利要求特征都被实施
  等同原则：实质相同的方式实现同样效果

外观设计：
  与外观设计相同或近似
  产品种类相同或类似
```

### 第四步：著作权侵权判断

```
两步判断：
1. 接触可能性：被控侵权人有接触原作品的可能
2. 实质性相似：受保护的表达实质性相似

常见侵权形式：
  复制/发行/表演/信息网络传播
```

### 第五步：商业秘密侵权判断（反不正当竞争法第9条）

```
三项条件：
1. 信息是商业秘密（秘密性/价值性/保密措施）
2. 对方不正当获取/使用/披露
3. 给权利人造成损失

不正当手段：盗窃/贿赂/欺诈/胁迫/电子入侵
```
