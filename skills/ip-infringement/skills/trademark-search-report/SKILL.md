# 商标注册前检索报告

## 工作流程

1. 读取 `../CLAUDE.md` → 获取管辖地/数据源配置/置信度规则
2. 收集商标基本信息 → 商标名称/图形/申请类别/指定商品或服务
3. **近似商标检索** → 判断是否与在先商标构成近似（音/形/义三要素）
4. **尼斯分类分析** → 判断申请类别与引证商标类别的类似商品/服务关系
5. **驳回风险评估** → 综合近似程度+类别类似+在先权利状态给出风险等级
6. **布局建议** → 防御商标/联合商标/全类注册等策略建议
7. 输出检索报告

## 一、适用情形

本技能适用于以下场景：

| 场景 | 说明 |
|------|------|
| 商标申请前初步检索 | 正式提交申请前，评估注册可行性 |
| 商标驳回后分析 | 收到驳回通知书后，分析近似引证商标及应对策略 |
| 商标侵权风险评估 | 使用某商标前，评估是否侵犯他人注册商标权 |
| 竞争对手商标监测 | 监测同行业竞争对手的商标注册动态 |
| 商标转让/许可尽职调查 | 商标交易前的权利清晰度评估 |

## 二、近似商标判断标准

### 2.1 判断原则：音/形/义综合判断

商标近似是指文字、图形、字母、数字、三维标志和颜色组合等商标，在声音、视觉含义或整体结构上相似，容易导致公众混淆。

**判断三要素：**

```
┌─────────────────────────────────────────────────────────────┐
│                      商标近似判断三要素                       │
├─────────────────────────────────────────────────────────────┤
│  音近似：读音相同或相似（中文拼音/英文发音）                   │
│         例如："NIKE" vs "NAIK"（音近似）                    │
│              "小米" vs "小蜜"（音近似）                      │
├─────────────────────────────────────────────────────────────┤
│  形近似：字体、图案、结构、线条等视觉元素相似                  │
│         例如：两个花的图形标识过于近似                        │
│              "M" vs "∞" 形态相似                             │
├─────────────────────────────────────────────────────────────┤
│  义近似：含义相同或相近，在认知上容易产生联想                  │
│         例如："Tiger" vs "老虎"（英文vs中文义近似）          │
│              "长城" vs "GREAT WALL"（中英文义近似）          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 文字商标近似判断细则

#### 中文文字商标

| 情形 | 示例 | 判断结果 |
|------|------|----------|
| 字形完全相同 | "华为" vs "华为" | 近似 |
| 字形相似，整体无明显区别 | "康师傅" vs "康师博" | 近似 |
| 读音相同，字形不同 | "英雄" vs "英豪" | 可能近似（需结合核定商品判断） |
| 字数不同但含义相同 | "格力" vs "格力电器" | 可能不近似（需整体判断） |
| 常用词汇组合，无独创性 | "科技" vs "科技" | 不近似（缺乏显著特征） |

#### 英文文字商标

| 情形 | 示例 | 判断结果 |
|------|------|----------|
| 字母完全相同 | "APPLE" vs "APPLE" | 近似 |
| 字母相似，容易混淆 | "MICROSOFT" vs "MICROSOFT"（字体微调） | 近似 |
| 读音相同或相似 | "NIKE" vs "NAIK" | 近似 |
| 含义对应中文商标 | "TIGER" vs "虎牌" | 可能近似（义近似） |
| 常用单词，无独创性 | "BEST" vs "BEST" | 不近似（缺乏显著特征） |

#### 中文拼音商标

| 情形 | 示例 | 判断结果 |
|------|------|----------|
| 拼音完全相同 | "HUAWEI" vs "HUAWEI" | 近似 |
| 拼音相似，容易混淆 | "HUAWEI" vs "HUAVEI" | 近似 |
| 拼音对应相同汉字 | "HUAWEI"（华为）vs "HUAWEI"（话伟） | 近似 |

### 2.3 图形商标近似判断细则

| 判断因素 | 评估标准 |
|----------|----------|
| 整体构图 | 整体视觉效果是否相似 |
| 组成元素 | 是否使用了相同的创意元素或表现手法 |
| 视觉印象 | 给公众的直观印象是否相同或近似 |
| 独创性程度 | 独创性越高的图形，受保护范围越大 |

### 2.4 组合商标近似判断

组合商标（文字+图形）近似的判断规则：

```
判断流程：
1. 将组合商标拆分为文字部分和图形部分
2. 分别判断文字部分和图形部分是否近似
3. 若任一部分近似，整体可能被判定为近似
4. 若两部分近似程度差异大，整体可能不判定为近似
```

### 3.1 尼斯分类概述

### 3.2 类似商品/服务的判断标准

类似商品/服务是指在功能、用途、生产部门、销售渠道、消费对象等方面相同或具有密切关联的商品或服务。

**类似判断因素：**

| 判断因素 | 说明 |
|----------|------|
| 功能用途 | 商品/服务的用途是否相同或相似 |
| 生产部门 | 生产制造是否来自相同或相关行业 |
| 销售渠道 | 是否通过相同或可替代的销售渠道 |
| 消费对象 | 目标消费群体是否相同或重叠 |
| 行业惯例 | 是否属于同一行业或存在配套关系 |
| 竞争关系 | 是否存在替代或竞争关系 |

### 3.3 常见类似商品/服务群组

以下是高频涉及的类似群组关系：

```
类似群组示例：

【第9类】0901（计算机）+【第42类】4220（计算机编程）
  → 软件开发：第9类商品（计算机软件）与第42类服务（编程）类似

【第25类】2501（服装）+【第18类】1806（皮具】
  → 服装与皮具：消费对象相同，销售渠道重叠

【第30类】3001（咖啡）+【第32类】3202（啤酒饮料）】
  → 咖啡与饮料：均属饮品，功能用途相似

【第35类】3501（广告）+【第38类】3802（通信服务）】
  → 广告与电信：广告宣传通过电信网络传播，关联密切

【第43类】4301（餐饮）+【第30类】3007（方便食品）】
  → 餐饮服务与食品：餐饮服务消费食品，关联密切
```

## 四、驳回风险评估判断树

### 4.1 风险等级定义

| 风险等级 | 定义 | 建议 |
|----------|------|------|
| 高风险（红） | 与在先注册商标近似，且商品/服务类似，驳回可能性大 | 建议修改商标或放弃申请 |
| 中风险（黄） | 存在部分近似因素，或类别部分交叉，需进一步分析 | 建议做进一步检索或修改部分商品 |
| 低风险（绿） | 与在先商标差异明显，或不在同一类别 | 可以提交申请 |

### 4.2 判断树：驳回风险评估流程

```
第一步：确定申请类别
│
├── 申请类别：____________（尼斯分类号）
└── 指定商品/服务：____________

第二步：同类近似商标检索
│
├── 步骤2.1：在相同类别（同一尼斯分类）中检索在先商标
├── 步骤2.2：筛选相同或类似商品/服务上的注册商标
└── 判断结果：
    ├── 有完全相同商标 → 高风险（直接近似）
    ├── 有高度近似商标 → 进入第三步
    └── 无近似商标 → 低风险/中风险

第三步：跨类近似商标检索（驰名商标）
│
├── 判断申请商标是否具有较高知名度（是否为驰名的可能）
├── 若是：检索跨类保护范围
│   ├── 跨类保护范围：相同或类似商品/服务
│   └── 若存在跨类保护商标 → 高风险
└── 若否：跳过本步

第四步：图形要素检索（如申请商标包含图形）
│
├── 步骤4.1：提取图形要素关键词
├── 步骤4.2：图形检索系统检索相似图形
└── 判断结果：
    ├── 有高度相似图形 → 高风险
    └── 无高度相似图形 → 继续

第五步：综合评估风险等级
│
├── 高风险因素：
│   ├── 文字/图形与引证商标完全相同
│   ├── 读音相同且字形相似（文字商标）
│   ├── 图形整体视觉效果近似
│   ├── 商品/服务相同或类似
│   └── 存在跨类保护的驰名商标
│
├── 中风险因素：
│   ├── 文字/图形部分近似
│   ├── 类别存在交叉或类似群组关系
│   └── 商标显著性较弱（常用词汇）
│
└── 低风险因素：
    ├── 商标与引证商标差异明显
    ├── 商品/服务不在同一类似群组
    └── 引证商标已被撤销/无效/注销（需确认状态）

第六步：给出最终风险评估
│
├── 高风险（红）：建议修改商标设计或更换类别
├── 中风险（黄）：建议做答复审查意见准备或部分修改
└── 低风险（绿）：建议提交注册申请
```

## 五、具体检索步骤

### 5.1 信息收集步骤

**第一步：收集商标基本信息**
```
1.1 商标名称（中文/英文）
1.2 商标类型（文字/图形/组合/三维/颜色组合）
1.3 申请类别（尼斯分类号）
1.4 指定商品/服务清单
1.5 申请人信息（国内/国外/港澳台）
1.6 优先权信息（如有）
```

**第二步：明确检索目标**
```
2.1 确认检索数据库（CNIPA中国商标网/TMVIEW全球库）
2.2 确认检索范围（同类/全类/图形检索）
2.3 确认检索截止日期（在先权利状态）
```

### 5.2 近似判断具体步骤

**第三步：文字近似检索**

```
3.1 中文商标检索
    ├── 精确检索：商标名称全称
    ├── 分词检索：拆分为词根或单字
    ├── 变形检索：常见异体字/繁简体
    ├── 拼音检索：中文拼音首字母或全拼
    └── 干扰词检索：添加常见修饰词

3.2 英文商标检索
    ├── 精确检索：英文全称
    ├── 拆分检索：常见词根/词缀
    ├── 发音检索：音似英文单词
    └── 中文译名检索：常见中文翻译

3.3 判断标准应用
    ├── 音近似：读音相同或相似度 > 80%
    ├── 形近似：字形/图形相似度 > 80%
    ├── 义近似：含义相同或高度关联
    └── 综合判断：任一要素近似即可能构成近似
```

**第四步：图形近似检索**

```
4.1 图形要素提取
    ├── 识别图形的主要组成元素
    ├── 确定图形的构图特点
    └── 提取图形的关键特征词

4.2 图形检索方式
    ├── 要素检索：按图形要素分类检索
    ├── 颜色检索：区分颜色组合
    └── 结构检索：按整体结构分类

4.3 判断标准应用
    ├── 整体视觉效果是否近似
    ├── 是否使用了相同的创意元素
    └── 独创性程度评估
```

**第五步：类别类似判断**

```
5.1 确定类似群组
    ├── 依据《类似商品和服务区分表》
    ├── 参考尼斯分类注释
    └── 结合实际商业场景判断

5.2 类似关系评估
    ├── 同一类似群组 → 类似
    ├── 不同类似群组但关联密切 → 可能类似
    └── 无关联 → 不类似

5.3 交叉检索
    ├── 同一申请人同日申请情况
    ├── 证明商标/集体商标特殊情况
    └── 地理标志商标保护范围
```

### 5.3 风险评估具体步骤

**第六步：在先权利状态核查**

```
6.1 注册商标状态核查
    ├── 有效注册：专用权在有效期内
    ├── 有效期届满：可能已续展或注销
    ├── 撤销/无效程序中：权利不稳定
    └── 撤回/放弃：权利已消灭

6.2 驳回风险量化
    ├── 引证商标数量：越多风险越高
    ├── 近似程度：完全相同/高度近似/部分近似
    ├── 商品类似程度：相同/类似/不类似
    └── 综合打分：高风险≥6分/中风险3-5分/低风险≤2分
```

## 七、升级决策门

以下情况须移交专业商标律师处理：

| 情形 | 说明 | 优先级 |
|------|------|--------|
| 涉及商标驳回复审 | 收到驳回决定后，15日内需提出复审申请 | 紧急 |
| 涉及商标异议程序 | 他人对申请商标提出异议，需答辩 | 紧急 |
| 涉及无效宣告程序 | 引证商标被提出无效，需答辩 | 紧急 |
| 涉及法院诉讼 | 商标侵权民事诉讼或行政诉讼 | 紧急 |
| 涉及侵权投诉/行政处罚 | 收到侵权投诉或被行政机关查处 | 紧急 |
| 存在多个高风险引证商标 | 需综合评估应对策略 | 高 |
| 涉及驰名商标认定 | 需提供在先使用证据证明知名度 | 高 |
| 涉及马德里国际注册 | 涉及海外商标注册布局 | 中 |
