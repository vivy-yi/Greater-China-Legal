# /evidence-organizer — 证据组织整理

## 加载上下文

**首次使用时：** 读取 `../CLAUDE.md` 获取证据规则/举证责任分配。

## 工作流程

1. 读取 `../CLAUDE.md` → 获取证据规则/举证责任
2. **逐项评估证据 → 三性判断（真实性/合法性/关联性）+ 证明力评级** ← 新增
3. 按争议焦点分组
4. 识别证据缺口
5. 输出证据目录 + 评估报告

## 证据目录格式

```
证据目录

原告/反诉被告提交：

证据1：[证据名称]
  证据类型：[书证/物证/视听资料等]
  证据来源：[自行制作/第三方机构]
  证明内容：[具体证明事实]
  页码：[P1-P5]

证据2：[...]
```

## 按案件类型组织证据

### 合同纠纷证据组织

| 争议焦点 | 所需证据 |
|---|---|
| 合同成立 | 合同原件/往来邮件/签约视频 |
| 合同履行 | 发货单/验收单/付款记录/沟通记录 |
| 违约事实 | 违约行为证据/损失计算依据 |
| 损害赔偿 | 损失明细/鉴定意见/市场数据 |

### 侵权纠纷证据组织

| 争议焦点 | 所需证据 |
|---|---|
| 侵权行为 | 侵权现场照片/视频/鉴定意见 |
| 损害结果 | 医疗票据/损失评估/鉴定意见 |
| 因果关系 | 鉴定意见/专家意见 |
| 免责事由 | 不可抗力证据/受害人过错证据 |

### 举证责任分配

| 案件类型 | 举证方 |
|---|---|
| 合同纠纷 | 主张合同成立/生效/履行方 |
| 侵权纠纷（一般） | 原告：侵权+损害+因果关系 |
| 侵权纠纷（特殊） | 被告：举证无过错/免责事由 |
| 劳动争议 | 用人单位：解除合法性 |

## 证据三性评估框架（THUYRan方法论）

> 本步骤参考 THUYRan/Legal-Skills-Chinese/evidence-evaluation 设计

### 三性评估维度

```
证据能力（可采性）
├── 真实性：证据载体是否真实未被篡改 + 内容是否与客观事实相符
├── 合法性：取证主体/程序/形式/内容是否合法
└── 关联性：与待证事实是否存在逻辑关联（直接/间接/无关联）

证明力（证明程度）
├── 高：国家机关公文/公证文书/原始证据/与己不利陈述
├── 中：第三方机构出具/间接证据
└── 低：当事人制作/传来证据/与己有利陈述
```

### 证明标准

| 诉讼类型 | 证明标准 | 说明 |
|---|---|---|
| 刑事（公诉） | 排除合理怀疑 | 证据确实充分 |
| 民事（一般） | 高度盖然性 | 75%以上可能性 |
| 民事（欺诈/胁迫） | 高于高度盖然性 | 排除合理怀疑以下 |
| 行政 | 明显优势 | 行政机关举证 |

### 逐证据评估表

```
## 证据评估：[证据名称]

| 评估维度 | 结论 | 说明 |
|---|---|---|
| 真实性 | ✅真实/⚠️存疑/❌存疑 | 原件/复印件/无法核实 |
| 合法性 | ✅合法/⚠️瑕疵/❌违法 | 取证主体/程序/形式 |
| 关联性 | ✅直接/⚠️间接/❌无关联 | 与待证事实的证明关系 |
| 证明力 | ★★★★★★★★★☆ | 1-5星 |
| 置信度 | 确定/较确定/不确定 | 综合判断 |

### 致命问题
[如有：真实性存疑/取证违法/无关联性]

### 补强建议
[如需要：申请鉴定/公证/证人出庭/调取原始载体]
```

### 证据链完整性检查

```
□ 每项待证事实是否有证据覆盖？
□ 核心争点是否有直接证据支撑？
□ 证据链是否存在断裂？
□ 间接证据是否形成完整逻辑链？
□ 对方可能提出什么质证意见？
```

### 特殊证据评估要点

```
书证：原件 > 复印件（有原件核对）> 复印件（无法核对）
电子数据：公证保全/区块链存证 > 平台原始记录 > 截图打印件
证人证言：出庭作证 > 书面证言（需说明未出庭原因）
鉴定意见：省级以上资质鉴定机构 > 一般机构
```

## 电子证据处理

```
微信/QQ记录：
- 须证明账号归属（实名认证页面截图）

系统日志：
- 须证明系统正常运行
- 须证明日志未被篡改
```
