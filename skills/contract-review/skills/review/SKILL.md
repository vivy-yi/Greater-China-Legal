# /review — 合同审查主技能（中国大陆）

## 目的

根据 `../CLAUDE.md` 中的 playbook 审查收到的协议文本。从协议标题识别协议结构，选择适当子技能进行处理，并在 `confirm_routing` 启用时先确认用户再继续。最终将各子技能输出整合为一份结构化备忘录。

---

## 一、准备工作

### 1.1 加载 playbook

**必须步骤：** 先读取 `../CLAUDE.md`

- 如文件不存在或包含占位符（如 `[待补充]`、`[填写你的要求]`），**立即停止**，不要继续审查。
- 提示用户先运行 `/shared/agent-ops:cold-start-interview`，等待 playbook 完善后再进行审查。

**读取 playbook 后提取以下配置：**

| 配置项 | 来源 | 默认值 | 说明 |
|--------|------|--------|------|
| `confirm_routing` | `## Review preferences` 段落 | `true` | 路由前是否需用户确认 |
| `default_risk_tolerance` | `## Review preferences` 段落 | `medium` | 默认风险容忍度 |
| `signature_requirements` | `## Review preferences` 段落 | — | 签章要求（是否需公章/签字） |
| `escalation_threshold` | `## Review preferences` 段落 | — | 升级阈值（超出该风险级别则上报） |

### 1.2 确认路由（条件分支）

```
如果 confirm_routing == true：
    → 输出路由确认提示（见附录A），等待用户回复 "确认" 后继续
    → 用户拒绝：停止，输出"已取消审查"
    → 用户修改目标：重新评估路由

如果 confirm_routing == false：
    → 直接进入协议获取步骤
```

## 二、获取协议文本

### 2.1 输入处理优先级

| 输入形式 | 处理方式 |
|----------|----------|
| 文件路径（如 `/user/uploads/contract.pdf`） | 调用文件读取工具读取内容；如为 PDF 则先提取文本 |
| 合同编号（如 `VC-2024-089`） | 在指定合同库中查找对应文本 |
| 直接粘贴的文本 | 直接使用粘贴内容 |
| URL/云盘链接 | 尝试下载并提取文本；失败则提示用户提供原文 |

### 2.2 输入缺失处理

**如用户未提供任何输入：**

回复以下提示（中文）：

```
请提供需要审查的协议文本，可以通过以下方式之一提供：

1. 文件路径：例如 /user/uploads/contract.pdf
2. 合同编号：例如 VC-2024-089
3. 直接粘贴协议文本内容

收到后我将立即开始审查流程。
```

**等待用户提供输入后继续。**

## 三、读取文档结构——协议类型识别

### 3.1 先读标题

提取并记录以下信息：

- **主协议标题**（含中英文名称，如 "Master Services Agreement"）
- **所有附件/附录/补充协议标题**
- **协议版本号**（如有）
- **签订日期**（如有）

### 3.2 协议类型识别——判断树

```
                              ┌─────────────────────────────┐
                              │ 协议标题包含哪些关键词？     │
                              └──────────────┬──────────────┘
                                             │
           ┌──────────────────────┐  ┌──────┴───────┐  ┌──────────────────────┐
           │ NDA 类               │  │ SaaS 类      │  │ 供应商服务类          │
           │ 关键词：              │  │ 关键词：       │  │ 关键词：              │
           │ • Non-Disclosure      │  │ • SaaS         │  │ • Master Services     │
           │ • NDA                │  │ • Subscription  │  │ • Professional Svc    │
           │ • Confidentiality    │  │ • Cloud         │  │ • Statement of Work   │
           │   Agreement          │  │ • Online        │  │ • Consulting          │
           │ • 保密协议            │  │   Services     │  │ • 服务协议            │
           │ • 保密条款            │  │ • 软件即服务   │  │ • 合作协议            │
           └──────────┬───────────┘  └───────┬───────┘  └──────────┬────────────┘
                     │                       │                      │
                     ▼                       ▼                      ▼
               ┌───────────┐         ┌───────────┐         ┌───────────────┐
               │ nda-review│         │saas-msa-  │         │vendor-        │
               │           │         │review     │         │agreement-     │
               │           │         │           │         │review         │
               └───────────┘         └───────────┘         └───────────────┘
```

### 3.3 关键词匹配规则

**优先级说明：**

1. **主协议标题**优先于附件标题
2. 如主协议标题无法判断，按附件标题中最核心的关键词判断
3. 如关键词交叉（如 "SaaS NDA"），按主业务性质判断——SaaS 订阅协议用 `saas-msa-review`，纯 NDA 用 `nda-review`
4. 如仍无法判断，使用 `vendor-agreement-review`（通用供应商协议）

**具体匹配表：**

| 匹配条件（标题含） | 路由目标技能 | 说明 |
|--------------------|--------------|------|
| `Non-Disclosure`（作为主协议） | `nda-review` | 纯 NDA 协议 |
| `NDA`（作为主协议） | `nda-review` | 纯 NDA 协议 |
| `Confidentiality Agreement` | `nda-review` | 纯 NDA 协议 |
| `保密协议` | `nda-review` | 中文 NDA |
| `保密条款`（且无其他服务条款） | `nda-review` | 中文 NDA |
| `SaaS` | `saas-msa-review` | SaaS 订阅协议 |
| `Subscription Agreement` | `saas-msa-review` | SaaS 订阅协议 |
| `Cloud Services` | `saas-msa-review` | 云服务协议 |
| `Software as a Service` | `saas-msa-review` | SaaS 协议 |
| `Master Services Agreement` | `vendor-agreement-review` | 主服务协议 |
| `Professional Services` | `vendor-agreement-review` | 专业服务协议 |
| `Statement of Work` | `vendor-agreement-review` | 工作说明书 |
| `Consulting Agreement` | `vendor-agreement-review` | 咨询服务协议 |
| `服务协议` | `vendor-agreement-review` | 中文服务协议 |
| `合作协议` | `vendor-agreement-review` | 中文合作协议 |
| 无匹配关键词 | `vendor-agreement-review` | 默认走通用供应商审查 |

## 四、执行子技能审查

### 4.1 调用对应技能

根据上一步判断树的结果，调用对应技能：

| 技能名称 | 适用场景 | 输入 |
|----------|----------|------|
| `nda-review` | NDA/保密协议 | 协议全文文本 |
| `saas-msa-review` | SaaS/订阅协议 | 协议全文文本 |
| `vendor-agreement-review` | 供应商服务协议 | 协议全文文本 |

**调用格式：**
```
/skill:run skill=nda-review
输入参数：protocol_text=<协议全文>
```

### 4.2 多附件处理

如协议包含多个附件：

1. **主协议** → 调用对应主技能（如 `vendor-agreement-review`）
2. **附件1-NDA** → 路由至 `nda-review`
3. **附件2-SaaS** → 路由至 `saas-msa-review`

各附件审查**独立运行**，结果汇总至最终备忘录。

## 附录B：子技能说明

| 技能 | 功能 | 关键输出字段 |
|------|------|--------------|
| `nda-review` | 审查 NDA 保密条款 | 保密信息定义、保密期限、例外情形、违约救济 |
| `saas-msa-review` | 审查 SaaS/订阅协议 | 服务可用性、数据所有权、安全措施、责任限制、终止条款 |
| `vendor-agreement-review` | 审查供应商服务协议 | 服务范围、付款条件、知识产权、违约责任、争议解决 |

## 本技能不做什么

- **不做实质性法律分析**：仅路由至具体审查技能，由子技能完成实质分析
- **不提供法律意见**：输出为审查备忘录，非正式法律意见书
- **不处理诉讼文件**：本技能仅处理商务合同，不处理诉讼/仲裁文件
- **不代替律师**：高风险或复杂问题需升级至持证律师复核
