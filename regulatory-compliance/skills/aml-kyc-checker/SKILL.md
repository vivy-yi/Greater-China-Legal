## 一、核心判断框架

### 1.1 反洗钱核心义务

| 义务类型 | 具体要求 | 适用对象 |
|---|---|---|
| 客户身份识别（KYC） | 核实客户身份、了解交易目的、保存身份资料 | 金融机构、特定非金融机构 |
| 大额交易报告 | 单笔或累计达到规定金额的报告 | 金融机构 |
| 可疑交易报告 | 发现可疑交易主动上报 | 全体义务机构 |
| 客户身份资料保存 | 至少保存5年 | 全体义务机构 |
| 名单监控 | 制裁名单、高风险名单实时监控 | 金融机构 |

### 1.2 客户身份识别等级

| 客户类型 | 识别要求 | 强化识别情形 |
|---|---|---|
| 基础客户 | 基本身份信息 + 证件核实 | - |
| 较高风险客户 | 基本识别 + 职业背景 + 资金来源 | 政要人士、离岸公司 |
| 高风险客户 | 强化尽调（EDD）+ 高级审批 | 制裁名单匹配、涉敏地区 |

### 1.3 可疑交易识别标准

| 类型 | 典型特征 |
|---|---|
| 资金交易异常 | 短期内频繁大额进出、分散转入集中转出 |
| 客户行为异常 | 客户身份模糊、拒绝提供补充信息、实际控制人不明确 |
| 交易对手异常 | 与高风险地区/制裁国家交易、与黑名单客户交易 |
| 行业特征异常 | 交易模式与客户主营明显不符 |

## 三、分析流程

### 3.1 反洗钱合规评估步骤

```
第一步：制度审阅 → 审阅反洗钱内控制度文本的完备性
第二步：执行抽查 → 抽查客户身份识别记录、可疑交易报告
第三步：系统评估 → 评估可疑交易监测系统的有效性
第四步：人员访谈 → 了解一线人员对反洗钱的理解和执行
第五步：差距分析 → 对标监管要求识别合规缺口
第六步：风险评级 → 评估整体反洗钱风险等级
第七步：整改建议 → 提出改进建议和优先级
```

### 3.2 反洗钱内控制度要素清单

| 要素 | 说明 |
|---|---|
| 组织架构 | 反洗钱领导小组、合规部门职责 |
| olicies | 客户身份识别、可疑交易识别、名单监控等 |
| 流程制度 | 各环节操作规程、审批流程 |
| 系统建设 | 客户风险分类、交易监测、名单扫描 |
| 培训体系 | 员工培训频率、内容、考核 |
| 审计机制 | 内部审计频次、覆盖范围 |
| 应急预案 | 可疑情况处置、监管部门沟通 |

### 3.3 客户身份识别核查要点

| 核查项 | 自然人客户 | 法人客户 |
|---|---|---|
| 基本信息 | 姓名、身份证号、联系方式 | 名称、统一社会信用代码 |
| 身份证明 | 身份证件核实 | 注册证书、营业执照 |
| 地址信息 | 通讯地址 | 注册地址、经营地址 |
| 受益人 | - | 实际控制人、股权结构 |
| 交易目的 | 职业、交易预期 | 主营业务、经营范围 |
| 风险分类 | 低/中/高风险评级 | 低/中/高风险评级 |

## 升级决策门

以下情形应向上级 Skill 升级：

- 发现重大洗钱风险或涉及制裁逃避
- 涉及跨境洗钱、疑似恐怖融资
- 监管检查已发现问题需要整改应对（→ enforcement-response-advisor）
- 反洗钱制度需要系统性重塑（→ compliance-gap-assessor）
- 发现客户涉及重大违规需要冻结资产/解除关系

## 本Skill不涵盖

- 代替企业向反洗钱监测中心提交可疑交易报告
- 提供具体的可疑交易监测系统技术方案
- 代理企业参加反洗钱行政处罚听证
- 涉及反洗钱案件的刑事辩护
- 具体的名单扫描系统技术实现
## 本 Skill 不涵盖

- 可疑交易报告的具体格式文本
- 反洗钱调查的陪同应对
- 跨境汇款的具体合规审查
- 刑事责任的辩护
