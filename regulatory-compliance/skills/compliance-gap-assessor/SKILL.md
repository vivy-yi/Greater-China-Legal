## 一、核心判断框架

### 1.1 合规评估维度

| 评估维度 | 核心问题 | 典型监管要求 |
|---|---|---|
| 组织架构 | 合规部门是否独立/有足够授权？ | 合规负责人设立、合规部门配置 |
| 制度建设 | 制度是否覆盖全部业务和监管要求？ | 内控制度、合规管理程序 |
| 流程执行 | 制度是否真正落地执行？ | 审批流程、留痕管理 |
| 人员能力 | 从业人员是否具备合规意识与技能？ | 培训要求、考核机制 |
| 技术系统 | 能否有效支撑合规管控？ | 数据报送、系统监控 |
| 文档管理 | 是否留存合规证据？ | 档案管理、备查制度 |

### 1.2 合规差距等级

| 差距等级 | 描述 | 监管风险 |
|---|---|---|
| 红色（严重） | 存在重大违规事实或系统性缺失 | 可能触发处罚或业务暂停 |
| 橙色（较高） | 存在多项制度缺失或执行不到位 | 整改通知/警示函 |
| 黄色（中等） | 个别流程存在缺陷 | 监管关注但尚未违规 |
| 绿色（正常） | 符合监管要求 | 无额外关注 |

### 1.3 合规成熟度模型

| 级别 | 特征 |
|---|---|
| 初始级 | 被动应对，无系统性合规管理 |
| 基础级 | 有基本制度但执行参差不齐 |
| 规范级 | 制度完善且能有效执行 |
| 优化级 | 持续改进，主动预防合规风险 |
| 卓越级 | 行业标杆，合规文化深入全员 |

## 三、分析流程

### 3.1 合规差距评估步骤

```
第一步：范围确认 → 确定评估涉及的业求领域和监管要求
第二步：文件审阅 → 收集并审阅企业内部合规制度文件
第三步：对照核查 → 将企业实践与监管要求逐项比对
第四步：差距标注 → 按严重程度标注每项差距
第五步：根因分析 → 分析差距产生的根本原因
第六步：优先级排序 → 基于风险程度和整改难度排序
第七步：整改建议 → 提出具体可执行的整改措施
```

### 3.2 合规风险热力图

```
                    整改难度
              低　　　　中　　　　高
风险　高　│ 紧急整改　 │ 重点关注　 │ 战略规划　 │
程　　 中　│ 短期整改　 │ 中期整改　 │ 长期跟进　 │
度　　低　│ 常规维护　 │ 日常改进　 │ 持续优化　 │
```

### 3.3 整改计划要素

| 要素 | 内容 |
|---|---|
| 整改事项 | 具体差距项描述 |
| 责任部门 | 主责/配合部门 |
| 整改措施 | 具体行动项 |
| 完成时限 | 监管要求的截止日期 |
| 资源需求 | 人员/系统/预算 |
| 验收标准 | 整改完成的可量化指标 |

## 升级决策门

以下情形应向上级 Skill 升级：

- 差距评估发现重大系统性违规（→ enforcement-response-advisor 联合应对）
- 涉及多个行业的综合合规评估（→ industry-specific-compliance）
- 需要制定年度合规计划或合规管理体系建设方案
- 涉及 IPO、重大资产重组等资本运作中的合规整改
- 合规差距需要引入外部专业机构协助整改

## 本Skill不涵盖

- 代替企业执行具体的整改行动
- 提供整改所需的专项法律意见书
- 设计和实施合规管理信息系统
- 具体的财务审计或会计处理建议
- 整改效果的事后验证（需由外部审计或监管确认）
## 本 Skill 不涵盖

- 具体的整改方案文本
- 内控制度的设计
- 第三方审计机构的聘请建议
- 监管检查的陪同应对
