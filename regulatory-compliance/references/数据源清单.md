## 四、合规数据源（公司内部）

### 4.1 内部合规文件（BD）

| 文件类型 | 用途 | 合规核查 |
|---|---|---|
| 营业执照 | 经营范围 | 经营范围 vs 实际业务 |
| 许可证/资质证书 | 证照完备性 | 有效性 |

| 内部控制制度 | 合规管理有效性 | 制度完备性 + 执行情况 |

| 财务凭证 | 财务合规 | 税务/资金流 |

### 4.2 内部合规档案管理

```
合规档案结构：
compliance/
├── _log.yaml                    # 合规检查日志
├── _index.yaml                  # 合规事项索引
├── [check-id]/                  # 每次合规检查
│   ├── checklist.md            # 检查清单

    └── [year]-report.md
```

## 八、来源标注规范

每个结论/数字必须挂载来源标签：

| 标注 | 含义 | 合规场景示例 |
|---|---|---|
| `[YD]` | 元典检索结果 | `[YD] 商业银行法第 X 条` |
| `[WKL]` | 无合力检索结果 | `[WKL] 行政处罚案例 XX 号` |
| `[BD]` | 北达/内部档案 | `[BD] 公司内控制度文件 X 号` |
| `[GOV]` | 政府平台数据 | `[GOV] 银保监 2024 年 X 号公告` |
| `[web]` | 网络搜索结果 | `[web] 央行 2024 反洗钱新规` |
| `[model]` | 模型知识（须核实） | 不允许直接引用，须经合规顾问审核 |
| `[user]` | 用户提供的文件 | `[user] 客户提供的内控制度` |
| `[settled — last confirmed YYYY-MM-DD]` | 已验证的稳定引用 | 已合规顾问审核的引用 |

**合规场景特殊规则**：
- 监管法规引用**必须**用最新版本（[YD] + [BD] 验证）
- 处罚案例引用**必须**有原文（[GOV] 处罚公示）
- 内部合规文件标注 `[BD]` + 文件编号
- 监管动态必须 [web] 时效核验

## 十、监管合规风险等级说明

regulatory-compliance 风险等级由 skill 内部设定：

| skill | 风险等级 | 原因 |
|---|---|---|
| aml-kyc-checker | high | 反洗钱合规是金融机构命脉 |
| compliance-gap-assessor | medium | 合规评估 |
| enforcement-response-advisor | high | 监管处罚应对涉及法律风险 |
| industry-specific-compliance | medium-high | 取决于具体行业 |
| license-eligibility-checker | high | 无证经营 = 🔴 严重违法 |
| regulatory-change-tracker | low | 监测类 |

**触发升级**：
- 反洗钱违规 → 律师 + 合规顾问
- 重大行政处罚应对 → 律师主导
- 刑事风险 → 律师 + 合规
- 涉外合规 → 律师专项
