## 二、行业监管体系全景

### 2.1 典型行业监管体系

| 行业 | 监管机构 | 核心法规 | 许可证/资质 |
|---|---|---|---|
| 商业银行 | 国家金融监督管理总局 | 《商业银行法》《银行业监督管理法》 | 金融许可证 |
| 证券公司 | 证监会 | 《证券法》《证券公司监督管理条例》 | 证券经营许可证 |
| 保险公司 | 国家金融监督管理总局 | 《保险法》 | 保险经营许可证 |
| 基金管理公司 | 证监会 | 《基金法》《证券投资基金管理公司管理办法》 | 公募/私募管理人登记 |
| 信托公司 | 国家金融监督管理总局 | 《信托法》《信托公司管理办法》 | 信托业务许可证 |
| 上市公司 | 证监会/交易所 | 《证券法》《上市公司信息披露办法》 | IPO/再融资许可 |
| 医药企业 | 国家药监局 | 《药品管理法》《医疗器械监督管理条例》 | GMP/GSP/注册证 |
| 房地产开发 | 住建部/地方 | 《城市房地产管理法》 | 房地产开发资质 |
| 民办教育 | 教育部/地方 | 《民办教育促进法》 | 办学许可证 |
| 电信业务 | 工信部 | 《电信条例》 | 电信业务经营许可证 |
| 互联网信息服务 | 网信办/工信部 | 《网络安全法》《互联网信息服务管理办法》 | ICP 许可证 |
| 数据处理 | 网信办 | 《数据安全法》《个人信息保护法》 | 数据出境安全评估 |
| 环境保护 | 生态环境部 | 《环境保护法》《环境影响评价法》 | 排污许可证 |
| 危险化学品 | 应急管理部/生态环境部 | 《危险化学品安全管理条例》 | 危化品经营许可证 |
| 安全生产 | 应急管理部 | 《安全生产法》 | 安全生产许可证 |

### 2.2 监管层级

```
国家层面
├── 全国人大及常委会 → 法律
├── 国务院 → 行政法规
├── 国务院各部委 → 部门规章、规范性文件
└── 国务院直属机构 → 行业规范

地方层面
├── 省级人大及常委会 → 地方性法规
├── 省级政府 → 地方政府规章
└── 地方监管部门 → 规范性文件

注：地方性法规/规章不得与上位法抵触
```

## 三、合规风险判断树

```
[节点 1] 业务范围识别
├─ 是否属于许可/备案经营范围？
│   ├─ 是 → 进入许可合规检查
│   └─ 否（超范围经营）→ 🔴 严重违规（可能吊销执照）
│
├─ 是否属于前置审批/许可行业？
│   ├─ 是 → 进入前置审批检查
│   └─ 否 → 进入一般合规检查

[节点 2] 合规差距识别
├─ 制度建设
│   ├─ 是否有内控制度？ → 不齐 → 🟡 中风险
│   ├─ 制度是否更新？ → 滞后 → 🟡 中风险
│   └─ 制度是否执行？ → 流于形式 → 🔴 高风险
│
├─ 人员配置
│   ├─ 是否配备合规负责人？ → 缺失 → 🟡 中风险
│   ├─ 合规人员是否具备资格？ → 不齐 → 🟡 中风险
│   └─ 培训是否到位？ → 不足 → 🟢 低风险
│
├─ 系统支持
│   ├─ 合规管理系统？ → 缺失 → 🟢 低风险
│   └─ 数据留痕？ → 缺失 → 🟡 中风险
│
└─ 应急响应
    ├─ 应急预案？ → 缺失 → 🔴 高风险
    └─ 应急演练？ → 不足 → 🟡 中风险

[节点 3] 监管处罚历史
├─ 3 年内无处罚 → 良好
├─ 3 年内有警告 → 需关注
├─ 3 年内有罚款 → 需整改
└─ 3 年内有重大处罚 → 🔴 高风险（可能影响业务资格）
```

## 四、6 大场景核心判断

### 4.1 反洗钱合规（aml-kyc-checker）

```
[节点 1] 是否适用反洗钱义务？
├─ 是（金融机构/特定非金融）→ 进入反洗钱合规检查
└─ 否 → 退出

[节点 2] 反洗钱内控制度？
├─ 客户身份识别制度 → 缺失？🔴
├─ 客户身份资料和交易记录保存 → 缺失？🔴
├─ 大额交易报告（50 万人民币）→ 未报？🔴
├─ 可疑交易报告 → 未报？🔴
└─ 名单监控（制裁名单/PEP）→ 未监控？🔴

[节点 3] 客户尽职调查（KYC）？
├─ 简化尽调（低风险客户）→ 是否合规？
├─ 标准尽调（一般客户）→ 是否完整？
└─ 强化尽调（高风险客户/PEP）→ 是否触发？

升级触发：
- 未建立反洗钱内控 → 🔴 重大违规
- 未报大额/可疑交易 → 🔴 重大违规
- 客户身份识别不完整 → 🟡 中等违规
```

### 4.2 合规差距评估（compliance-gap-assessor）

```
[节点 1] 合规范围
├─ 业务合规 → 业务模式合规性
├─ 财务合规 → 税务/财务/审计
├─ 数据合规 → 数据安全/隐私保护
└─ 行业合规 → 行业特定要求

[节点 2] 评估方法
├─ 文档审查 → 制度文件/合同/记录
├─ 现场访谈 → 关键人员/业务部门
├─ 系统核查 → IT 系统/数据流
└─ 比对法规 → 与法律法规比对

[节点 3] 差距分级
├─ 🔴 重大差距（违反效力性强制规定）
├─ 🟡 中等差距（违反管理性规定/制度不完善）
├─ 🟢 轻微差距（流程瑕疵）
└─ 改进建议（按优先级）

输出：差距清单 + 整改路线图 + 时间表
```

### 4.3 监管执法应对（enforcement-response-advisor）

```
[节点 1] 执法类型
├─ 日常检查 → 配合提供材料
├─ 专项检查 → 重点准备
├─ 举报调查 → 主动配合
└─ 立案调查 → 严肃应对（建议律师介入）

[节点 2] 应对策略
├─ 行政检查
│   ├─ 准备材料清单
│   ├─ 现场配合人员安排
│   ├─ 法律意见准备
│   └─ 整改承诺准备
│
├─ 行政处罚
│   ├─ 听证申请（处罚决定前）
│   ├─ 陈述申辩
│   └─ 行政复议/诉讼（60 日内）
│
└─ 刑事移送
    ├─ 立即律师介入
    └─ 配合调查

升级触发：
- 涉及刑事风险 → 律师主导
- 涉及重大经营影响 → 律师 + 合规顾问
```

### 4.4 行业特定合规（industry-specific-compliance）

跨行业合规要点（详见 §二、行业监管体系全景）：

| 行业 | 关键合规点 |
|---|---|
| 金融业 | 牌照 + 资本充足率 + 关联交易 + 客户保护 |
| 医药业 | GMP/GSP + 注册申报 + 不良反应 + 反商业贿赂 |
| 电信互联网 | ICP/EDI 许可证 + 个人信息保护 + 数据出境 + 网络安全 |
| 房地产 | 开发资质 + 预售许可 + 资金监管 + 工程验收 |
| 教育 | 办学许可 + 招生宣传 + 收费规范 + 消防/卫生 |

### 4.5 许可证完备性核查（license-eligibility-checker）

```
[节点 1] 业务范围 vs 许可证
├─ 业务范围明确 → 查对应许可证
├─ 业务范围变更 → 是否需要变更许可
└─ 业务范围超出许可 → 🔴 无证经营

[节点 2] 许可证状态
├─ 现有许可证 → 是否在有效期
├─ 即将到期 → 是否安排续期
├─ 已过期 → 🔴 无证经营
└─ 即将到期 → 🟡 需续期

[节点 3] 许可证要求
├─ 法人资格 → 公司章程
├─ 注册资本 → 实际到位
├─ 经营场所 → 实际使用
├─ 人员资质 → 从业资格
└─ 设施设备 → 符合要求

升级触发：
- 无证经营 → 🔴 立即停业 + 律师
- 许可证过期 → 🔴 立即补办
- 许可证变更未办 → 🟡 限时整改
```

### 4.6 监管政策动态追踪（regulatory-change-tracker）

```
[节点 1] 监测范围
├─ 法律法规（人大/国务院）
├─ 部门规章（各部委）
├─ 行业规范（行业协会）
├─ 监管动态（处罚案例/检查通报）
└─ 司法解释（最高法/最高检）

[节点 2] 监测频率
├─ 高频更新：金融/数据/反垄断（每周）
├─ 中频更新：医药/环境/教育（每月）
└─ 低频更新：一般行业（每季度）

[节点 3] 影响评估
├─ 新法规 → 影响业务？影响范围？
├─ 修订法规 → 是否需要调整制度？
├─ 废止法规 → 是否需要清理依据？
└─ 处罚案例 → 是否涉及同类业务？

输出：监管动态月报 + 合规影响评估
```

## 五、监管处罚应对时间表

### 5.1 时效规定

| 事项 | 时效 | 法条 |
|---|---|---|
| 行政处罚决定 | 6 个月内发现 | 行政处罚法第 36 条 |
| 听证申请 | 收到通知 5 日内 | 行政处罚法第 64 条 |
| 复议申请 | 知道作出 60 日内 | 行政复议法第 60 条 |
| 行政诉讼 | 6 个月 | 行政诉讼法第 46 条 |
| 复议后起诉 | 15 日 | 行政诉讼法第 46 条 |

### 5.2 应急响应清单

```
收到检查通知（24 小时内）：
  - 确认检查范围 + 时间
  - 通知律师 + 合规负责人
  - 准备基础材料
  - 安排配合人员

收到处罚事先告知（5 日内）：
  - 评估处罚合理性
  - 准备陈述申辩材料
  - 申请听证（如适用）
  - 准备整改方案

收到处罚决定书（60 日内）：
  - 评估是否复议/诉讼
  - 准备复议/诉讼材料
  - 同时准备整改执行
  - 评估合规体系完善
```

## 六、合规风险等级矩阵

### 6.1 风险等级判断

| 风险因素 | 高 | 中 | 低 |
|---|---|---|---|
| 法规层级 | 法律/行政法规 | 部门规章 | 行业规范 |
| 处罚后果 | 吊销许可/重大罚款 | 罚款/警告 | 责令改正 |
| 业务影响 | 影响业务存续 | 影响业务开展 | 影响轻微 |
| 合规差距 | 完全缺失 | 部分缺失 | 流程瑕疵 |
| 监管动态 | 重点监管行业 | 一般监管 | 弱监管 |

### 6.2 风险等级综合评估

```
高风险 + 高影响 → 🔴 重大合规风险（建议律师专项）
中风险 + 中影响 → 🟡 中等合规风险（合规顾问介入）
低风险 + 低影响 → 🟢 轻微合规风险（自查整改）
```

## 七、合规体系成熟度模型

```
Level 1 - 被动合规（最低）
  ├─ 出现问题才应对
  ├─ 无系统合规管理
  └─ 高风险

Level 2 - 制度合规
  ├─ 建立基本合规制度
  ├─ 配备合规人员
  └─ 中风险

Level 3 - 主动合规
  ├─ 定期合规审查
  ├─ 主动识别风险
  └─ 低风险

Level 4 - 战略合规（最高）
  ├─ 合规融入业务战略
  ├─ 全员合规文化
  └─ 极低风险
```
